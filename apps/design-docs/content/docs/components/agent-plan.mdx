---
title: AgentPlan
description: Hierarchical task planning UI for AI agents with expandable tasks, subtasks, status indicators, and MCP tool tags.
---

## Usage

```tsx
import { AgentPlan } from "@nebutra/ui";
import type { AgentTask } from "@nebutra/ui";

const tasks: AgentTask[] = [
  {
    id: "1",
    title: "Research Requirements",
    description: "Gather and analyze project requirements from stakeholders",
    status: "in-progress",
    subtasks: [
      {
        id: "1.1",
        title: "Interview stakeholders",
        description: "Schedule and conduct 30-min interviews",
        status: "completed",
        tools: ["meeting-scheduler", "notes"],
      },
      {
        id: "1.2",
        title: "Review existing docs",
        description: "Read through all design docs and specs",
        status: "in-progress",
        tools: ["file-system", "browser"],
      },
    ],
  },
];

<AgentPlan
  tasks={tasks}
  defaultExpandedTasks={["1"]}
  onTaskStatusChange={(id, status) => console.log(id, status)}
/>
```

## Examples

### Multi-phase AI Workflow

```tsx
const workflowTasks: AgentTask[] = [
  {
    id: "plan",
    title: "Plan Architecture",
    description: "Design the system architecture",
    status: "completed",
    subtasks: [
      {
        id: "plan-1",
        title: "Define API contracts",
        description: "Draft OpenAPI spec",
        status: "completed",
        tools: ["code-editor", "openapi"],
      },
    ],
  },
  {
    id: "impl",
    title: "Implement Backend",
    description: "Build the API endpoints",
    status: "in-progress",
    dependencies: ["plan"],
    subtasks: [
      {
        id: "impl-1",
        title: "Database schema",
        description: "Create migration files",
        status: "completed",
        tools: ["postgres", "prisma"],
      },
      {
        id: "impl-2",
        title: "API routes",
        description: "Implement CRUD endpoints",
        status: "in-progress",
        tools: ["node", "express"],
      },
    ],
  },
  {
    id: "test",
    title: "Write Tests",
    description: "Unit and integration tests",
    status: "pending",
    dependencies: ["impl"],
    subtasks: [],
  },
];

<AgentPlan
  tasks={workflowTasks}
  defaultExpandedTasks={["impl"]}
  allowStatusToggle
/>
```

### Read-only Status Display

```tsx
<AgentPlan
  tasks={executedTasks}
  allowStatusToggle={false}
  defaultExpandedTasks={executedTasks.map((t) => t.id)}
/>
```

### Custom Tools Label

```tsx
<AgentPlan
  tasks={tasks}
  toolsLabel="Tools used:"
/>
```

## Status Types

| Status | Icon | Description |
|--------|------|-------------|
| `pending` | Circle (muted) | Task not yet started |
| `in-progress` | CircleDotDashed (blue) | Currently executing |
| `completed` | CheckCircle2 (green) | Successfully finished |
| `need-help` | CircleAlert (yellow) | Blocked or requires human input |
| `failed` | CircleX (red) | Execution failed |

## Props

### AgentPlan

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tasks` | `AgentTask[]` | — | Array of tasks to display |
| `onTaskStatusChange` | `(taskId: string, newStatus: TaskStatus) => void` | — | Callback when task status cycles |
| `onSubtaskStatusChange` | `(taskId: string, subtaskId: string, newStatus: TaskStatus) => void` | — | Callback when subtask status toggles |
| `defaultExpandedTasks` | `string[]` | `[]` | Task IDs to expand on initial render |
| `allowStatusToggle` | `boolean` | `true` | Whether clicking the status icon cycles through statuses |
| `toolsLabel` | `string` | `"MCP Servers:"` | Label shown before tool tags in expanded subtasks |
| `className` | `string` | — | Additional Tailwind classes for the container |

### AgentTask

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier |
| `title` | `string` | Task title shown in the row |
| `description` | `string` | Detailed description shown when expanded |
| `status` | `TaskStatus` | Current task status |
| `priority` | `"low" \| "medium" \| "high"` | Optional priority level |
| `level` | `number` | Nesting level (0 = root) |
| `dependencies` | `string[]` | IDs of tasks this depends on — shown as badges |
| `subtasks` | `AgentSubtask[]` | Child tasks (one level deep) |

### AgentSubtask

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier |
| `title` | `string` | Subtask title |
| `description` | `string` | Shown when subtask is clicked to expand |
| `status` | `TaskStatus` | Current subtask status |
| `priority` | `"low" \| "medium" \| "high"` | Optional priority level |
| `tools` | `string[]` | MCP server or tool names — shown as chip tags |

## Notes

- **Status cycling**: When `allowStatusToggle` is true, clicking the status icon cycles: pending → in-progress → completed → need-help → failed → pending.
- **Subtask completion**: When a subtask is toggled to completed, if all subtasks are complete, the parent task is automatically marked as completed.
- **Completing a task**: When a task is set to `completed`, all its subtasks are automatically marked completed.
- **Reduced motion**: The component reads `window.matchMedia("(prefers-reduced-motion: reduce)")` and adjusts animation behavior accordingly.
- **Dependencies**: The `dependencies` array contains **ID strings** that are rendered as small chip badges — no automatic dependency enforcement is built in.
