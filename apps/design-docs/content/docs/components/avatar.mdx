## Import

```tsx
import {
  Avatar,
  AvatarImage,
  AvatarFallback,
  AvatarGroup,
  GitHubAvatar,
  GitLabAvatar,
  BitbucketAvatar,
  AvatarWithIcon,
  DiceBearAvatar,
} from "@nebutra/custom-ui/primitives"
```

## Size Variants

Five sizes controlled by the `size` prop.

<ComponentPreview>
  <AvatarSizeDemo />
</ComponentPreview>

```tsx
<Avatar size="xs">
  <AvatarImage src="https://avatar.vercel.sh/user" alt="user" />
  <AvatarFallback size="xs">XS</AvatarFallback>
</Avatar>

<Avatar size="sm">...</Avatar>
<Avatar size="md">...</Avatar>
<Avatar size="lg">...</Avatar>
<Avatar size="xl">...</Avatar>
```

| `size` | px | Tailwind class |
|--------|-----|----------------|
| `xs` | 20 | `h-5 w-5` |
| `sm` | 32 | `h-8 w-8` |
| `md` | 40 | `h-10 w-10` (default) |
| `lg` | 56 | `h-14 w-14` |
| `xl` | 80 | `h-20 w-20` |

## Fallback

When the image URL fails or is omitted, `AvatarFallback` renders initials on a muted background.

<ComponentPreview>
  <AvatarFallbackDemo />
</ComponentPreview>

```tsx
<Avatar size="md">
  <AvatarFallback size="md">AC</AvatarFallback>
</Avatar>
```

<Tip>
  Always pass the same `size` to both `Avatar` and `AvatarFallback` — this ensures the font size matches the container.
</Tip>

## Group

`AvatarGroup` stacks avatars with an automatic `+N` overflow indicator. The second row below has 5 members with `max={4}`, showing `+2`.

<ComponentPreview>
  <AvatarGroupDemo />
</ComponentPreview>

```tsx
<AvatarGroup
  items={[
    { src: "https://avatar.vercel.sh/evilrabbit", alt: "evilrabbit", fallback: "ER" },
    { src: "https://avatar.vercel.sh/leerob",     alt: "leerob",     fallback: "LR" },
    { src: "https://avatar.vercel.sh/rauchg",     alt: "rauchg",     fallback: "RG" },
    { src: "https://avatar.vercel.sh/shuding",    alt: "shuding",    fallback: "SH" },
    { src: "https://avatar.vercel.sh/almonk",     alt: "almonk",     fallback: "AL" },
  ]}
  max={4}
  size="sm"
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `AvatarGroupItem[]` | — | Array of `{ src?, alt, fallback }` |
| `max` | `number` | `4` | Max avatars before collapsing to `+N` |
| `size` | `AvatarSize` | `"sm"` | Size applied to all avatars |

## Avatar with Status Icon

`AvatarWithIcon` adds a small badge anchored to the bottom-right corner. Useful for presence indicators, role badges, or verification marks.

<ComponentPreview>
  <AvatarWithIconDemo />
</ComponentPreview>

```tsx
<AvatarWithIcon
  src="https://avatar.vercel.sh/rauchg"
  alt="rauchg"
  icon={<span className="block h-2 w-2 rounded-full bg-green-500" />}
  iconBackground="bg-background"
  size="md"
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `ReactNode` | — | Icon element in the badge |
| `iconBackground` | `string` | `"bg-background"` | Tailwind bg class for the badge |
| `src` | `string` | — | Avatar image URL |
| `alt` | `string` | `"Avatar"` | Alt text |
| `fallback` | `string` | auto-initials | Initials when image fails |
| `size` | `AvatarSize` | `"md"` | Size preset |

## Generated Avatars (DiceBear)

Procedurally generated avatars via [DiceBear](https://www.dicebear.com/) — deterministic for the same `seed`. No npm package required.

<ComponentPreview>
  <DiceBearAvatarDemo />
</ComponentPreview>

```tsx
<DiceBearAvatar seed="rauchg" avatarStyle="bottts-neutral" size="md" />

<DiceBearAvatar
  seed="user@example.com"
  avatarStyle="pixel-art"
  options={{ radius: 50 }}
  size="md"
/>

<DiceBearAvatar seed="guest-session-123" avatarStyle="fun-emoji" size="md" />

<DiceBearAvatar seed="my-bot-agent" avatarStyle="bottts" size="md" />
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `seed` | `string` | — | Deterministic seed (username, email, UUID…) |
| `avatarStyle` | `DiceBearStyle` | `"bottts-neutral"` | Avatar collection |
| `options` | `Record<string, string \| number>` | `{}` | Extra DiceBear query params |
| `size` | `AvatarSize` | `"md"` | Size preset |

<Tip>
  Use `bottts-neutral` for system/bot entities, `avataaars` or `lorelei` for human placeholders, `fun-emoji` for anonymous/guest sessions.
</Tip>

## Props Reference

### Avatar

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"xs" \| "sm" \| "md" \| "lg" \| "xl"` | `"md"` | Size preset |
| `className` | `string` | — | Additional CSS classes |

### AvatarImage

| Prop | Type | Description |
|------|------|-------------|
| `src` | `string` | Image URL |
| `alt` | `string` | **Required.** Screen reader label |

### AvatarFallback

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `AvatarSize` | `"md"` | Match parent Avatar size for correct font |
| `delayMs` | `number` | `600` | Delay before showing — prevents flash on fast loads |

## Accessibility

- Always provide a meaningful `alt` on `AvatarImage`.
- When the avatar triggers an action (e.g., opens a user menu), wrap in `<button aria-label="...">`.
- `AvatarFallback` background (`bg-muted`) meets 4.5:1 contrast against `text-muted-foreground`.
- `DiceBearAvatar` SVGs are decorative — the `AvatarFallback` provides a text alternative.
