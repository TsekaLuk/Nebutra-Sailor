---
title: Heading
description: Semantic heading primitive rendering h1–h6 with typography tokens and optional display styles.
---

## Usage

```tsx
import { Heading } from "@nebutra/ui";

<Heading level={2}>Section Title</Heading>
```

## Examples

### Heading Levels

```tsx
<Heading level={1}>Heading 1</Heading>
<Heading level={2}>Heading 2</Heading>
<Heading level={3}>Heading 3</Heading>
<Heading level={4}>Heading 4</Heading>
<Heading level={5}>Heading 5</Heading>
<Heading level={6}>Heading 6</Heading>
```

### Display Style

Use `display` for hero-size headings (display-1 and display-2) on `level={1}` and `level={2}`.

```tsx
<Heading level={1} display>
  Hero Headline
</Heading>

<Heading level={2} display>
  Section Display
</Heading>
```

### SEO Override with `as`

Render the visual size of `h1` while outputting `h2` in the DOM — useful for SEO-correct page outlines.

```tsx
{/* Visually h1, semantically h2 */}
<Heading level={1} as="h2">
  Featured Article
</Heading>
```

### With Color Tokens

```tsx
<Heading level={2} color="muted">
  Secondary Section
</Heading>

<Heading level={1} color="default" display>
  Primary Hero
</Heading>
```

### Centered Alignment

```tsx
<Heading level={2} align="center" className="mb-4">
  Centered Section Heading
</Heading>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `2` | Heading level — controls both HTML element and visual size |
| `as` | `"h1" \| "h2" \| "h3" \| "h4" \| "h5" \| "h6"` | — | Override the rendered element (for SEO purposes) |
| `color` | `TextColor` | `"default"` | Text color from the typography token system |
| `display` | `boolean` | `false` | Use display style (`display-1` or `display-2`) for hero headings |
| `align` | `"left" \| "center" \| "right"` | — | Text alignment |
| `className` | `string` | — | Additional Tailwind classes |

## Notes

- Levels 5 and 6 use the same visual style as level 4 (`heading-4`) — differentiation is purely semantic.
- `display` mode only applies to levels 1 and 2; higher levels use standard heading styles regardless.
- When using `as` to override the element, ensure the resulting HTML heading hierarchy is still logical for screen readers.
