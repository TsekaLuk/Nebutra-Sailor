## Overview

`Menu` is a semantic dropdown component built on top of `DropdownMenu` (Radix). It exposes a high-level API with `prefix`, `suffix`, `locked`, `disabled`, and `href` slots on each item — covering all Geist Menu variants without exposing Radix internals at the usage site.

Use `DropdownMenu` directly when you need checkbox items, radio groups, or sub-menus. Use `Menu` for standard navigation and action lists.

## Installation

```tsx
import { Menu } from "@nebutra/custom-ui/primitives";
```

## Usage

### Default

```tsx
<Menu.Root>
  <Menu.Trigger>Menu</Menu.Trigger>
  <Menu.Content>
    <Menu.Item>Profile</Menu.Item>
    <Menu.Item>Settings</Menu.Item>
    <Menu.Separator />
    <Menu.Item>Sign out</Menu.Item>
  </Menu.Content>
</Menu.Root>
```

### With chevron

```tsx
<Menu.Trigger chevron>Menu</Menu.Trigger>
```

### Disabled items

```tsx
<Menu.Item disabled>Settings</Menu.Item>
```

### Locked items

Locked items are permission-gated — visible but non-interactive, with a lock icon. Semantically distinct from `disabled`.

```tsx
<Menu.Item locked>Analytics</Menu.Item>
```

### Link items

Renders the item as an `<a>` tag. Keyboard and mouse navigation is preserved via Radix `asChild`.

```tsx
<Menu.Item href="/docs">Documentation</Menu.Item>
```

### Prefix and suffix

```tsx
<Menu.Item prefix={<User className="h-4 w-4" />} suffix="Pro">
  Billing
</Menu.Item>
```

### Custom trigger

Replace `Menu.Trigger` content entirely — pass `className` to override default styles.

```tsx
<Menu.Trigger
  className="h-8 w-8 rounded-full bg-muted p-0 flex items-center justify-center"
  aria-label="Open menu"
>
  <UserCircle className="h-4 w-4" />
</Menu.Trigger>
```

### Menu position

```tsx
<Menu.Content align="end">...</Menu.Content>
```

## Props

### `Menu.Trigger`

Extends `<button>` HTML attributes.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `chevron` | `boolean` | `false` | Adds a rotating ChevronDown icon |
| `className` | `string` | — | Override trigger styles |

### `Menu.Content`

Extends Radix `DropdownMenuContent` props.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `align` | `"start" \| "center" \| "end"` | `"start"` | Horizontal alignment relative to trigger |
| `sideOffset` | `number` | `4` | Gap between trigger and content in px |

### `Menu.Item`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `prefix` | `ReactNode` | — | Icon/element on the left |
| `suffix` | `ReactNode` | — | Icon/element on the right (overridden by lock icon when `locked`) |
| `disabled` | `boolean` | `false` | Grays out item, removes pointer events |
| `locked` | `boolean` | `false` | Permission-gated state — shows lock icon, prevents interaction |
| `href` | `string` | — | Renders item as `<a>` navigating to this URL |
| `onSelect` | `(e: Event) => void` | — | Called on item selection |
| `className` | `string` | — | Additional CSS classes |

## Accessibility

- Built on Radix `DropdownMenu` — full keyboard navigation (`Arrow`, `Enter`, `Space`, `Escape`) is handled automatically.
- `locked` items set `aria-disabled` via Radix's `disabled` prop while remaining visible.
- Icon-only triggers must include `aria-label`.
- `prefix` and `suffix` icons are marked `aria-hidden="true"` — label text carries all meaning.
