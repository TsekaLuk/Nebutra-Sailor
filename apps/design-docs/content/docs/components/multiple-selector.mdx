## Overview

`MultipleSelector` is a feature-rich multi-select built on top of `cmdk`. Selected options appear as dismissable badge tags inside the input. Supports:

- Fixed (non-removable) options
- Grouped options with `groupBy`
- Async search via `onSearch`
- Sync search via `onSearchSync`
- Maximum selection limit
- Keyboard navigation (Tab, Enter, Backspace to remove)

## Usage

```tsx
import { MultipleSelector } from "@nebutra/custom-ui/primitives";

const options = [
  { value: "next", label: "Next.js" },
  { value: "react", label: "React" },
  { value: "vue", label: "Vue.js" },
];

// Basic
<MultipleSelector
  defaultOptions={options}
  placeholder="Select frameworks..."
/>

// Controlled
const [selected, setSelected] = useState([]);

<MultipleSelector
  defaultOptions={options}
  value={selected}
  onChange={setSelected}
/>

// Async search
<MultipleSelector
  onSearch={async (query) => {
    const results = await searchApi(query);
    return results.map((r) => ({ value: r.id, label: r.name }));
  }}
  placeholder="Search users..."
/>

// With max selection
<MultipleSelector
  defaultOptions={options}
  maxSelected={3}
  onMaxSelected={(max) => toast(`Maximum ${max} selections`)}
/>

// Grouped
const grouped = [
  { value: "react", label: "React", group: "Frontend" },
  { value: "node", label: "Node.js", group: "Backend" },
];

<MultipleSelector
  defaultOptions={grouped}
  groupBy="group"
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOptions` | `MultipleSelectorOption[]` | — | Static options list |
| `options` | `MultipleSelectorOption[]` | — | Manually controlled options |
| `value` | `MultipleSelectorOption[]` | — | Controlled selected values |
| `onChange` | `(options: MultipleSelectorOption[]) => void` | — | Change callback |
| `placeholder` | `string` | — | Input placeholder |
| `onSearch` | `(query: string) => Promise<MultipleSelectorOption[]>` | — | Async search function |
| `onSearchSync` | `(query: string) => MultipleSelectorOption[]` | — | Sync search function |
| `delay` | `number` | — | Debounce delay for `onSearch` in ms |
| `maxSelected` | `number` | — | Maximum number of selections |
| `onMaxSelected` | `(max: number) => void` | — | Called when max is reached |
| `groupBy` | `string` | — | Option key to group by |
| `disabled` | `boolean` | `false` | Disable the selector |
| `hidePlaceholderWhenSelected` | `boolean` | — | Hide placeholder when options selected |
| `emptyIndicator` | `ReactNode` | — | Content when no options found |
| `loadingIndicator` | `ReactNode` | — | Content during async search |
| `className` | `string` | — | Container CSS class |

### `MultipleSelectorOption`

```ts
interface MultipleSelectorOption {
  value: string;
  label: string;
  disable?: boolean;   // Non-selectable option
  fixed?: boolean;     // Cannot be removed once selected
  [key: string]: string | boolean | undefined; // For groupBy key
}
```

## Notes

- Selected options show as badge chips with an `X` button to remove
- Fixed options (with `fixed: true`) render without the remove button
- Backspace on empty input removes the last selected option
- `triggerSearchOnFocus` triggers the `onSearch` function when the input gains focus
