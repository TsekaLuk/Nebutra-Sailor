## Demo

```tsx
import { Separator } from "@nebutra/ui/primitives"

export function SeparatorDemo() {
  return (
    <div className="space-y-4 w-80">
      <div>
        <h4 className="text-sm font-medium">Nebutra Design System</h4>
        <p className="text-sm text-muted-foreground">Composable UI for modern apps.</p>
      </div>
      <Separator />
      <div className="flex gap-4 text-sm">
        <span>Blog</span>
        <Separator orientation="vertical" className="h-4" />
        <span>Docs</span>
        <Separator orientation="vertical" className="h-4" />
        <span>Source</span>
      </div>
    </div>
  )
}
```

## Usage

```tsx
import { Separator } from "@nebutra/ui/primitives"
```

## Props

Extends `@radix-ui/react-separator` `Root` props.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Direction of the separator line |
| `decorative` | `boolean` | `true` | When `true`, the separator is `aria-hidden` (purely visual) |
| `className` | `string` | — | Additional CSS classes |

## Examples

### Horizontal Separator

```tsx
<div>
  <p>Section one content</p>
</div>
<Separator />
<div>
  <p>Section two content</p>
</div>
```

### Vertical Separator in Flex Row

```tsx
<div className="flex items-center gap-3 h-5 text-sm">
  <span>Home</span>
  <Separator orientation="vertical" />
  <span>About</span>
  <Separator orientation="vertical" />
  <span>Contact</span>
</div>
```

### Semantic Separator (with role)

```tsx
{/* decorative={false} exposes role="separator" to screen readers */}
<Separator decorative={false} />
```

### With Label

```tsx
<div className="relative">
  <Separator />
  <span className="absolute left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2 bg-background px-2 text-xs text-muted-foreground">
    OR
  </span>
</div>
```

## Accessibility

- `decorative={true}` (default) sets `aria-hidden="true"` and `role="none"` — invisible to screen readers.
- `decorative={false}` sets `role="separator"` with `aria-orientation` for semantic content boundaries.
- Use `decorative={false}` only when the separator conveys meaningful structure.
