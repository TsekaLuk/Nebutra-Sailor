---

## Demo

```tsx
import { toast } from "sonner"
import { Button } from "@nebutra/ui/primitives"
import { Toaster } from "@nebutra/ui/primitives"

export function SonnerDemo() {
  return (
    <>
      <Toaster />
      <div className="flex flex-wrap gap-2">
        <Button onClick={() => toast("Event has been created")}>
          Default Toast
        </Button>
        <Button onClick={() => toast.success("Profile updated!")}>
          Success
        </Button>
        <Button onClick={() => toast.error("Something went wrong.")}>
          Error
        </Button>
        <Button onClick={() => toast.warning("Low disk space.")}>
          Warning
        </Button>
        <Button onClick={() => toast.info("New update available.")}>
          Info
        </Button>
      </div>
    </>
  )
}
```

## Usage

```tsx
// 1. Add Toaster to your root layout (once):
import { Toaster } from "@nebutra/ui/primitives"
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Toaster />
      </body>
    </html>
  )
}

// 2. Trigger toasts anywhere in the app:
import { toast } from "sonner"
toast("Hello world")
```

## Props

### Toaster

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `position` | `"top-left" \| "top-center" \| "top-right" \| "bottom-left" \| "bottom-center" \| "bottom-right"` | `"bottom-right"` | Position on screen |
| `expand` | `boolean` | `false` | Always show all toasts expanded |
| `richColors` | `boolean` | `false` | Use rich semantic colors |
| `duration` | `number` | `4000` | Default duration in ms |
| `closeButton` | `boolean` | `false` | Show close button on all toasts |
| `theme` | `"light" \| "dark" \| "system"` | `"system"` | Color theme |
| `toastOptions` | `ToastOptions` | — | Default options for all toasts |

## Examples

### Promise Toast

```tsx
toast.promise(saveUser(userData), {
  loading: "Saving...",
  success: "Profile updated successfully!",
  error: "Failed to save profile.",
})
```

### Toast with Action

```tsx
toast("File deleted", {
  action: {
    label: "Undo",
    onClick: () => restoreFile(),
  },
  duration: 5000,
})
```

### Custom Description Toast

```tsx
toast("Scheduled maintenance", {
  description: "The system will be down from 2–4 AM UTC.",
})
```

### Dismiss Programmatically

```tsx
const toastId = toast.loading("Processing...")
await doSomething()
toast.dismiss(toastId)
toast.success("Done!")
```

## Accessibility

- Toasts are rendered in a `<section>` with `aria-label="Notifications"`.
- Each toast has `role="status"` (or `role="alert"` for errors) with `aria-live`.
- Toasts are announced to screen readers on appearance.
- The Toaster is appended at the bottom of the document — outside modal focus traps.
