# Nebutra API Reference

Welcome to the Nebutra API documentation. The API Gateway provides a comprehensive set of endpoints for system monitoring, legal compliance, and event processing.

## Base URL

```
Production: https://api.nebutra.com
Development: http://localhost:3002
```

## Authentication

Most API endpoints require a Clerk-issued JWT token passed as a Bearer token in the `Authorization` header.

```bash
curl -H "Authorization: Bearer <your-jwt-token>" \
  https://api.nebutra.com/api/v1/events/ingest
```

## Rate Limiting

API endpoints under `/api/*` are rate-limited (excluding health, status, and webhook endpoints). Rate limit headers are included in responses:

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests per window |
| `X-RateLimit-Remaining` | Remaining requests in current window |
| `X-RateLimit-Reset` | Time when the rate limit resets |

## Response Headers

All responses include:

| Header | Description |
|--------|-------------|
| `X-Request-ID` | Unique request identifier for debugging |
| `X-Trace-ID` | OpenTelemetry trace ID (when available) |
| `API-Version` | Current API version |
