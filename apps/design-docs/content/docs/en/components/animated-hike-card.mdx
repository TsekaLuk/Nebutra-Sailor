## Usage

```tsx
import { AnimatedHikeCard } from "@nebutra/ui";
import { Clock, Mountain } from "lucide-react";

<AnimatedHikeCard
  title="Mountain Trek"
  images={["/img1.jpg", "/img2.jpg", "/img3.jpg"]}
  stats={[
    { icon: <Clock className="h-4 w-4" />, label: "~6 Hours" },
    { icon: <Mountain className="h-4 w-4" />, label: "8 km" },
  ]}
  description="A beautiful trail through alpine meadows and rocky peaks."
  href="/hikes/mountain-trek"
/>;
```

## Props

| Prop          | Type       | Default | Description                                            |
| ------------- | ---------- | ------- | ------------------------------------------------------ |
| `title`       | `string`   | —       | Card heading (large, bold)                             |
| `images`      | `string[]` | —       | Array of image URLs — 3 recommended for the fan effect |
| `stats`       | `Stat[]`   | —       | Array of `{ icon: ReactNode, label: string }` chips    |
| `description` | `string`   | —       | Description text below the stats                       |
| `href`        | `string`   | —       | Link destination (renders as `<a>`)                    |
| `className`   | `string`   | —       | Additional Tailwind classes                            |

## Notes

- The card renders as an `<a>` element — `href` is required for proper link semantics.
- **Image fan effect**: 3 images stack with `absolute` positioning. On hover, images fan out to the left and right using CSS group-hover transforms.
- **Accessibility**: The card has `aria-label="Learn more about {title}"` on the outer anchor.
- `max-w-sm` (mobile) and `max-w-md` (desktop) constrain card width — override with `className` for different sizes.
