title: Announcement

## Usage

```tsx
import { Announcement, AnnouncementTag, AnnouncementTitle } from "@nebutra/ui";

<Announcement>
  <AnnouncementTag>New</AnnouncementTag>
  <AnnouncementTitle>
    Introducing our latest feature
    <ArrowUpRightIcon size={14} />
  </AnnouncementTitle>
</Announcement>;
```

## Examples

### Basic Announcement

The standard pattern with a category tag and a title:

```tsx
<Announcement>
  <AnnouncementTag>Beta</AnnouncementTag>
  <AnnouncementTitle>Multi-agent workflows are now available</AnnouncementTitle>
</Announcement>
```

### With Arrow Icon

```tsx
import { ArrowUpRight } from "lucide-react";

<Announcement>
  <AnnouncementTag>New</AnnouncementTag>
  <AnnouncementTitle>
    Read the release notes
    <ArrowUpRight size={14} />
  </AnnouncementTitle>
</Announcement>;
```

### As a Link

Wrap in an anchor tag to make the entire badge clickable:

```tsx
<a href="/blog/release-v2">
  <Announcement className="cursor-pointer">
    <AnnouncementTag>v2.0</AnnouncementTag>
    <AnnouncementTitle>
      See what's new
      <ArrowUpRight size={14} />
    </AnnouncementTitle>
  </Announcement>
</a>
```

### Themed Variants

Use `themed` with a color `className` for status-style announcements:

```tsx
{
  /* Success */
}
<Announcement themed className="bg-green-100 text-green-700">
  <AnnouncementTag>Success</AnnouncementTag>
  <AnnouncementTitle>Deployment completed</AnnouncementTitle>
</Announcement>;

{
  /* Error */
}
<Announcement themed className="bg-rose-100 text-rose-700">
  <AnnouncementTag>Error</AnnouncementTag>
  <AnnouncementTitle>Build failed — check logs</AnnouncementTitle>
</Announcement>;

{
  /* Warning */
}
<Announcement themed className="bg-amber-100 text-amber-700">
  <AnnouncementTag>Warning</AnnouncementTag>
  <AnnouncementTitle>Deprecated API in use</AnnouncementTitle>
</Announcement>;
```

### Title Only (No Tag)

```tsx
<Announcement>
  <AnnouncementTitle>
    Free for open-source projects — get started today
  </AnnouncementTitle>
</Announcement>
```

## Props

### Announcement

| Prop        | Type           | Default     | Description                                                             |
| ----------- | -------------- | ----------- | ----------------------------------------------------------------------- |
| `themed`    | `boolean`      | `false`     | Applies themed styling — use with colored `className` for status badges |
| `variant`   | `BadgeVariant` | `"outline"` | Badge variant inherited from Badge                                      |
| `className` | `string`       | —           | Additional Tailwind classes                                             |

### AnnouncementTag

| Prop        | Type              | Default | Description                             |
| ----------- | ----------------- | ------- | --------------------------------------- |
| `className` | `string`          | —       | Additional Tailwind classes             |
| `children`  | `React.ReactNode` | —       | Tag label (e.g., "New", "Beta", "v2.0") |

### AnnouncementTitle

| Prop        | Type              | Default | Description                  |
| ----------- | ----------------- | ------- | ---------------------------- |
| `className` | `string`          | —       | Additional Tailwind classes  |
| `children`  | `React.ReactNode` | —       | Title text and optional icon |

## Notes

- `AnnouncementTag` and `AnnouncementTitle` **must** be used inside `Announcement` — they use React Context to read the `themed` flag. Using them outside will throw an error.
- The `themed` prop changes `AnnouncementTag`'s background to `bg-background/60` instead of `bg-foreground/5`, which works better on colored backgrounds.
- Hover state (`hover:shadow-md`) is built in — ideal for clickable announcement banners.
