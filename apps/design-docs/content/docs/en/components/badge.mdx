## Sizes

Three sizes controlled by the `size` prop:

| `size` | Font | Padding | Use case |
|--------|------|---------|----------|
| `sm` | 10px | `px-1.5 py-px` | Tight UI, table cells |
| `md` | 12px | `px-2.5 py-0.5` | Default — most contexts |
| `lg` | 14px | `px-3 py-1` | Prominent labels, hero sections |

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | see below | `"default"` | Color / semantic variant |
| `size` | `"sm" \| "md" \| "lg"` | `"md"` | Size preset |
| `icon` | `ReactNode` | — | Icon rendered before label, auto-sized to 12×12px |
| `dot` | `boolean` | `false` | Renders a 6px status dot before label |
| `asChild` | `boolean` | `false` | Renders as child element via Radix Slot |
| `className` | `string` | — | Additional CSS classes |
| `children` | `ReactNode` | — | Badge label |

### Variant values

**Semantic:** `default` `secondary` `destructive` `outline` `success` `warning` `info` `error`

**Color (solid):** `gray` `blue` `purple` `amber` `red` `pink` `green` `teal` `inverted`

**Color (subtle):** `gray-subtle` `blue-subtle` `purple-subtle` `amber-subtle` `red-subtle` `pink-subtle` `green-subtle` `teal-subtle`

## Accessibility

- Renders as `<div>` by default — not interactive.
- For clickable badges, use `asChild` with a native `<a>` or Next.js `<Link>`.
- For dynamic status updates, add `aria-live="polite"` to the parent container.
- When used as a notification count, pair with `aria-label` on the triggering element.
- The `dot` and `icon` slots are `aria-hidden="true"` — label text provides the accessible name.
