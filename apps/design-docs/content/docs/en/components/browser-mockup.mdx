## Overview

`Browser` renders a decorative browser chrome inspired by Vercel's Geist Design System. It includes:

- Traffic light buttons (close, minimize, maximize)
- Navigation buttons (back, forward, reload) — decorative
- Address bar with copy-to-clipboard button
- Flexible content area that accepts arbitrary `children`

## Usage

```tsx
import { Browser } from "@nebutra/ui/primitives";

// Children mode — arbitrary JSX
<Browser address="vercel.com">
  <div className="p-6">
    <h1>Hello World</h1>
  </div>
</Browser>

// Image convenience mode
<Browser address="example.com" imageSrc="/screenshot.png" />

// Empty frame
<Browser address="app.example.com" />
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `address` | `string` | — | URL text shown in the address bar |
| `imageSrc` | `string` | — | Convenience prop: renders an `<img>` when no children are provided |
| `imageAlt` | `string` | `"Preview"` | Alt text for the image (only used with `imageSrc`) |
| `children` | `ReactNode` | — | Arbitrary content rendered in the browser viewport (takes priority over `imageSrc`) |
| `className` | `string` | — | Additional CSS class for the outer container |

All other `<div>` props are forwarded to the container element.

## Migration from BrowserMockup

`BrowserMockup` is still exported as a deprecated alias. To migrate:

```diff
- import { BrowserMockup } from "@nebutra/ui/primitives";
+ import { Browser } from "@nebutra/ui/primitives";

- <BrowserMockup url="example.com" image="/screenshot.png" />
+ <Browser address="example.com" imageSrc="/screenshot.png" />

- <BrowserMockup url="example.com" placeholder={<MyContent />} />
+ <Browser address="example.com"><MyContent /></Browser>
```

## Notes

- The address bar copy button uses `navigator.clipboard.writeText()` — it silently fails in contexts where the Clipboard API is unavailable
- Navigation buttons (back, forward, reload) are purely decorative — they do not trigger any action
- The content area height is determined by `children` content; when no content is provided, it falls back to a `aspect-video` empty state
- No `next/image` dependency — uses a plain `<img>` tag for the `imageSrc` convenience prop
- Dark mode is fully supported via Tailwind `dark:` classes
