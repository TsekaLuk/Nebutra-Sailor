## Overview

`Combobox` is a self-contained searchable select built on `Popover` + `cmdk`. It supports two usage modes:

1. **Simple mode** — pass an `options` array and the component handles everything (search, filtering, rendering, selection).
2. **Composition mode** — use `Combobox.Input`, `Combobox.Option`, `Combobox.Group` etc. as children for custom rendering.

## Props

### Combobox

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `ComboboxOption[]` | — | Simple mode: array of options to render |
| `value` | `string` | — | Controlled selected value |
| `onChange` | `(value: string) => void` | — | Callback when selection changes |
| `defaultValue` | `string` | `""` | Initial value for uncontrolled usage |
| `disabled` | `boolean` | `false` | Disable the entire combobox |
| `error` | `boolean` | `false` | Show error styling (red border) |
| `size` | `"sm" \| "default" \| "lg"` | `"default"` | Size variant — `sm` (32px), `default` (40px), `lg` (48px) |
| `width` | `string` | — | Tailwind width class applied to both trigger and dropdown |
| `label` | `string` | — | Accessible label shown above the trigger |
| `hideLabel` | `boolean` | `false` | Visually hide the label (still accessible via aria) |
| `placeholder` | `string` | `"Select..."` | Text when nothing is selected |
| `emptyMessage` | `string` | `"No results found."` | Text when search has no matches |
| `searchPlaceholder` | `string` | `"Search..."` | Placeholder in the search input |
| `children` | `ReactNode` | — | Composition mode: provide sub-components manually |
| `className` | `string` | — | Additional CSS classes |

### ComboboxOption

| Prop | Type | Description |
|------|------|-------------|
| `value` | `string` | Machine value used for selection matching |
| `label` | `string` | Human-readable label displayed in trigger and list |
| `disabled` | `boolean` | Disable this specific option |
| `group` | `string` | Assign to a group heading |

### Sub-Components

| Component | Based On | Description |
|-----------|----------|-------------|
| `Combobox.Input` | `CommandInput` | Search input for filtering options |
| `Combobox.Option` | `CommandItem` | Selectable option with auto check icon |
| `Combobox.Empty` | `CommandEmpty` | Message when no options match search |
| `Combobox.Group` | `CommandGroup` | Group heading for related options |
| `Combobox.Separator` | `CommandSeparator` | Visual divider between groups |

## Accessibility

- Trigger button has `role="combobox"` with `aria-expanded`.
- Label uses `aria-labelledby` to associate with the trigger.
- `CommandInput` provides a live search accessible to screen readers.
- `CommandItem` elements have `role="option"` with `aria-selected`.
- Keyboard navigation: type to filter, `ArrowUp`/`ArrowDown` to navigate, `Enter` to select, `Escape` to close.

## Notes

- The dropdown width automatically matches the trigger width via `--radix-popover-trigger-width`.
- When using `options` with grouped items, options are automatically sorted into `CommandGroup` headings.
- `Combobox.Option` accepts `keywords` passthrough to `CommandItem` for enhanced search matching.
- Toggle-deselect: clicking an already-selected option clears the selection.
