## Overview

`Confetti` wraps the `canvas-confetti` library in a React component that supports:

- **Auto-fire on mount** (default) — fire immediately when the component renders
- **Manual control via ref** — call `ref.current.fire(options)` imperatively
- **ConfettiButton** — convenience `<Button>` that fires confetti from its own position on click
- Web Worker support for off-main-thread rendering
- Auto-resize canvas

Typical use cases: purchase success, achievement unlock, form submission, milestone celebrations.

## Usage

```tsx
import { Confetti, ConfettiButton, type ConfettiRef } from "@nebutra/ui/primitives";
import { useRef } from "react";

// Auto-fire on mount (overlay entire viewport)
<Confetti className="fixed inset-0 pointer-events-none" />

// Manual control via ref
const confettiRef = useRef<ConfettiRef>(null);

<Confetti ref={confettiRef} manualstart className="fixed inset-0 pointer-events-none" />
<button onClick={() => confettiRef.current?.fire({ particleCount: 100, spread: 80 })}>
  Celebrate!
</button>

// Convenience button (fires from button position)
<ConfettiButton variant="default">
  🎉 Celebrate
</ConfettiButton>

// Custom colors
confettiRef.current?.fire({
  particleCount: 150,
  spread: 100,
  colors: ["#0033FE", "#00e5ff", "#ffffff"],
  origin: { y: 0.8 },
});
```

## Confetti Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `ConfettiOptions` | — | Default confetti animation options (overridden per `fire()` call) |
| `globalOptions` | `ConfettiGlobalOptions` | `{ resize: true, useWorker: true }` | Canvas-confetti global options |
| `manualstart` | `boolean` | `false` | When `true`, suppresses auto-fire on mount |
| `children` | `ReactNode` | — | Children can access `fire` via `ConfettiContext` |
| `className` | `string` | — | CSS class for the `<canvas>` element |

All other `<canvas>` props are forwarded.

## ConfettiButton Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `ConfettiOptions & ConfettiGlobalOptions` | — | Confetti options. `origin` is automatically set to the button's center position |

All `Button` props (`variant`, `size`, `disabled`, `onClick`, etc.) are forwarded.

## ConfettiRef

```ts
type ConfettiRef = {
  fire: (options?: ConfettiOptions) => Promise<void>;
} | null;
```

Use `useRef<ConfettiRef>(null)` and pass it to `<Confetti ref={confettiRef} manualstart />`.

## Notes

- The canvas has `pointer-events-none` applied by default to avoid blocking clicks
- `ConfettiButton` computes the launch `origin` from the button's `getBoundingClientRect()` — no configuration needed
- When `manualstart` is omitted (or `false`), confetti fires once automatically on mount
- `options` passed to `fire()` are merged with (and override) the component-level `options` prop
- Requires `canvas-confetti`: `pnpm add canvas-confetti`
