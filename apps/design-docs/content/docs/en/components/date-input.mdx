## Overview

`DateInput` re-exports `@heroui/date-input` as a drop-in. Each date part (year, month, day) is an individually editable segment. Powered by `@internationalized/date`.

Features:
- Keyboard-editable segments (Tab between year/month/day/hour/minute/second)
- Multiple variants: `flat`, `bordered`, `faded`, `underlined`
- Label placements: `inside`, `outside`, `outside-left`
- Time zone support with `ZonedDateTime`
- International calendar support (Hebrew, Indian, Islamic, Buddhist, etc.)
- Min/max date validation
- Granularity control (`day`, `hour`, `minute`, `second`)

## Usage

```tsx
import { DateInput } from "@nebutra/ui/primitives";
import { CalendarDate } from "@internationalized/date";

// Basic
<DateInput label="Birth date" />

// Controlled
const [date, setDate] = useState(new CalendarDate(2024, 4, 4));

<DateInput
  label="Event date"
  value={date}
  onChange={setDate}
/>

// With time zone
import { parseZonedDateTime } from "@internationalized/date";

<DateInput
  label="Meeting time"
  defaultValue={parseZonedDateTime("2025-03-01T09:00[America/New_York]")}
/>

// Validation
<DateInput
  label="End date"
  isRequired
  isInvalid={!date}
  errorMessage="Date is required"
/>
```

## Key Props

| Prop | Type | Description |
|------|------|-------------|
| `label` | `string` | Input label |
| `value` | `CalendarDate \| ZonedDateTime` | Controlled date value |
| `defaultValue` | `CalendarDate \| ZonedDateTime` | Uncontrolled initial value |
| `onChange` | `(date) => void` | Change callback |
| `variant` | `"flat" \| "bordered" \| "faded" \| "underlined"` | Visual variant |
| `granularity` | `"day" \| "hour" \| "minute" \| "second"` | Segment detail level |
| `minValue` | `CalendarDate` | Minimum selectable date |
| `maxValue` | `CalendarDate` | Maximum selectable date |
| `isRequired` | `boolean` | Mark as required |
| `isInvalid` | `boolean` | Show error state |
| `isDisabled` | `boolean` | Disable the input |
| `errorMessage` | `string` | Validation error text |

## Notes

- Requires `@internationalized/date` for `CalendarDate`/`ZonedDateTime` types
- All HeroUI `DateInput` props are supported — this is a direct re-export
- See [HeroUI DateInput docs](https://heroui.com/docs/components/date-input) for the full prop reference
