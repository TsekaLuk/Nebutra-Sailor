## Usage

```tsx
import { FeatureArrowCard } from "@nebutra/ui";
import { Zap } from "lucide-react";

<FeatureArrowCard
  icon={Zap}
  title="Performance"
  subtitle="Sub-second response times"
  description="across 35+ global regions"
  onArrowClick={() => router.push("/features/performance")}
/>
```

## Examples

### Basic Card

```tsx
<FeatureArrowCard
  icon={Shield}
  title="Security"
  subtitle="Enterprise-grade protection"
  description="with zero-trust architecture and SOC 2 certification"
  onArrowClick={() => console.log("Navigate to security")}
/>
```

### With Card Content

Use `cardContent` for a thumbnail pinned to the bottom-right corner:

```tsx
<FeatureArrowCard
  icon={BarChart}
  title="Analytics"
  subtitle="Real-time insights"
  description="into your product usage and growth metrics"
  cardContent={
    <img
      src="/analytics-preview.png"
      alt="Analytics preview"
      className="object-cover w-full h-full"
    />
  }
  onArrowClick={() => router.push("/analytics")}
/>
```

### Grid Layout

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 gap-4">
  {features.map((feature) => (
    <FeatureArrowCard
      key={feature.id}
      icon={feature.icon}
      title={feature.title}
      subtitle={feature.subtitle}
      description={feature.description}
      onArrowClick={() => router.push(feature.href)}
    />
  ))}
</div>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `LucideIcon` | — | Feature icon (rendered at 16×16) |
| `title` | `string` | — | Small muted label shown above the subtitle |
| `subtitle` | `string` | — | Main heading in normal weight |
| `description` | `string` | — | Secondary text shown inline after subtitle (muted) |
| `cardContent` | `ReactNode` | — | Content slot pinned to bottom-right corner |
| `onArrowClick` | `() => void` | — | Handler for arrow button click |
| `className` | `string` | — | Additional Tailwind classes |

## Notes

- The arrow button rotates `-45deg` on hover via `hover:-rotate-45` — this creates a diagonal "navigate" affordance.
- `cardContent` is placed in an absolute-positioned card (`h-20 w-24` on mobile, `h-32 w-40` on desktop) in the bottom-right corner with rounded top-left border.
- The outer container has `relative` positioning to anchor both the card content and the arrow button.
