## Usage

```tsx
import {
  FeatureCard,
  FeatureCardHeader,
  FeatureCardContent,
} from "@nebutra/ui";
import { MapPin } from "lucide-react";

<FeatureCard>
  <FeatureCardHeader
    icon={MapPin}
    title="Real-time tracking"
    description="Know where your team is at all times."
  />
  <FeatureCardContent>
    <img src="/map-screenshot.png" alt="Map view" />
  </FeatureCardContent>
</FeatureCard>
```

### With DualModeImage

`DualModeImage` renders different images per theme:

```tsx
<FeatureCard>
  <FeatureCardHeader
    icon={Monitor}
    title="Dashboard"
    description="Full visibility into your operations."
  />
  <FeatureCardContent>
    <DualModeImage
      lightSrc="/dashboard-light.png"
      darkSrc="/dashboard-dark.png"
      alt="Dashboard screenshot"
      width={800}
      height={500}
    />
  </FeatureCardContent>
</FeatureCard>
```

### CircularUI Decoration

```tsx
<FeatureCard>
  <FeatureCardHeader
    icon={Layers}
    title="Flexible permissions"
    description="Fine-grained role-based access control."
  />
  <FeatureCardContent className="flex gap-4 p-6">
    <CircularUI
      label="Admins"
      circles={[{ pattern: "primary" }, { pattern: "border" }]}
    />
    <CircularUI
      label="Members"
      circles={[{ pattern: "border" }, { pattern: "none" }]}
    />
  </FeatureCardContent>
</FeatureCard>
```

## Props

### FeatureCard

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Card content |
| `className` | `string` | — | Additional Tailwind classes |

### FeatureCardHeader

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `LucideIcon` | — | Feature icon |
| `title` | `string` | — | Feature label (small, muted) |
| `description` | `string` | — | Main heading (large, semibold) |
| `className` | `string` | — | Additional Tailwind classes |

### FeatureCardContent

Re-export of `CardContent` — accepts standard `div` props and `className`.

### DualModeImage

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `lightSrc` | `string` | — | Image URL for light mode |
| `darkSrc` | `string` | — | Image URL for dark mode |
| `alt` | `string` | — | Accessibility alt text |
| `width` | `number` | — | Image width |
| `height` | `number` | — | Image height |
| `className` | `string` | — | Additional classes |

### CircularUI

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Text below the circles |
| `circles` | `CircleConfig[]` | — | Array of circle configs with `pattern: "none" \| "border" \| "primary" \| "blue"` |
| `className` | `string` | — | Additional classes |

## Notes

- `FeatureCard` renders with `rounded-none` by default — butts up to adjacent cards cleanly. Add a border to the parent grid, not individual cards.
- Corner bracket decorators (`FeatureCardDecorator`) are automatically included and use `text-primary` color.
- `DualModeImage` uses CSS `dark:` variants to switch images — requires Tailwind dark mode configured in your app.
