## Usage

```tsx
import { FeatureIconItem } from "@nebutra/ui";
import { Zap } from "lucide-react";

<FeatureIconItem
  icon={Zap}
  title="Lightning fast"
  description="Sub-second response times globally."
/>
```

```tsx
  <FeatureIconItem icon={Shield} title="Secure" description="Zero-trust architecture." />
  <FeatureIconItem icon={Globe} title="Global" description="35+ edge locations." />
  <FeatureIconItem icon={Sparkles} title="AI-ready" description="Built-in intelligence." />

```

### Compact List (No Description)

```tsx
<div className="space-y-3">
  <FeatureIconItem icon={Check} title="Custom domains" />
  <FeatureIconItem icon={Check} title="Unlimited requests" />
  <FeatureIconItem icon={Check} title="Priority support" />
```

```tsx

  icon={Star}

  title="Featured"
  description="Top-rated by enterprise customers."
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `LucideIcon` | — | Icon rendered inline with the title |
| `title` | `string` | — | Feature name |
| `description` | `string` | — | Optional description below the title |
| `iconClassName` | `string` | — | Override icon color/size |
| `className` | `string` | — | Additional container classes |

## Notes

- **FeatureIconItem vs FeatureCheckItem**: `FeatureIconItem` uses a required `icon` prop with no default — ideal for feature grids with distinct icons. `FeatureCheckItem` defaults to a `Check` icon — ideal for lists where all items share the same confirmation visual.
