## Usage

```tsx
import { Field } from "@nebutra/ui/primitives"
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string \| ReactNode` | — | Field label text |
| `description` | `string \| ReactNode` | — | Helper text shown below the input |
| `error` | `string \| ReactNode` | — | Error message; sets error state when provided |
| `htmlFor` | `string` | — | Associates label with input via `id` |
| `required` | `boolean` | `false` | Adds required indicator to label |
| `className` | `string` | — | Additional CSS classes |
| `children` | `ReactNode` | — | The input control (Input, Select, etc.) |

## Examples

### Required Field

```tsx
<Field label="Full name" htmlFor="name" required>
  <Input id="name" placeholder="Jane Doe" />
</Field>
```

### With Error

```tsx
<Field
  label="Password"
  htmlFor="password"
  error="Password must be at least 8 characters."
>
  <Input id="password" type="password" />
</Field>
```

### With Select

```tsx
<Field label="Country" htmlFor="country" description="Select your billing country.">
  <Select>
    <SelectTrigger id="country">
      <SelectValue placeholder="Select country" />
    </SelectTrigger>
    <SelectContent>
      <SelectItem value="us">United States</SelectItem>
      <SelectItem value="uk">United Kingdom</SelectItem>
    </SelectContent>
  </Select>
</Field>
```

## Accessibility

- `label` renders as a `<label>` element linked to the input via `htmlFor`.
- `description` is associated with the input via `aria-describedby`.
- `error` triggers `aria-invalid="true"` on the child input and is linked via `aria-errormessage`.
- Required fields have a visual indicator and `aria-required="true"` on the input.
