## Overview

`InfiniteSlider` creates a seamless loop by duplicating children and continuously translating them. Supports horizontal and vertical directions, configurable speed, and optional hover slow-down.

## Installation

```tsx
import { InfiniteSlider } from "@nebutra/ui/primitives";
```

## Usage

### Logo strip

```tsx
<InfiniteSlider gap={24} speed={50}>
  <img src="/logos/vercel.svg" alt="Vercel" className="h-8" />
  <img src="/logos/next.svg" alt="Next.js" className="h-8" />
  <img src="/logos/react.svg" alt="React" className="h-8" />
  <img src="/logos/typescript.svg" alt="TypeScript" className="h-8" />
</InfiniteSlider>
```

### Slow on hover

```tsx
<InfiniteSlider gap={32} duration={20} durationOnHover={60}>
  {logos.map((logo) => <Logo key={logo.name} {...logo} />)}
</InfiniteSlider>
```

### Reverse direction

```tsx
<InfiniteSlider gap={16} reverse>
  {items}
</InfiniteSlider>
```

### Vertical scroll

```tsx
<InfiniteSlider direction="vertical" gap={16} className="h-64">
  {testimonials.map((t) => <TestimonialCard key={t.id} {...t} />)}
</InfiniteSlider>
```

### Speed in px/s

```tsx
<InfiniteSlider speed={80} speedOnHover={20} gap={24}>
  {items}
</InfiniteSlider>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | **Required.** Content to scroll (duplicated internally) |
| `gap` | `number` | `16` | Gap between items in pixels |
| `duration` | `number` | `25` | Animation loop duration in seconds |
| `durationOnHover` | `number` | — | Slow-down duration when hovered (seconds) |
| `speed` | `number` | — | Speed in px/s (overrides `duration`) |
| `speedOnHover` | `number` | — | Speed on hover in px/s (overrides `durationOnHover`) |
| `direction` | `"horizontal" \| "vertical"` | `"horizontal"` | Scroll direction |
| `reverse` | `boolean` | `false` | Reverse scroll direction |
| `className` | `string` | — | Additional CSS classes for the overflow wrapper |

## Notes

- Children are duplicated once internally to create the seamless loop — avoid relying on a fixed number of children.
- When `durationOnHover` or `speedOnHover` is set, the animation smoothly transitions to the new speed on mouse enter/leave.
- Dimensions are measured with `react-use-measure` — ensure the container has a defined size before first render.
- Requires `framer-motion` and `react-use-measure` as peer dependencies.
