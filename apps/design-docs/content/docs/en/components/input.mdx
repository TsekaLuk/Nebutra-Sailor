## Prefix

Use the `prefix` prop to render a non-interactive icon or node inside the left edge. Clicks on the prefix area pass through to the input (it gets focused).

<ComponentPreview name="input-with-icon-demo" className="max-w-[350px] flex-col gap-4" />

---

## Suffix

Use the `suffix` prop to render a non-interactive label or icon inside the right edge.

<Warning>
  `suffix` has `pointer-events: none` — it is purely decorative. For interactive right-side elements
  (e.g. a password reveal button) build them manually with `className="pr-9"` and an `absolute` button.
  See the [Password Reveal](#password-reveal) example below.
</Warning>

## Clearable

Set `clearable` to render a ✕ button whenever the field has a value. The button disappears when the field is empty.

### Uncontrolled

<ComponentPreview name="input-clearable-demo" className="max-w-[350px]" />

### Controlled

```tsx
const [value, setValue] = useState("")

<Input
  value={value}
  onChange={(e) => setValue(e.target.value)}
  clearable
  onClear={() => setValue("")}
  placeholder="Controlled clearable…"
/>
```

### With Prefix

<ComponentPreview name="input-2-demo" className="max-w-[350px]">
  <Input
    prefix={<Search className="h-4 w-4" />}
    clearable
    placeholder="Search…"
  />
</ComponentPreview>

<Callout>
  When `clearable` and `suffix` are both set, the ✕ button replaces the suffix while the field has a value.
  The suffix reappears when the field is cleared — useful for keyboard shortcut hints or unit labels.
</Callout>

## Props

`Input` extends all native `React.InputHTMLAttributes<HTMLInputElement>`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `string` | `"text"` | HTML input type |
| `placeholder` | `string` | — | Placeholder text |
| `disabled` | `boolean` | `false` | Disables the input and hides the clear button |
| `className` | `string` | — | Applied to the `<input>` element |
| `prefix` | `ReactNode` | — | Non-interactive adornment inside the left edge |
| `suffix` | `ReactNode` | — | Non-interactive adornment inside the right edge |
| `clearable` | `boolean` | `false` | Show ✕ button when the field has a value |
| `onClear` | `() => void` | — | Called when the ✕ button is clicked |
| `wrapperClassName` | `string` | — | Applied to the outer `<div>` wrapper |
| `value` | `string \| number` | — | Controlled value |
| `defaultValue` | `string \| number` | — | Uncontrolled default value |
| `onChange` | `ChangeEventHandler` | — | Change handler |
| `ref` | `Ref<HTMLInputElement>` | — | Forwarded ref — always points to `<input>` |

## Design Tokens

Height locked to `h-10` (40px) from `inputTokens.height`.
Focus ring: `--focus-ring` — brand-blue 2px ring, 2px offset.
Prefix/suffix positioned at `left-3` / `right-3` (12px inset).

---

## Accessibility

- Renders native `<input>` — always pair with a visible `<label>` or `aria-label`.
- Use `aria-invalid="true"` + `aria-describedby` to associate error messages.
- The ✕ clear button has `aria-label="Clear"` — screen readers announce it correctly.
- Prefix and suffix spans carry `aria-hidden` — they are decorative and ignored by assistive tech.
- Focus ring is always visible for keyboard navigation.
