## Prefix

Use the `prefix` prop to render a non-interactive icon or node inside the left edge. Clicks on the prefix area pass through to the input (it gets focused).

<ComponentPreview name="input-with-icon-demo" className="max-w-[350px] flex-col gap-4"  />


## Suffix

Use the `suffix` prop to render a non-interactive label or icon inside the right edge.

<Warning>
  `suffix` has `pointer-events: none` вАФ it is purely decorative. For interactive right-side elements
  (e.g. a password reveal button) build them manually with `className="pr-9"` and an `absolute` button.
  See the [Password Reveal](#password-reveal) example below.
</Warning>

## Clearable

Set `clearable` to render a вЬХ button whenever the field has a value. The button disappears when the field is empty.

### Uncontrolled

<ComponentPreview name="input-clearable-demo" className="max-w-[350px]"  />


### Controlled

```tsx
const [value, setValue] = useState("")

<Input
  value={value}
  onChange={(e) => setValue(e.target.value)}
  clearable
  onClear={() => setValue("")}
  placeholder="Controlled clearableвА¶"
/>
```

  The suffix reappears when the field is cleared вАФ useful for keyboard shortcut hints or unit labels.
</Callout>

## Props

`Input` extends all native `React.InputHTMLAttributes<HTMLInputElement>`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `string` | `"text"` | HTML input type |
| `placeholder` | `string` | вАФ | Placeholder text |
| `disabled` | `boolean` | `false` | Disables the input and hides the clear button |
| `className` | `string` | вАФ | Applied to the `<input>` element |
| `prefix` | `ReactNode` | вАФ | Non-interactive adornment inside the left edge |
| `suffix` | `ReactNode` | вАФ | Non-interactive adornment inside the right edge |
| `clearable` | `boolean` | `false` | Show вЬХ button when the field has a value |
| `onClear` | `() => void` | вАФ | Called when the вЬХ button is clicked |
| `wrapperClassName` | `string` | вАФ | Applied to the outer `<div>` wrapper |
| `value` | `string \| number` | вАФ | Controlled value |
| `defaultValue` | `string \| number` | вАФ | Uncontrolled default value |
| `onChange` | `ChangeEventHandler` | вАФ | Change handler |
| `ref` | `Ref<HTMLInputElement>` | вАФ | Forwarded ref вАФ always points to `<input>` |

## Design Tokens

Height locked to `h-10` (40px) from `inputTokens.height`.
Focus ring: `--focus-ring` вАФ brand-blue 2px ring, 2px offset.
Prefix/suffix positioned at `left-3` / `right-3` (12px inset).

---

## Accessibility

- Renders native `<input>` вАФ always pair with a visible `<label>` or `aria-label`.
- Use `aria-invalid="true"` + `aria-describedby` to associate error messages.
- The вЬХ clear button has `aria-label="Clear"` вАФ screen readers announce it correctly.
- Prefix and suffix spans carry `aria-hidden` вАФ they are decorative and ignored by assistive tech.
- Focus ring is always visible for keyboard navigation.
