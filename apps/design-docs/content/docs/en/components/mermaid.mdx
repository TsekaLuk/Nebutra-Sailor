## Usage

```tsx
import { Mermaid } from "@nebutra/ui/primitives"
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `chart` | `string` | — | **Required.** Mermaid diagram definition string |
| `className` | `string` | — | Additional CSS classes on the container |

## Examples

### Sequence Diagram

```tsx
const sequenceDiagram = `
  sequenceDiagram
    participant Browser
    participant Server
    participant Database

    Browser->>Server: GET /api/user
    Server->>Database: SELECT * FROM users
    Database-->>Server: user data
    Server-->>Browser: 200 OK + JSON
`

<Mermaid chart={sequenceDiagram} />
```

### Entity Relationship

```tsx
const erDiagram = `
  erDiagram
    USER ||--o{ ORDER : places
    ORDER ||--|{ LINE-ITEM : contains
    PRODUCT }|--|{ LINE-ITEM : includes
`

<Mermaid chart={erDiagram} />
```

### State Diagram

```tsx
const stateDiagram = `
  stateDiagram-v2
    [*] --> Idle
    Idle --> Loading: fetch()
    Loading --> Success: resolve
    Loading --> Error: reject
    Error --> Idle: retry
    Success --> [*]
`

<Mermaid chart={stateDiagram} />
```

## Accessibility

- Renders as an SVG element — add `aria-label` or a visible title for screen reader context.
- Consider providing a text description alongside complex diagrams.
- SVG diagrams are not announced by default; wrap with a `<figure>` and `<figcaption>` for context.

```tsx
<figure>
  <Mermaid chart={chart} />
  <figcaption className="text-sm text-muted-foreground mt-2">
    User authentication flow diagram
  </figcaption>
</figure>
```
