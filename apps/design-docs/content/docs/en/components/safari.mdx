## Overview

`Safari` renders a macOS Safari browser window SVG frame. Unlike `BrowserMockup`, it uses a detailed SVG with native-looking toolbar icons (back/forward, share, tabs) in `"default"` mode, or a simplified clean address bar in `"simple"` mode.

Content (image or video) is clipped to the browser viewport area below the toolbar.

## Usage

```tsx
import { Safari } from "@nebutra/ui/primitives";

// URL only (no content)
<div className="w-[800px]">
  <Safari url="https://nebutra.com" />
</div>

// With screenshot
<div className="w-[800px]">
  <Safari
    url="https://app.nebutra.com"
    imageSrc="/screenshots/dashboard.png"
  />
</div>

// Simple mode (clean toolbar)
<div className="w-[800px]">
  <Safari
    url="https://nebutra.com"
    imageSrc="/screenshots/homepage.png"
    mode="simple"
  />
</div>

// With video
<div className="w-[800px]">
  <Safari
    url="https://nebutra.com"
    videoSrc="/demos/product.webm"
  />
</div>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `url` | `string` | — | URL displayed in the address bar |
| `imageSrc` | `string` | — | Screenshot or image URL |
| `videoSrc` | `string` | — | Video URL (autoplay, loop, muted) |
| `mode` | `"default" \| "simple"` | `"default"` | Toolbar detail level |
| `className` | `string` | — | Container CSS class |

## Notes

- `videoSrc` takes priority over `imageSrc`
- Video plays with `autoPlay loop muted playsInline`
- Frame aspect ratio is fixed at `1203/753`; control size via container width
- Frame adapts to dark mode via Tailwind `dark:fill-*` classes
- The content viewport starts at `y=52` (below the toolbar) and has rounded bottom corners
