## Overview

`ScrollVelocityContainer` + `ScrollVelocityRow` create a marquee-style horizontal content strip that speeds up and slows down based on your scroll velocity.

Key features:
- Pauses automatically when off-screen (`IntersectionObserver`)
- Pauses when tab is hidden (`visibilitychange`)
- Respects `prefers-reduced-motion`
- Duplicate-free seamless loop via `ResizeObserver` measuring

## Usage

```tsx
import { ScrollVelocityContainer, ScrollVelocityRow } from "@nebutra/ui/primitives";

// Dual rows with shared velocity
<ScrollVelocityContainer className="text-2xl font-bold">
  <ScrollVelocityRow baseVelocity={8} direction={1}>
    <span className="mx-8">Next.js</span>
    <span className="mx-8">TypeScript</span>
    <span className="mx-8">Tailwind</span>
  </ScrollVelocityRow>
  <ScrollVelocityRow baseVelocity={8} direction={-1}>
    <span className="mx-8">React</span>
    <span className="mx-8">Framer Motion</span>
    <span className="mx-8">Storybook</span>
  </ScrollVelocityRow>
</ScrollVelocityContainer>

// Standalone row (uses its own velocity)
<ScrollVelocityRow baseVelocity={15}>
  <span className="mx-8">Logo</span>
  <span className="mx-8">Partner</span>
</ScrollVelocityRow>
```

## Props

### `ScrollVelocityContainer`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | `ScrollVelocityRow` children |
| `className` | `string` | — | Additional CSS class |

Shares a smoothed scroll velocity (`useSpring`) across all child rows.

### `ScrollVelocityRow`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Content to repeat in the marquee |
| `baseVelocity` | `number` | `5` | Base speed (% of content width per second) |
| `direction` | `1 \| -1` | `1` | `1` = left-to-right, `-1` = right-to-left |
| `className` | `string` | — | Row container CSS class |

## Notes

- Content is duplicated internally to fill the container — you don't need to repeat items yourself
- Wrap with `ScrollVelocityContainer` for the velocity-on-scroll effect to work; standalone rows do have their own local scroll tracking but require a scrollable parent
- Higher `baseVelocity` means faster baseline speed; scroll multiplies it up to 5×
