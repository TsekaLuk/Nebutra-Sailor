## Overview

`Terminal` is a three-component set for displaying command-line style content:

- **`Terminal`** — the outer macOS-style frame with traffic light buttons
- **`TypingAnimation`** — types text character-by-character
- **`AnimatedSpan`** — fades in an entire line at once

By default, children execute in sequence: each item starts after the previous one completes. Supports viewport-triggered start via `startOnView`.

## Usage

```tsx
import { Terminal, TypingAnimation, AnimatedSpan } from "@nebutra/ui/primitives";

<Terminal>
  <TypingAnimation>pnpm dlx shadcn@latest init</TypingAnimation>
  <AnimatedSpan className="text-green-400">✔ Preflight checks.</AnimatedSpan>
  <AnimatedSpan className="text-green-400">✔ Validating Tailwind CSS.</AnimatedSpan>
  <TypingAnimation>Success! Project initialized.</TypingAnimation>
</Terminal>
```

### Without sequencing

```tsx
<Terminal sequence={false}>
  <AnimatedSpan delay={0}>Line 1 (instant)</AnimatedSpan>
  <AnimatedSpan delay={500}>Line 2 (after 500ms)</AnimatedSpan>
  <AnimatedSpan delay={1000}>Line 3 (after 1s)</AnimatedSpan>
</Terminal>
```

## Props

### `Terminal`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | `TypingAnimation` and `AnimatedSpan` children |
| `sequence` | `boolean` | `true` | Enable auto-sequencing |
| `startOnView` | `boolean` | `true` | Start when terminal enters viewport |
| `className` | `string` | — | Additional CSS class |

### `TypingAnimation`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `string` | — | Text to type (must be a string) |
| `duration` | `number` | `60` | Milliseconds per character |
| `delay` | `number` | `0` | Delay before starting (unsequenced mode) |
| `as` | `React.ElementType` | `"span"` | HTML element or component to render |
| `startOnView` | `boolean` | `true` | Start on viewport entry (unsequenced mode) |

### `AnimatedSpan`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Content to fade in |
| `delay` | `number` | `0` | Delay before appearing (unsequenced mode) |
| `startOnView` | `boolean` | `false` | Start on viewport entry (unsequenced mode) |
| `className` | `string` | — | Additional CSS class |

## Notes

- `TypingAnimation` children must be a plain `string` — it will throw if passed JSX
- In sequenced mode, `delay` props on individual items are ignored; timing is driven by completion of the previous item
- The terminal container has `max-h-[400px]` by default — use `className` to override
