title: Text
description: Typography primitive for rendering body text, captions, labels, and other inline content with semantic styling.
---

<ComponentPreview name="text-demo" />


## Usage

```tsx
import { Text } from "@nebutra/ui";

<Text variant="body">Body text content</Text>
```

## Examples

### Variants

```tsx
<Text variant="body">Body text — default paragraph style</Text>
<Text variant="body-sm">Small body — captions and secondary content</Text>
<Text variant="label">Label — form labels and metadata</Text>
<Text variant="label-sm">Small label — micro labels and tags</Text>
<Text variant="code">Monospace — code snippets inline</Text>
```

### Color Tokens

```tsx
<Text color="default">Default foreground color</Text>
<Text color="muted">Muted — secondary text, placeholders</Text>
<Text color="accent">Accent — highlighted text</Text>
```

### Truncation

```tsx
{/* Single line truncation */}
<Text truncate className="max-w-xs">
  This text will be truncated with an ellipsis if it exceeds the container width
</Text>

{/* Multi-line clamp */}
<Text lineClamp={2} className="max-w-sm">
  This text will be clamped to a maximum of two lines with an ellipsis at the end. Any content beyond the second line will be hidden from view.
</Text>
```

### Alignment

```tsx
<Text align="center" variant="body-sm" color="muted">
  Centered supporting text
</Text>
```

### Polymorphic Element

```tsx
{/* Render as a span for inline use */}
<Text as="span" variant="label" color="muted">
  Published on Jan 15, 2025
</Text>

{/* Render as a blockquote */}
<Text as="blockquote" variant="body" className="border-l-4 pl-4 italic">
  "Design is not just what it looks like and feels like. Design is how it works."
</Text>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `as` | `React.ElementType` | `"p"` | Render as a different HTML element |
| `variant` | `TextStyle` | `"body"` | Visual style from the typography token system |
| `color` | `TextColor` | `"default"` | Text color token |
| `truncate` | `boolean` | `false` | Truncate text with ellipsis at one line |
| `lineClamp` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | — | Clamp text to N lines with ellipsis |
| `align` | `"left" \| "center" \| "right"` | — | Text alignment |
| `className` | `string` | — | Additional Tailwind classes |

## Notes

- `truncate` and `lineClamp` are mutually exclusive — `truncate` takes precedence. Use one or the other.
- `lineClamp` requires `overflow: hidden` on the element, which is applied automatically via the Tailwind `line-clamp-*` utility.
- For semantic heading elements (h1–h6), use the `Heading` component instead.
