## Overview

`XPostCard` and `ClientXPostCard` embed X/Twitter posts using the `react-tweet` library. `XPostCard` wraps the client component in a Suspense boundary, making it the recommended choice for most use cases. A built-in `XPostSkeleton` is shown while the post loads.

## Installation

```tsx
import {
  XPostCard,
  ClientXPostCard,
  XPostSkeleton,
} from "@nebutra/ui/primitives";
```

`react-tweet` is a peer dependency — install if not already present:

```bash
pnpm add react-tweet
```

## Usage

### Basic (recommended)

```tsx
<XPostCard id="1441032681968212480" />
```

### With custom loading fallback

```tsx
<XPostCard
  id="1441032681968212480"
  fallback={<div className="animate-pulse h-48 rounded-lg bg-muted" />}
/>
```

### Client-only component

Use `ClientXPostCard` when you need to embed in a client component that already manages its own Suspense boundaries:

```tsx
"use client";

import { ClientXPostCard } from "@nebutra/ui/primitives";

export default function TweetSection() {
  return <ClientXPostCard id="1441032681968212480" />;
}
```

### Custom API URL

Override the tweet fetch endpoint (e.g., for caching or proxy):

```tsx
<XPostCard
  id="1441032681968212480"
  apiUrl="/api/tweet"
/>
```

### Error handling

```tsx
<XPostCard
  id="1441032681968212480"
  onError={(error) => {
    console.error("Failed to load tweet:", error);
    return error;
  }}
/>
```

### Skeleton standalone

```tsx
<XPostSkeleton className="max-w-md" />
```

## Props

### XPostCard / ClientXPostCard

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | — | **Required.** X/Twitter post ID |
| `className` | `string` | — | Additional CSS classes for the card container |
| `apiUrl` | `string` | — | Custom API URL for fetching tweet data |
| `fetchOptions` | `RequestInit` | — | Custom options passed to the tweet fetch request |
| `fallback` | `ReactNode` | `<XPostSkeleton />` | Content to show while loading |
| `components` | `TwitterComponents` | — | Custom component overrides for react-tweet internals |
| `onError` | `(error: Error \| undefined) => Error \| undefined` | — | Error handler callback |

### XPostSkeleton

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | — | Additional CSS classes |

## Notes

- The card uses design system tokens (`bg-card`, `text-card-foreground`, `border`) so it matches your theme automatically.
- `ClientXPostCard` is the actual rendering component — `XPostCard` just wraps it in Suspense.
- Use `XPostCard` in Server Components or anywhere a Suspense boundary is needed.
- When `onError` is not provided, the default `TweetNotFound` component from `react-tweet` is shown.
