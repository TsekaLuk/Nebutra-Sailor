---
title: "FormItemLayout"
description: "Structural wrapper combining a label, input slot, helper text, and validation message into a single layout unit."
---

## Demo

```tsx
<FormItemLayout
  label="Display name"
  htmlFor="display-name"
  helperText="This is how your name appears across Nebutra."
>
  <Input id="display-name" placeholder="Jane Smith" />
</FormItemLayout>
```

## Usage

```tsx
import { FormItemLayout } from "@nebutra/ui";
import { Input } from "@nebutra/ui";

export function ProfileForm() {
  const { register, formState: { errors } } = useForm<ProfileSchema>();

  return (
    <form>
      <FormItemLayout
        label="Display name"
        htmlFor="display-name"
        error={errors.displayName?.message}
        required
      >
        <Input
          id="display-name"
          {...register("displayName")}
          aria-invalid={!!errors.displayName}
        />
      </FormItemLayout>

      <FormItemLayout
        label="Bio"
        htmlFor="bio"
        helperText="Max 160 characters."
        error={errors.bio?.message}
      >
        <Textarea id="bio" {...register("bio")} rows={3} />
      </FormItemLayout>
    </form>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | `string` | — | **Required.** Text for the `<label>` element |
| `htmlFor` | `string` | — | `id` of the associated input; wires the label's `for` attribute |
| `children` | `React.ReactNode` | — | The input control(s) to render in the body slot |
| `helperText` | `string` | — | Muted hint text shown below the input |
| `error` | `string` | — | Validation error; replaces helper text and applies error styling |
| `required` | `boolean` | `false` | Appends a required indicator to the label |
| `labelSuffix` | `React.ReactNode` | — | Optional content rendered to the right of the label (e.g. a badge or tooltip) |
| `className` | `string` | — | Additional classes on the root `<div>` |

## Examples

### With label suffix (tooltip)

```tsx
<FormItemLayout
  label="Webhook secret"
  htmlFor="webhook-secret"
  labelSuffix={<InfoTooltip content="Used to validate incoming payloads." />}
>
  <Input id="webhook-secret" type="password" />
</FormItemLayout>
```

### Error state

```tsx
<FormItemLayout
  label="Slug"
  htmlFor="slug"
  error="Slug must be lowercase and contain no spaces."
>
  <Input id="slug" value="My Project" aria-invalid />
</FormItemLayout>
```

### Horizontal layout

```tsx
<FormItemLayout
  label="Enable notifications"
  htmlFor="notifications"
  layout="horizontal"
>
  <Switch id="notifications" />
</FormItemLayout>
```

## Accessibility

- The `<label>` element is always associated to the input via `htmlFor` / `id` — never via wrapping alone.
- Error messages render in a `<span role="alert">` so they are announced immediately by screen readers.
- Helper text is linked via `aria-describedby`; error text overrides it when both are present.
- The required indicator renders as `aria-hidden="true"` with `required` / `aria-required="true"` on the input itself.
- `FormItemLayout` does not render any focusable element itself — focus management is owned by the child input.
