---
title: "PageContainer"
description: "Section wrapper that enforces consistent max-width, vertical rhythm, and responsive padding across pages."
---

## Demo

```tsx
import { SectionContainer } from "@nebutra/ui";

<SectionContainer size="lg" spacing="md">
  <SectionContainer.Header>
    <h2 className="text-3xl font-bold">Features</h2>
    <p className="text-gray-500 mt-2">Everything you need to ship faster.</p>
  </SectionContainer.Header>

  <SectionContainer.Content>
    <FeatureGrid />
  </SectionContainer.Content>

  <SectionContainer.Footer>
    <Button>View all features</Button>
  </SectionContainer.Footer>
</SectionContainer>
```

## Usage

```tsx
import { SectionContainer } from "@nebutra/ui";

export function PricingPage() {
  return (
    <main>
      <SectionContainer size="xl" spacing="lg" as="section">
        <SectionContainer.Header spacing="lg">
          <h1 className="text-5xl font-bold">Pricing</h1>
          <p className="text-xl text-gray-600 mt-4">Start for free. Scale as you grow.</p>
        </SectionContainer.Header>

        <SectionContainer.Content>
          <PricingTable />
        </SectionContainer.Content>
      </SectionContainer>
    </main>
  );
}
```

## Props

### `SectionContainer` (`SectionContainerProps`)

| Prop | Type | Default | Description |
|---|---|---|---|
| `as` | `"section" \| "div" \| "article"` | `"section"` | HTML element to render as the outer wrapper |
| `size` | `"sm" \| "md" \| "lg" \| "xl" \| "full"` | `"lg"` | Max-width constraint applied to the inner content wrapper |
| `spacing` | `"sm" \| "md" \| "lg" \| "xl"` | `"md"` | Vertical padding applied to the outer wrapper |
| `centered` | `boolean` | `true` | When `true`, the inner wrapper is horizontally centered via `mx-auto` |
| `noPadding` | `boolean` | `false` | Removes the default horizontal `px-4 sm:px-6 lg:px-8` padding |
| `className` | `string` | — | Additional classes on the outer wrapper |
| `children` | `React.ReactNode` | — | Content — use sub-components for structured layout |

#### `size` values

| Value | Max-width |
|---|---|
| `sm` | `max-w-3xl` (48rem) |
| `md` | `max-w-5xl` (64rem) |
| `lg` | `max-w-6xl` (72rem) |
| `xl` | `max-w-7xl` (80rem) |
| `full` | No max-width constraint |

#### `spacing` values

| Value | Padding |
|---|---|
| `sm` | `py-12 md:py-16` |
| `md` | `py-16 md:py-24` |
| `lg` | `py-24 md:py-32` |
| `xl` | `py-32 md:py-48` |

### `SectionContainer.Header` (`SectionHeaderProps`)

| Prop | Type | Default | Description |
|---|---|---|---|
| `centered` | `boolean` | `true` | Centers header text |
| `spacing` | `"sm" \| "md" \| "lg"` | `"md"` | Bottom margin before the content area (`mb-8`, `mb-12`, `mb-16`) |
| `className` | `string` | — | Additional classes |
| `children` | `React.ReactNode` | — | Heading, subheading, badges, etc. |

### `SectionContainer.Content` (`SectionContentProps`)

Thin wrapper with `position: relative`. Accepts any `React.ReactNode` and `className`.

### `SectionContainer.Footer` (`SectionFooterProps`)

| Prop | Type | Default | Description |
|---|---|---|---|
| `centered` | `boolean` | `true` | Centers footer content |
| `spacing` | `"sm" \| "md" \| "lg"` | `"md"` | Top margin after the content area (`mt-8`, `mt-12`, `mt-16`) |
| `className` | `string` | — | Additional classes |
| `children` | `React.ReactNode` | — | CTA buttons, footnotes, etc. |

## Examples

### Full-width hero (no max-width)

```tsx
<SectionContainer size="full" spacing="xl" noPadding>
  <HeroBanner />
</SectionContainer>
```

### Article layout

```tsx
<SectionContainer as="article" size="sm" spacing="md">
  <SectionContainer.Content>
    <BlogPost post={post} />
  </SectionContainer.Content>
</SectionContainer>
```

### Without sub-components

```tsx
<SectionContainer size="lg" spacing="sm">
  <SimpleContent />
</SectionContainer>
```

## Accessibility

- The default `as="section"` requires a heading (`h1`–`h6`) as a descendant to give the landmark a name.
- If no visible heading is present, add `aria-label` to the `SectionContainer` to identify the landmark.
- `SectionContainer.Header`, `SectionContainer.Content`, and `SectionContainer.Footer` are purely structural `<div>` elements with no implicit ARIA roles.
