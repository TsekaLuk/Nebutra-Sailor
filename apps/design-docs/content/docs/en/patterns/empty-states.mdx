## When to Use

Every data surface that can be in one of three non-normal conditions needs a corresponding state:

| Condition | Component | Trigger |
|---|---|---|
| Zero data | `EmptyState` | Query returns an empty array |
| In-flight | `LoadingState` | Async request pending |
| Failed | `ErrorState` | Request throws or rejects |

Apply these states at the same level as the content they replace вЂ” inside a `Card`, a `Section`, or a full page region. Never show a blank area.

## Composition

### EmptyState

Use when a list, table, or data region has no records. Always include an action when the user can resolve the emptiness directly.

```tsx
import { EmptyState } from "@nebutra/ui";
import { PlusIcon } from "@primer/octicons-react";
import { Button } from "@nebutra/ui";

<EmptyState
  icon={<PlusIcon />}
  title="No API keys yet"
  description="Create your first key to start making requests."
  actions={
    <Button variant="primary" onClick={onCreateKey}>
      Create key
    </Button>
  }
/>
```

Props:

| Prop | Type | Default |
|---|---|---|
| `title` | `string` | required |
| `description` | `string` | вЂ” |
| `icon` | `ReactNode` | вЂ” |
| `actions` | `ReactNode` | вЂ” |
| `className` | `string` | вЂ” |

### LoadingState

Replace data regions while a request is in-flight. Use `size="large"` for full-page loading; `size="small"` inside compact cards.

```tsx
import { LoadingState } from "@nebutra/ui";

// Inside a section
<LoadingState size="medium" message="Fetching usage dataвЂ¦" />

// Full-page
<div className="min-h-screen flex items-center justify-center">
  <LoadingState size="large" />
</div>
```

For complex pages with multiple regions, prefer per-region `LoadingState` over a single full-page spinner. Users can see partial content sooner.

### ErrorState

Provide `onRetry` whenever the operation is idempotent and can be retried without user input.

```tsx
import { ErrorState } from "@nebutra/ui";

<ErrorState
  title="Failed to load usage"
  error={error}
  onRetry={refetch}
  retryLabel="Retry"
/>
```

For unrecoverable errors (auth, permissions), omit `onRetry` and render a `Button` linking to a resolution path instead.

### Skeleton (alternative loading pattern)

For content-heavy pages where layout stability matters, use `Skeleton` atoms in place of `LoadingState`. Skeletons match the shape of real content and reduce layout shift.

```tsx
import { Skeleton } from "@nebutra/ui";

// Placeholder for a metric card
<div className="space-y-2">
  <Skeleton className="h-4 w-24" />
  <Skeleton className="h-8 w-16" />
</div>
```

Use `LoadingState` (spinner) when load time is unpredictable. Use `Skeleton` when the shape and count of content are known.

## Examples

### Table with all three states

```tsx
function UsageTable() {
  const { data, isLoading, error, refetch } = useUsage();

  if (isLoading) return <LoadingState size="medium" />;
  if (error)     return <ErrorState error={error} onRetry={refetch} />;
  if (!data?.length) {
    return (
      <EmptyState
        title="No usage recorded"
        description="Usage will appear here once you make your first API call."
      />
    );
  }

  return <DataTable data={data} columns={columns} />;
}
```
