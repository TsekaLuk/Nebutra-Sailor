## Semantic Colors

Semantic colors communicate meaning. Each role has a light and dark mode value.

| Role | Light Mode | Dark Mode | CSS Variable | Usage |
|------|-----------|-----------|-------------|-------|
| **Success** | `#22c55e` | `#16a34a` | `--color-success` | Confirmations, completion states |
| **Warning** | `#f59e0b` | `#d97706` | `--color-warning` | Caution states, pending actions |
| **Error** | `#ef4444` | `#dc2626` | `--color-error` | Destructive actions, validation errors |
| **Info** | `#0033FE` | `#5c7cfa` | `--color-info` | Informational callouts, tooltips |

<Warning>
  **Info = Brand Blue** — The VI manual explicitly specifies that the Info semantic color uses `#0033FE` (brand blue), not a sky or teal color.
  Never use `sky-500` (`#0ea5e9`) or `cyan-500` (`#06b6d4`) for info states.
</Warning>

### Semantic Background Variants

For backgrounds and subtle fills, use 10% opacity of the semantic color:

```css
/* Success background */
background: rgba(34, 197, 94, 0.1);

/* Error background */
background: rgba(239, 68, 68, 0.1);

/* Warning background */
background: rgba(245, 158, 11, 0.1);

/* Info background */
background: rgba(0, 51, 254, 0.1);
```

## Color Accessibility

### Contrast Ratios (WCAG 2.1 AA)

| Foreground | Background | Ratio | Pass AA | Pass AAA |
|-----------|-----------|-------|---------|----------|
| `#0033FE` (blue-500) | `#ffffff` (white) | 6.4:1 | Yes | Yes (large text) |
| `#0033FE` (blue-500) | `#f0f4ff` (blue-50) | 5.8:1 | Yes | Yes (large text) |
| `#ffffff` (white) | `#0033FE` (blue-500) | 6.4:1 | Yes | Yes (large text) |
| `#0BF1C3` (cyan-500) | `#020617` (neutral-950) | 12.1:1 | Yes | Yes |
| `#0BF1C3` (cyan-500) | `#ffffff` (white) | 1.7:1 | No | No |
| `#0f172a` (neutral-900) | `#ffffff` (white) | 16.1:1 | Yes | Yes |

<Warning>
  **Cyan on white fails WCAG contrast.** Never use `#0BF1C3` as text on light backgrounds.
  Cyan is safe as text on dark backgrounds (`neutral-900`, `neutral-950`) and as decorative/gradient fills.
</Warning>

### Guidelines

- Use `blue-500` for interactive text (links, buttons) on light backgrounds — it passes WCAG AA
- Use `blue-400` (`#5c7cfa`) for interactive text on dark backgrounds
- Use `cyan-500` only on dark backgrounds or as gradient/decorative fills — it fails contrast on white
- Body text must always use `neutral-700` (light) or `neutral-100` (dark) for maximum readability
- Minimum contrast ratio: **4.5:1** for normal text, **3:1** for large text (>= 18px bold or ≥ 24px regular)

## Prohibited Uses

<Warning>
  The following color substitutions are strictly forbidden:
</Warning>

| Prohibited | Reason |
|-----------|--------|
| Tailwind `sky-500` for brand blue | Sky blue is not Nebutra blue |
| Tailwind `teal-500` for brand cyan | Teal is not Nebutra cyan |
| Tailwind `gray-*` for neutrals | Nebutra neutrals have a blue undertone |
| Orange or red as primary action color | Only brand blue is used for primary actions |
| Cyan as text on light backgrounds | Fails WCAG contrast — use only on dark backgrounds |
| Brand blue as a flat background fill | Use the brand gradient for hero/CTA surfaces |
| Any non-brand color for `info` semantic | Info must use brand blue `#0033FE`, not sky/teal |

---

## Do / Don't

**Do** use `blue-500` (`#0033FE`) for all primary interactive elements (buttons, links, focus rings).

**Do** use the full 11-step scale for subtle tints, hover states, and dark mode adaptations.

**Do** pair `blue-500` with `cyan-500` exclusively in gradients — this is the brand signature.

**Do** use `neutral-950` as the dark mode page background.

**Don't** use cyan on white — it fails WCAG contrast.

**Don't** approximate brand colors with Tailwind defaults (`blue-600`, `emerald-400`).

**Don't** introduce new color scales outside the defined blue, cyan, neutral, and semantic palettes.

**Don't** use brand blue as a solid fill where the brand gradient is expected (hero sections, primary CTAs).
