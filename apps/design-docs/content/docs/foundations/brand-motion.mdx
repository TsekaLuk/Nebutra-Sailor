<Callout>
  Motion is not decoration. Every animation in Nebutra maps to a data metaphor — data materializing, streaming, or pulsing.
  Arbitrary animation is prohibited.
</Callout>

## Motion Philosophy: Cloud Nurture — 云端孕育

Nebutra's motion language is inspired by cloud computing data flows. Three core motions represent the lifecycle of data:

1. **Emerge (涌现)** — data materializing from the cloud into view
2. **Flow (流动)** — data streaming through pipelines and interfaces
3. **Pulse (脉动)** — systems alive, breathing, actively processing

Every UI animation must map to one of these three metaphors.

---

## Core Motions

### Emerge (涌现)

Data materializing from the cloud. Use for content entering the viewport, modals opening, and page transitions.

| Property | From | To |
|----------|------|----|
| `translateY` | `16px` | `0` |
| `filter: blur` | `6px` | `0` |
| `opacity` | `0` | `1` |

| Timing | Value |
|--------|-------|
| Duration | `0.6s` |
| Easing | `cubic-bezier(0.16, 1, 0.3, 1)` |

```tsx
import { motion } from "framer-motion"
import { brandMotion } from "@nebutra/brand"

// Using brand preset
<motion.div {...brandMotion.emerge}>
  <h1>Welcome to Nebutra</h1>
</motion.div>

// Manual implementation
<motion.div
  initial={{ y: 16, filter: "blur(6px)", opacity: 0 }}
  animate={{ y: 0, filter: "blur(0px)", opacity: 1 }}
  transition={{ duration: 0.6, ease: [0.16, 1, 0.3, 1] }}
>
  <h1>Welcome to Nebutra</h1>
</motion.div>
```

### Flow (流动)

Data streaming through pipes. Use for horizontal transitions, slide-in panels, navigation changes, and sequential content.

| Property | From | To |
|----------|------|----|
| `translateX` | `-20px` | `0` |
| `opacity` | `0` | `1` |

| Timing | Value |
|--------|-------|
| Duration | `0.4s` |
| Easing | `cubic-bezier(0, 0, 0.2, 1)` |

```tsx
import { motion } from "framer-motion"
import { brandMotion } from "@nebutra/brand"

// Using brand preset
<motion.div {...brandMotion.flow}>
  <SidePanel />
</motion.div>

// Manual implementation
<motion.div
  initial={{ x: -20, opacity: 0 }}
  animate={{ x: 0, opacity: 1 }}
  transition={{ duration: 0.4, ease: [0, 0, 0.2, 1] }}
>
  <SidePanel />
</motion.div>
```

### Pulse (脉动)

System alive and breathing. Use for status indicators, loading states, and ambient background elements. This is the only infinite animation.

| Property | Keyframes |
|----------|-----------|
| `scale` | `1 → 1.015 → 1` |
| `opacity` | `1 → 0.85 → 1` |

| Timing | Value |
|--------|-------|
| Duration | `3s` |
| Repeat | `Infinity` |

```tsx
import { motion } from "framer-motion"
import { brandMotion } from "@nebutra/brand"

// Using brand preset
<motion.div {...brandMotion.pulse}>
  <StatusIndicator />
</motion.div>

// Manual implementation
<motion.div
  animate={{
    scale: [1, 1.015, 1],
    opacity: [1, 0.85, 1],
  }}
  transition={{
    duration: 3,
    repeat: Infinity,
    ease: "easeInOut",
  }}
>
  <StatusIndicator />
</motion.div>
```

## Brand Easing Curves

| Name | Value | Description | Usage |
|------|-------|-------------|-------|
| **Brand** | `[0.16, 1, 0.3, 1]` | Custom deceleration — fast start, smooth stop | Primary easing for emerge and reveal |
| **Enter** | `[0, 0, 0.2, 1]` | Material deceleration | Elements entering the viewport |
| **Exit** | `[0.4, 0, 1, 1]` | Material acceleration | Elements leaving the viewport |
| **Spring** | Spring physics (see below) | Natural bounce | Interactive micro-motions |

### CSS Custom Properties

```css
:root {
  --ease-brand: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-enter: cubic-bezier(0, 0, 0.2, 1);
  --ease-exit: cubic-bezier(0.4, 0, 1, 1);
}
```

### Framer Motion Usage

```tsx
import { brandEasing } from "@nebutra/brand"

// brandEasing.brand  → [0.16, 1, 0.3, 1]
// brandEasing.enter  → [0, 0, 0.2, 1]
// brandEasing.exit   → [0.4, 0, 1, 1]

<motion.div
  transition={{ ease: brandEasing.brand, duration: 0.6 }}
/>
```

<Warning>
  Never use generic `ease-in-out` or `linear` easing in Nebutra products.
  Always use brand easing curves for predictable, on-brand motion.
</Warning>

## Brand Duration Scale

| Token | Value | Usage |
|-------|-------|-------|
| `instant` | `0.1s` | Hover color changes, toggle states |
| `fast` | `0.2s` | Button presses, checkbox toggles, small UI feedback |
| `normal` | `0.3s` | Dropdown open/close, tooltip appear, tab switch |
| `moderate` | `0.4s` | Flow transitions, panel slides, navigation changes |
| `slow` | `0.6s` | Emerge animations, modal open, page transitions |
| `reveal` | `0.8s` | Maximum duration — hero section reveal, large viewport enters |

### Code

```tsx
import { brandDuration } from "@nebutra/brand"

// brandDuration.instant  → 0.1
// brandDuration.fast     → 0.2
// brandDuration.normal   → 0.3
// brandDuration.moderate → 0.4
// brandDuration.slow     → 0.6
// brandDuration.reveal   → 0.8

<motion.div
  transition={{ duration: brandDuration.slow, ease: brandEasing.brand }}
/>
```

## Brand Spring Configs

Spring physics create natural, responsive motion for interactive elements. Four presets cover all use cases.

| Name | Stiffness | Damping | Mass | Character | Usage |
|------|-----------|---------|------|-----------|-------|
| **Default** | 200 | 24 | 1.0 | Balanced, professional | General UI transitions |
| **Bouncy** | 300 | 15 | 0.8 | Energetic, playful | Button feedback, notifications, success states |
| **Heavy** | 120 | 28 | 1.5 | Weighty, deliberate | Modal transitions, large panels, drawer open |
| **Gentle** | 80 | 20 | 1.0 | Soft, ambient | Background elements, subtle repositioning |

### Framer Motion Usage

```tsx
import { brandSpring } from "@nebutra/brand"

// Default spring
<motion.div
  animate={{ scale: 1 }}
  transition={{ type: "spring", ...brandSpring.default }}
/>

// Bouncy spring — button press
<motion.button
  whileTap={{ scale: 0.95 }}
  transition={{ type: "spring", ...brandSpring.bouncy }}
>
  Click Me
</motion.button>

// Heavy spring — modal entrance
<motion.div
  initial={{ y: 100, opacity: 0 }}
  animate={{ y: 0, opacity: 1 }}
  transition={{ type: "spring", ...brandSpring.heavy }}
>
  <ModalContent />
</motion.div>

// Gentle spring — background element
<motion.div
  animate={{ x: targetX }}
  transition={{ type: "spring", ...brandSpring.gentle }}
/>
```

### Raw Values

```tsx
import { brandSpring } from "@nebutra/brand"

// brandSpring.default → { stiffness: 200, damping: 24, mass: 1 }
// brandSpring.bouncy  → { stiffness: 300, damping: 15, mass: 0.8 }
// brandSpring.heavy   → { stiffness: 120, damping: 28, mass: 1.5 }
// brandSpring.gentle  → { stiffness: 80,  damping: 20, mass: 1 }
```

## Stagger Pattern

Use stagger for list and grid entrance animations. Each child enters with a slight delay after the previous one.

```tsx
import { motion } from "framer-motion"
import { brandMotion, brandDuration, brandEasing } from "@nebutra/brand"

const container = {
  hidden: { opacity: 0 },
  show: {
    opacity: 1,
    transition: {
      staggerChildren: 0.08,
      delayChildren: 0.1,
    },
  },
}

const item = {
  hidden: { y: 16, filter: "blur(6px)", opacity: 0 },
  show: {
    y: 0,
    filter: "blur(0px)",
    opacity: 1,
    transition: {
      duration: brandDuration.slow,
      ease: brandEasing.brand,
    },
  },
}

<motion.ul variants={container} initial="hidden" animate="show">
  {items.map((data) => (
    <motion.li key={data.id} variants={item}>
      <Card {...data} />
    </motion.li>
  ))}
</motion.ul>
```

| Parameter | Value | Description |
|-----------|-------|-------------|
| `staggerChildren` | `0.08s` | Delay between each child's entrance |
| `delayChildren` | `0.1s` | Initial delay before the first child enters |

## Interactive Micro-Motions

### Hover

```tsx
import { brandSpring } from "@nebutra/brand"

<motion.div
  whileHover={{ scale: 1.02 }}
  transition={{ type: "spring", ...brandSpring.default }}
>
  <Card />
</motion.div>
```

### Tap

```tsx
<motion.button
  whileTap={{ scale: 0.97 }}
  transition={{ type: "spring", ...brandSpring.bouncy }}
>
  Submit
</motion.button>
```

### Hover Lift

Combines vertical lift with shadow elevation. Use for cards and interactive surfaces.

```tsx
<motion.div
  whileHover={{
    y: -4,
    boxShadow: "0 20px 40px rgba(0, 51, 254, 0.15)",
  }}
  transition={{ type: "spring", ...brandSpring.default }}
>
  <FeatureCard />
</motion.div>
```

## Viewport Triggers

### Once (Default)

Animate only the first time the element enters the viewport. Use for content sections, feature grids, and testimonials.

```tsx
<motion.section
  initial={{ y: 16, filter: "blur(6px)", opacity: 0 }}
  whileInView={{ y: 0, filter: "blur(0px)", opacity: 1 }}
  viewport={{ once: true, margin: "-100px" }}
  transition={{ duration: 0.6, ease: [0.16, 1, 0.3, 1] }}
>
  <FeatureSection />
</motion.section>
```

### Always

Re-animate every time the element enters the viewport. Use sparingly — only for ambient background elements and decorative motions.

```tsx
<motion.div
  initial={{ opacity: 0 }}
  whileInView={{ opacity: 1 }}
  viewport={{ once: false, amount: 0.3 }}
  transition={{ duration: 0.4 }}
>
  <DecorativeElement />
</motion.div>
```

<Tip>
  Default to `once: true` for all content animations.
  Use `once: false` only for ambient/decorative elements that benefit from re-entrance.
</Tip>

## Do / Don't

**Do** use `emerge` for all content entering the viewport (vertical reveal with blur).

**Do** use `flow` for horizontal transitions (panels, navigation, sequential content).

**Do** use `pulse` for status indicators and ambient breathing effects.

**Do** use brand easing curves (`[0.16, 1, 0.3, 1]`) for all tween animations.

**Do** use brand springs for all interactive micro-motions (hover, tap, drag).

**Do** stagger list/grid entrances with `0.08s` delay between children.

**Do** respect `prefers-reduced-motion` — disable all non-essential animation.

**Don't** bounce everything — use `brandSpring.bouncy` only for button feedback and success states.

**Don't** use generic `ease-in-out` or `linear` easing — always use brand curves.

**Don't** exceed `0.8s` duration for any UI transition — `reveal` (0.8s) is the absolute maximum.

**Don't** use `pulse` for content elements — it is reserved for status indicators and ambient effects.

**Don't** animate more than 3 properties simultaneously — keep motion focused and readable.

**Don't** create infinite animations outside of `pulse` — only status indicators loop.
