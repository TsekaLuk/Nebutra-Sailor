---
title: "ConfirmationModal"
description: "Confirmation dialog with a destructive action guard to prevent accidental irreversible operations."
---

## Demo

```tsx
<ConfirmationModal
  open={isOpen}
  onOpenChange={setIsOpen}
  title="Delete workspace"
  description="This action cannot be undone. All projects, pipelines, and members will be permanently removed."
  confirmLabel="Delete workspace"
  cancelLabel="Cancel"
  variant="destructive"
  onConfirm={handleDelete}
/>
```

## Usage

```tsx
import { ConfirmationModal } from "@nebutra/ui";
import { useState } from "react";

export function WorkspaceSettings() {
  const [open, setOpen] = useState(false);

  async function handleDelete() {
    await deleteWorkspace(workspaceId);
    router.push("/dashboard");
  }

  return (
    <>
      <Button variant="danger" onClick={() => setOpen(true)}>
        Delete workspace
      </Button>

      <ConfirmationModal
        open={open}
        onOpenChange={setOpen}
        title="Delete workspace"
        description="This action is permanent. All data will be lost."
        confirmLabel="Delete workspace"
        variant="destructive"
        onConfirm={handleDelete}
      />
    </>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `open` | `boolean` | — | Controls dialog open state (controlled mode) |
| `onOpenChange` | `(open: boolean) => void` | — | Called when the dialog requests a state change |
| `title` | `string` | — | Dialog heading text |
| `description` | `string` | — | Explanatory body copy below the title |
| `confirmLabel` | `string` | `"Confirm"` | Label for the confirm button |
| `cancelLabel` | `string` | `"Cancel"` | Label for the cancel button |
| `variant` | `"default" \| "destructive"` | `"default"` | `"destructive"` renders the confirm button in danger red |
| `onConfirm` | `() => void \| Promise<void>` | — | Called when the user clicks the confirm button |
| `onCancel` | `() => void` | — | Called when the user clicks cancel or presses Escape |
| `isLoading` | `boolean` | `false` | Shows a spinner on the confirm button during async operations |
| `className` | `string` | — | Additional classes on the dialog panel |

## Examples

### Non-destructive confirmation

```tsx
<ConfirmationModal
  open={open}
  onOpenChange={setOpen}
  title="Publish changes"
  description="Your changes will be visible to all users immediately."
  confirmLabel="Publish"
  variant="default"
  onConfirm={handlePublish}
/>
```

### Async confirm with loading state

```tsx
const [loading, setLoading] = useState(false);

async function handleConfirm() {
  setLoading(true);
  try {
    await deleteResource(id);
    setOpen(false);
  } finally {
    setLoading(false);
  }
}

<ConfirmationModal
  open={open}
  onOpenChange={setOpen}
  title="Remove integration"
  description="This will disconnect your Slack workspace."
  confirmLabel="Remove"
  variant="destructive"
  onConfirm={handleConfirm}
  isLoading={loading}
/>
```

## Accessibility

- Renders a native `<dialog>` element with `role="alertdialog"` for destructive variants and `role="dialog"` for default.
- `aria-labelledby` points to the title element; `aria-describedby` points to the description.
- Focus is trapped inside the dialog while open and returns to the trigger element on close.
- Pressing `Escape` closes the dialog and calls `onCancel`.
- Confirm button of `variant="destructive"` includes `aria-describedby` referencing the description to give screen reader users full context before acting.
