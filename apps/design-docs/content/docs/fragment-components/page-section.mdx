---
title: "PageSection"
description: "Content section with a heading, optional description, and a divider for grouping related UI."
---

## Demo

```tsx
<PageSection
  title="Danger zone"
  description="Irreversible actions. Proceed with caution."
  variant="danger"
>
  <DeleteWorkspaceCard />
</PageSection>
```

## Usage

```tsx
import { PageSection } from "@nebutra/ui";

export function WorkspaceSettings() {
  return (
    <div className="space-y-12">
      <PageSection title="General" description="Basic workspace configuration.">
        <GeneralSettingsForm />
      </PageSection>

      <PageSection title="Members" description="Manage access and roles.">
        <MemberTable />
      </PageSection>

      <PageSection
        title="Danger zone"
        description="These actions are permanent and cannot be undone."
        variant="danger"
      >
        <DeleteWorkspaceCard />
      </PageSection>
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | — | **Required.** Section heading text |
| `description` | `string` | — | Muted body copy below the heading |
| `variant` | `"default" \| "danger"` | `"default"` | `"danger"` applies red accent styling to the title and border |
| `titleLevel` | `"h2" \| "h3" \| "h4"` | `"h2"` | Heading element for correct document outline |
| `actions` | `React.ReactNode` | — | Optional controls rendered to the right of the title |
| `divider` | `boolean` | `true` | Renders a horizontal rule above the section |
| `className` | `string` | — | Additional classes on the root container |
| `children` | `React.ReactNode` | — | Section body content |

## Examples

### With section-level action

```tsx
<PageSection
  title="API keys"
  description="Manage programmatic access to your workspace."
  actions={<Button size="sm" onClick={() => setOpen(true)}>Generate key</Button>}
>
  <ApiKeyTable />
</PageSection>
```

### No divider (first section)

```tsx
<PageSection title="General" divider={false}>
  <GeneralSettingsForm />
</PageSection>
```

### Nested h3 for sub-sections

```tsx
<PageSection title="Notifications" description="Configure alert delivery channels.">
  <PageSection title="Email" titleLevel="h3" divider={false}>
    <EmailAlertForm />
  </PageSection>

  <PageSection title="Slack" titleLevel="h3">
    <SlackAlertForm />
  </PageSection>
</PageSection>
```

## Accessibility

- The `title` renders at the heading level specified by `titleLevel` — use `h2` for top-level settings sections and `h3` for nested sub-sections to maintain a correct document outline.
- `variant="danger"` does not change the heading's semantic role; it is purely visual.
- `divider` renders as `<hr aria-hidden="true">`.
- The `actions` slot should contain focusable elements in logical tab order after the heading.
