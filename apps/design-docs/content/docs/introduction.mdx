## Principles

<AccordionGroup>
  <Accordion title="Source of truth — specs come from production">
    Every token, component, and pattern on this site is extracted from code that ships.
    Nothing is invented or aspirational. If a pattern is not in production it is not documented here.
  </Accordion>
  <Accordion title="Reuse first — catalogue before creating">
    Before building a new component, search the atom catalogue. If a close match exists, extend it.
    Duplication is the primary enemy of a healthy design system.
  </Accordion>
  <Accordion title="Abstract, don't refactor">
    New code follows the system immediately. Existing code migrates gradually.
    We do not hold up shipping to achieve perfect consistency everywhere at once.
  </Accordion>
  <Accordion title="No inline CSS">
    All styling uses design tokens (CSS variables), Tailwind utility classes, or component `variant` props.
    Inline `style=` is forbidden except for truly dynamic values (e.g. chart colours at runtime).
  </Accordion>
  <Accordion title="Minimal scope — only high-frequency UI is formalised">
    We document components and patterns that appear on 3 or more distinct pages.
    One-off UI stays local to the feature — it does not belong in the system.
  </Accordion>
</AccordionGroup>
