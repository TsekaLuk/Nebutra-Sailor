# Nebutra API 参考

欢迎查阅 Nebutra API 文档。API 网关提供了一整套系统监控、法律合规和事件处理的 API 端点。

## 基础 URL

```
生产环境: https://api.nebutra.com
开发环境: http://localhost:3002
```

## 认证

大多数 API 端点需要通过 `Authorization` Header 传递 Clerk 签发的 JWT 令牌。

```bash
curl -H "Authorization: Bearer <your-jwt-token>" \
  https://api.nebutra.com/api/v1/events/ingest
```

## 速率限制

`/api/*` 下的 API 端点均受速率限制（健康检查、状态和 Webhook 端点除外）。响应中包含速率限制头：

| 头部 | 描述 |
|------|------|
| `X-RateLimit-Limit` | 窗口期内最大请求数 |
| `X-RateLimit-Remaining` | 当前窗口期剩余请求数 |
| `X-RateLimit-Reset` | 速率限制重置时间 |

## 响应头

所有响应包含：

| 头部 | 描述 |
|------|------|
| `X-Request-ID` | 调试用的唯一请求标识符 |
| `X-Trace-ID` | OpenTelemetry 追踪 ID（可用时） |
| `API-Version` | 当前 API 版本 |
