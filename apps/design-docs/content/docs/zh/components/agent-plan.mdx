## 使用方法 (Usage)

```tsx
import { AgentPlan } from "@nebutra/ui";
import type { AgentTask } from "@nebutra/ui";

const tasks: AgentTask[] = [
  {
    id: "1",
    title: "调研需求 (Research Requirements)",
    description: "从利益相关者处收集并分析项目需求",
    status: "in-progress",
    subtasks: [
      {
        id: "1.1",
        title: "访谈利益相关者 (Interview stakeholders)",
        description: "安排并进行 30 分钟的访谈",
        status: "completed",
        tools: ["meeting-scheduler", "notes"],
      },
      {
        id: "1.2",
        title: "审查现有文档 (Review existing docs)",
        description: "阅读所有设计文档和规范",
        status: "in-progress",
        tools: ["file-system", "browser"],
      },
    ],
  },
];

<AgentPlan
  tasks={tasks}
  defaultExpandedTasks={["1"]}
  onTaskStatusChange={(id, status) => console.log(id, status)}
/>;
```

### 只读状态展示 (Read-only Status Display)

```tsx
<AgentPlan
  tasks={executedTasks}
  allowStatusToggle={false}
  defaultExpandedTasks={executedTasks.map((t) => t.id)}
/>
```


## 状态类型 (Status Types)

| 状态 (Status) | 图标 (Icon) | 描述 (Description) |
| ------------- | ---------------------- | ------------------------------- |
| `pending`     | 圆圈 (置灰) | 任务尚未开始 |
| `in-progress` | 虚线圆点 (蓝色) | 正在执行中 |
| `completed`   | 打钩圆圈 (绿色) | 已成功完成 |
| `need-help`   | 警告圆圈 (黄色) | 已阻塞或需要人工介入 |
| `failed`      | 叉号圆圈 (红色) | 执行失败 |

## 属性 (Props)

### AgentPlan

| 属性 (Prop) | 类型 | 默认值 | 描述 |
| ----------------------- | -------------------------------------------------------------------- | ---------------- | -------------------------------------------------------- |
| `tasks`                 | `AgentTask[]`                                                        | —                | 要显示的任务数组 |
| `onTaskStatusChange`    | `(taskId: string, newStatus: TaskStatus) => void`                    | —                | 任务状态循环切换时的回调 |
| `onSubtaskStatusChange` | `(taskId: string, subtaskId: string, newStatus: TaskStatus) => void` | —                | 子任务状态切换时的回调 |
| `defaultExpandedTasks`  | `string[]`                                                           | `[]`             | 初始渲染时展开的任务 ID 数组 |
| `allowStatusToggle`     | `boolean`                                                            | `true`           | 点击状态图标是否循环切换状态 |
| `toolsLabel`            | `string`                                                             | `"MCP Servers:"` | 展开的子任务中在工具标签前显示的文本 |
| `className`             | `string`                                                             | —                | 容器的额外 Tailwind 类名 |

### AgentTask

| 字段 (Field) | 类型 | 描述 |
| -------------- | ----------------------------- | ---------------------------------------------- |
| `id`           | `string`                      | 唯一标识符 |
| `title`        | `string`                      | 任务行中显示的标题 |
| `description`  | `string`                      | 展开后显示的详细描述 |
| `status`       | `TaskStatus`                  | 当前任务状态 |
| `priority`     | `"low" \| "medium" \| "high"` | 可选的优先级 |
| `level`        | `number`                      | 嵌套层级 (0 = 根级) |
| `dependencies` | `string[]`                    | 该任务依赖的任务 ID —— 以徽章形式显示 |
| `subtasks`     | `AgentSubtask[]`              | 子任务（一层嵌套） |

### AgentSubtask

| 字段 (Field) | 类型 | 描述 |
| ------------- | ----------------------------- | --------------------------------------------- |
| `id`          | `string`                      | 唯一标识符 |
| `title`       | `string`                      | 子任务标题 |
| `description` | `string`                      | 点击子任务展开时显示的内容 |
| `status`      | `TaskStatus`                  | 当前子任务状态 |
| `priority`    | `"low" \| "medium" \| "high"` | 可选的优先级 |
| `tools`       | `string[]`                    | MCP 服务或工具名称 —— 以标签形式显示 |

## 注意事项 (Notes)

- **状态循环**: 当 `allowStatusToggle` 为 true 时，点击状态图标的切换顺序为：pending → in-progress → completed → need-help → failed → pending。
- **子任务完成**: 当子任务切换为已完成时，如果所有子任务都已完成，父任务会自动标记为已完成。
- **完成任务**: 当一个任务被设为 `completed` 时，其所有子任务都会自动标记为已完成。
- **减弱动画**: 组件会读取 `window.matchMedia("(prefers-reduced-motion: reduce)")` 并据此调整动画行为。
- **依赖关系**: `dependencies` 数组包含 **ID 字符串**，它们会被渲染为小的标签徽章 —— 组件内部没有内置自动的依赖强制执行逻辑。
