## 概览 (Overview)

`Browser` 渲染一个受 Vercel Geist 设计系统启发的装饰性浏览器框架。它包括：

- 红绿灯按钮（关闭、最小化、最大化）
- 导航按钮（后退、前进、刷新）—— 装饰性
- 带有内容复制按钮的地址栏
- 接受任意 `children` 的灵活内容区域

## 使用方法 (Usage)

```tsx
import { Browser } from "@nebutra/ui/primitives";

// 子组件模式 —— 任意 JSX
<Browser address="vercel.com">
  <div className="p-6">
    <h1>Hello World</h1>
  </div>
</Browser>

// 图像便捷模式
<Browser address="example.com" imageSrc="/screenshot.png" />

// 空框架
<Browser address="app.example.com" />
```

## 属性 (Props)

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `address` | `string` | — | 地址栏中显示的 URL 文本 |
| `imageSrc` | `string` | — | 便捷属性：在未提供子组件时渲染 `<img>` |
| `imageAlt` | `string` | `"Preview"` | 图像的替代文本（仅在使用 `imageSrc` 时生效） |
| `children` | `ReactNode` | — | 浏览器视口中渲染的任意内容（优先级高于 `imageSrc`） |
| `className` | `string` | — | 外部容器的额外 CSS 类名 |

所有其他 `<div>` 属性都会转发给容器元素。

## 从 BrowserMockup 迁移 (Migration from BrowserMockup)

`BrowserMockup` 仍作为已弃用的别名进行导出。迁移方法：

```diff
- import { BrowserMockup } from "@nebutra/ui/primitives";
+ import { Browser } from "@nebutra/ui/primitives";

- <BrowserMockup url="example.com" image="/screenshot.png" />
+ <Browser address="example.com" imageSrc="/screenshot.png" />

- <BrowserMockup url="example.com" placeholder={<MyContent />} />
+ <Browser address="example.com"><MyContent /></Browser>
```

## 注意事项 (Notes)

- 地址栏复制按钮使用 `navigator.clipboard.writeText()` —— 在剪贴板 API 不可用的上下文中会静默失败。
- 导航按钮（后退、前进、刷新）纯粹是装饰性的 —— 它们不会触发任何操作。
- 内容区域的高度由 `children` 内容决定；如果未提供内容，则回退到 `aspect-video` 的空状态。
- 无 `next/image` 依赖 —— 为 `imageSrc` 便捷属性使用普通的 `<img>` 标签。
- 通过 Tailwind 的 `dark:` 类名完全支持暗黑模式。
