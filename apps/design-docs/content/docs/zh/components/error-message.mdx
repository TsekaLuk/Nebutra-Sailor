## 概览 (Overview)

`ErrorMessage` 渲染一个行内错误指示器 —— 包含一个红色警示图标、一个可选的标签前缀、消息文本以及一个可选的操作链接。它对应于 Geist 的 `<Error>` 组件（为了避免与 JavaScript 内置的 `Error` 对象冲突，我们将其导出命名为 `ErrorMessage`）。

## 安装 (Installation)

```tsx
import { ErrorMessage } from "@nebutra/ui/primitives";
```

## 使用方法 (Usage)

### 默认用法 (Default)

```tsx
<ErrorMessage>此邮箱地址已被使用 (This email address is already in use.)</ErrorMessage>
```

### 自定义标签 (Custom label)

```tsx
<ErrorMessage label="邮箱错误 (Email Error)">
  此邮箱地址已被使用 (This email address is already in use.)
</ErrorMessage>
```

### 尺寸 (Sizes)

```tsx
<ErrorMessage size="small">此邮箱已被使用。</ErrorMessage>
<ErrorMessage size="medium">此邮箱已被使用。</ErrorMessage>
<ErrorMessage size="large">此邮箱已被使用。</ErrorMessage>
```

### 带有操作链接（传入错误对象） (With action link)

```tsx
<ErrorMessage
  error={{
    message: "请求失败 (The request failed.)",
    action: "联系我们 (Contact Us)",
    link: "https://vercel.com/contact",
  }}
/>
```

### 在表单字段中使用 (In a form field)

```tsx
<div className="flex flex-col gap-1.5">
  <label htmlFor="email">邮箱 (Email)</label>
  <input id="email" type="email" className="border-destructive ..." />
  <ErrorMessage>此邮箱地址已被使用。</ErrorMessage>
</div>
```

## 属性 (Props)

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | 行内错误文本（不传递 `error` 属性时使用） |
| `label` | `string` | — | 显示在消息前的可选标签前缀 |
| `size` | `"small" \| "medium" \| "large"` | `"medium"` | 字体和图标尺寸 |
| `error` | `ErrorObject` | — | 结构化错误对象 —— 包含 `message`, `action`, `link` |
| `className` | `string` | — | 额外的 CSS 类名 |

### `ErrorObject`

| 字段 (Field) | 类型 | 描述 |
|-------|------|-------------|
| `message` | `string` | 错误描述 |
| `action` | `string` | 操作链接的标签 |
| `link` | `string` | 操作链接的 href 地址 |

## 可访问性 (Accessibility)

- 渲染时带有 `role="alert"`，以便屏幕阅读器立即播报错误。
- 图标带有 `aria-hidden="true"` —— 文本内容提供了完整的上下文。
- 外部操作链接包含 `rel="noopener noreferrer"`。
