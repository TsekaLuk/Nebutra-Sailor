## 概览 (Overview)

`GithubInlineDiff` 渲染一个与 GitHub PR 评审界面完全一致的统一差异视图。特性包括：

- 块片段头 (`@@ -x,y +x,y @@`)
- 上下文行（灰色背景）
- 添加行（绿色背景，`+` 前缀）
- 删除行（红色背景，`-` 前缀）
- 行内评论线程（点击 `+` 按钮打开）
- 线程解决/取消解决切换
- 带缩写的当前用户头像

## 使用方法 (Usage)

```tsx
import { GithubInlineDiff, type DiffLine } from "@nebutra/ui/primitives";

const diff: DiffLine[] = [
  { kind: "hunk", content: "@@ -1,5 +1,6 @@" },
  { kind: "context", old: 1, new: 1, content: "import React from 'react';" },
  { kind: "del", old: 2, new: null, content: "const title = '旧标题 (Old Title)';" },
  { kind: "add", old: null, new: 2, content: "const title = '新标题 (New Title)';" },
  { kind: "add", old: null, new: 3, content: "const version = '2.0.0';" },
  { kind: "context", old: 3, new: 4, content: "" },
];

<GithubInlineDiff
  diff={diff}
  fileName="src/config.ts"
  fileStatus="已修改 (Modified)"
  currentUser={{ name: "Alice Chen", initials: "AC" }}
  onAddComment={(lineIndex, comment) => console.log("收到评论", comment)}
/>
```

## 属性 (Props)

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `diff` | `readonly DiffLine[]` | — | **必填。** 差异行数组 |
| `fileName` | `string` | — | **必填。** 头部显示的文件名 |
| `fileStatus` | `string` | — | 徽章文本（例如 "Modified", "Added"） |
| `initialComments` | `Record<number, DiffComment[]>` | — | 每一行下标对应的预存评论 |
| `onAddComment` | `(lineIndex: number, comment: DiffComment) => void` | — | 提交新评论时的回调 |
| `onResolveChange` | `(lineIndex: number, resolved: boolean) => void` | — | 当评论线程被标记为“已解决/未解决”时的回调 |
| `currentUser` | `{ name: string; initials: string }` | — | 表示当前发表回复的用户头像/姓名 |
| `className` | `string` | — | 容器 CSS 类名 |

## 类型定义 (Types)

### `DiffLine` (差异行)

```ts
type DiffLine =
  | { kind: "hunk"; content: string }
  | { kind: "context"; old: number | null; new: number | null; content: string }
  | { kind: "add"; old: null; new: number; content: string }
  | { kind: "del"; old: number; new: null; content: string };
```

### `DiffComment` (差异评论)

```ts
interface DiffComment {
  id: string;
  author: string;
  initials: string;
  body: string;
  createdAt: string;
}
```

## 注意事项 (Notes)

- 悬停在某一行上会显示 `+` 评论按钮（出现在 `context`, `add`, 和 `del` 类型的行左侧）。
- 评论线程存储在组件内部状态中；请使用 `onAddComment` 将其持久化到您的后端。
- `initialComments` 属性的键是差异数组（0-indexed）中的 **行下标**，而不是原始文件的行号。
