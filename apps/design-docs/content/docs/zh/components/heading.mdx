## 使用方法 (Usage)

```tsx
import { Heading } from "@nebutra/ui";

<Heading level={2}>章节标题 (Section Title)</Heading>
```

## 示例 (Examples)

### 标题层级 (Heading Levels)

```tsx
<Heading level={1}>标题 1 (Heading 1)</Heading>
<Heading level={2}>标题 2 (Heading 2)</Heading>
<Heading level={3}>标题 3 (Heading 3)</Heading>
<Heading level={4}>标题 4 (Heading 4)</Heading>
<Heading level={5}>标题 5 (Heading 5)</Heading>
<Heading level={6}>标题 6 (Heading 6)</Heading>
```

### 展示样式 (Display Style)

对 `level={1}` 和 `level={2}` 使用 `display` 属性，以获得 Hero 级别的超大标题样式（display-1 和 display-2）。

```tsx
<Heading level={1} display>
  Hero 头条标题
</Heading>

<Heading level={2} display>
  章节展示标题
</Heading>
```

### 使用 `as` 进行 SEO 覆盖 (SEO Override with `as`)

在 DOM 中渲染为 `h2` 以符合语义，但视觉上呈现 `h1` 的大小 —— 这对 SEO 友好的页面大纲非常有用。

```tsx
{/* 视觉上是 h1，语义上是 h2 */}
<Heading level={1} as="h2">
  精选文章 (Featured Article)
</Heading>
```

### 使用颜色 Token (With Color Tokens)

```tsx
<Heading level={2} color="muted">
  次级章节 (Secondary Section)
</Heading>

<Heading level={1} color="default" display>
  主要 Hero (Primary Hero)
</Heading>
```

### 居中对齐 (Centered Alignment)

```tsx
<Heading level={2} align="center" className="mb-4">
  居中的章节标题
</Heading>
```

## 属性 (Props)

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `2` | 标题层级 —— 同时控制 HTML 元素和视觉大小 |
| `as` | `"h1" \| "h2" \| "h3" \| "h4" \| "h5" \| "h6"` | — | 覆盖渲染的 HTML 元素（用于 SEO 目的） |
| `color` | `TextColor` | `"default"` | 来自排版 Token 系统的文本颜色 |
| `display` | `boolean` | `false` | 对 Hero 标题使用展示样式 (`display-1` 或 `display-2`) |
| `align` | `"left" \| "center" \| "right"` | — | 文本对齐方式 |
| `className` | `string` | — | 额外的 Tailwind 类名 |

## 注意事项 (Notes)

- 5 级和 6 级标题使用与 4 级相同的视觉样式 (`heading-4`) —— 区别仅在于语义。
- `display` 模式仅适用于 1 级和 2 级；更高级别将始终使用标准的标题样式。
- 使用 `as` 覆盖元素时，请确保生成的 HTML 标题层级结构对于屏幕阅读器仍然逻辑通顺。
