## 使用方法 (Usage)

```tsx
import {
  HoverCard,
  HoverCardTrigger,
  HoverCardContent,
} from "@nebutra/ui/primitives"
```

## 属性 (Props)

### HoverCard (根组件)

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `open` | `boolean` | — | 受控的打开状态 |
| `onOpenChange` | `(open: boolean) => void` | — | 打开状态改变时的回调函数 |
| `openDelay` | `number` | `700` | 打开前的延迟（毫秒） |
| `closeDelay` | `number` | `300` | 关闭前的延迟（毫秒） |

### HoverCardTrigger (触发器)

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | 将属性合并到直接子元素上 |
| `children` | `ReactNode` | — | 触发悬停卡片的元素 |

### HoverCardContent (内容区)

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"bottom"` | 首选显示侧 |
| `align` | `"start" \| "center" \| "end"` | `"center"` | 对齐方式 |
| `sideOffset` | `number` | `4` | 距离触发器的距离 |
| `className` | `string` | — | 额外的 CSS 类名 |
| `children` | `ReactNode` | — | 卡片内容 |

## 可访问性 (Accessibility)

- 基于 Radix UI HoverCard 构建。
- 悬停卡片的内容对于通过 Tab 键导航的屏幕阅读器用户不可见 — 它仅作为视觉补充。
- 对于键盘用户，请考虑使用 `Tooltip` 或 `Popover` 作为关键信息的替代方案。
- 一旦通过鼠标悬停打开，该卡片是可聚焦且可探索的。
