---
title: 无限滚动条 (InfiniteSlider)
description: 用于展示 Logo、证言和图片轮播的连续自动滚动内容容器。
---

## 概览 (Overview)

`InfiniteSlider` 通过内部复制子元素并持续对其进行平移运动，从而创建一个无缝的循环滚动效果。支持水平和垂直方向、可配置的滚动速度，并提供悬停减速的可选功能。

## 安装 (Installation)

```tsx
import { InfiniteSlider } from "@nebutra/ui/primitives";
```

## 使用方法 (Usage)

### Logo 展示条 (Logo strip)

```tsx
<InfiniteSlider gap={24} speed={50}>
  <img src="/logos/vercel.svg" alt="Vercel" className="h-8" />
  <img src="/logos/next.svg" alt="Next.js" className="h-8" />
  <img src="/logos/react.svg" alt="React" className="h-8" />
  <img src="/logos/typescript.svg" alt="TypeScript" className="h-8" />
</InfiniteSlider>
```

### 悬停减速 (Slow on hover)

```tsx
<InfiniteSlider gap={32} duration={20} durationOnHover={60}>
  {logos.map((logo) => <Logo key={logo.name} {...logo} />)}
</InfiniteSlider>
```

### 反向滚动 (Reverse direction)

```tsx
<InfiniteSlider gap={16} reverse>
  {items}
</InfiniteSlider>
```

### 垂直滚动 (Vertical scroll)

```tsx
<InfiniteSlider direction="vertical" gap={16} className="h-64">
  {testimonials.map((t) => <TestimonialCard key={t.id} {...t} />)}
</InfiniteSlider>
```

### 以 px/s 为单位设置速度 (Speed in px/s)

```tsx
<InfiniteSlider speed={80} speedOnHover={20} gap={24}>
  {items}
</InfiniteSlider>
```

## 属性 (Props)

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | **必需。** 需要滚动的内容（内部会自动复制） |
| `gap` | `number` | `16` | 项目之间的间距（像素） |
| `duration` | `number` | `25` | 动画循环一周所需的秒数 |
| `durationOnHover` | `number` | — | 鼠标悬停时的减速持续时间（秒） |
| `speed` | `number` | — | 滚动速度（单位：px/s），优先级高于 `duration` |
| `speedOnHover` | `number` | — | 悬停时的滚动速度（单位：px/s），优先级高于 `durationOnHover` |
| `direction` | `"horizontal" \| "vertical"` | `"horizontal"` | 滚动方向 |
| `reverse` | `boolean` | `false` | 是否反向滚动 |
| `className` | `string` | — | 额外的 CSS 类名，应用于溢出包装容器 |

## 注意事项 (Notes)

- 子元素在内部会被复制一次以实现无缝循环效果 —— 请避免依赖于固定数量的子元素。
- 当设置了 `durationOnHover` 或 `speedOnHover` 时，动画会在鼠标移入/移出时平滑地过渡到新的速度。
- 尺寸是通过 `react-use-measure` 进行测量的 —— 请确保容器在首次渲染前具有明确定义的尺寸。
- 需要将 `framer-motion` 和 `react-use-measure` 作为同级依赖（peer dependencies）。
