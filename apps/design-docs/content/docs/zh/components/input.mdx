## 前缀 (Prefix)

使用 `prefix` 属性在输入框左侧边缘渲染一个不可交互的图标或节点。点击前缀区域将穿透到输入框（输入框会获得焦点）。

<ComponentPreview className="max-w-[350px] flex-col gap-4">
  <Input5Demo />

  ```tsx
import { Search, Mail } from "lucide-react"

{/* Search icon */}
<Input prefix={<span className="text-xs">🔍</span>} placeholder="搜索… (Search…)" />

{/* Mail icon */}
<Input prefix={<span className="text-xs">✉️</span>} type="email" placeholder="contact@nebutra.com" />
  ```
</ComponentPreview>

## 后缀 (Suffix)

使用 `suffix` 属性在输入框右侧边缘渲染一个不可交互的标签或图标。

<Warning>
  `suffix` 具有 `pointer-events: none` 样式 — 它纯粹是装饰性的。对于有交互的右侧元素
  (例如密码显示按钮)，请通过 `className="pr-9"` 和 `absolute` 绝对定位按钮来手动构建。
  请参阅下面的 [密码显示](#password-reveal) 示例。
</Warning>

<ComponentPreview className="max-w-[350px] flex-col gap-4">
  <Input6Demo />

  ```tsx
{/* Domain hint */}
<Input suffix={<span className="text-xs">@nebutra.com</span>} placeholder="用户名 (username)" />

{/* Currency */}
<Input
  prefix={<span className="text-xs">$</span>}
  suffix={<span className="text-xs font-medium border-l pl-2">USD</span>}
  type="number"
  placeholder="0.00"
/>
```
</ComponentPreview>

## 可清除 (Clearable)

设置 `clearable` 属性后，无论输入框何时有值，它都会渲染一个 ✕ 按钮。当输入框为空时，该按钮会消失。

### 非受控用法 (Uncontrolled)

<ComponentPreview className="max-w-[350px]">
  <Input7Demo />

  ```tsx
<Input clearable defaultValue="contact@nebutra.com" placeholder="输入内容以查看 × 按钮… (Type to see the × button…)" />
```
</ComponentPreview>

### 受控用法 (Controlled)

<ComponentPreview className="max-w-[350px]">
  <Input clearable defaultValue="contact@nebutra.com" placeholder="受控的可清除输入框… (Controlled clearable…)" />

  ```tsx
const [value, setValue] = useState("")

<Input
  value={value}
  onChange={(e) => setValue(e.target.value)}
  clearable
  onClear={() => setValue("")}
  placeholder="受控的可清除输入框… (Controlled clearable…)"
/>
  ```
</ComponentPreview>

### 带有前缀 (With Prefix)

<ComponentPreview className="max-w-[350px]">
  <Input
    prefix={<Search className="h-4 w-4" />}
    clearable
    placeholder="搜索… (Search…)"
  />

  ```tsx
<Input
  prefix={<Search className="h-4 w-4" />}
  clearable
  placeholder="搜索… (Search…)"
/>
  ```
</ComponentPreview>

<Callout>
  当同时设置 `clearable` 和 `suffix` 时，只要字段有值，✕ 按钮就会替换后缀。
  后缀在字段被清除时会重新出现 — 这对于键盘快捷键提示或单位标签非常有用。
</Callout>

## 密码显示 (Password Reveal)

对于具有交互的右侧控件，请使用 `className="pr-9"` 并绝对定位 (`absolute`) 配置该按钮：

<ComponentPreview className="max-w-[350px]">
  <Input3Demo />

  ```tsx
const [shown, setShown] = useState(false)
const Icon = shown ? EyeOff : Eye

<div className="relative">
  <Input type={shown ? "text" : "password"} className="pr-9" placeholder="密码… (Password…)" />
  <button
    type="button"
    aria-label={shown ? "隐藏密码 (Hide password)" : "显示密码 (Show password)"}
    onClick={() => setShown((v) => !v)}
    className="absolute right-2 top-1/2 -translate-y-1/2 flex h-5 w-5 items-center justify-center
               rounded text-muted-foreground hover:text-foreground transition-colors
               focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring"
  >
    <Icon className="h-3.5 w-3.5" />
  </button>
</div>
  ```
</ComponentPreview>

## 属性 (Props)

`Input` 扩展并集成了所有的原生 `React.InputHTMLAttributes<HTMLInputElement>` 属性。

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `type` | `string` | `"text"` | HTML 输入类型 |
| `placeholder` | `string` | — | 占位符文本 |
| `disabled` | `boolean` | `false` | 禁用输入框并隐藏清除按钮 |
| `className` | `string` | — | 应用在 `<input>` 元素上的样式 |
| `prefix` | `ReactNode` | — | 左侧边缘内的不可交互装饰 |
| `suffix` | `ReactNode` | — | 右侧边缘内的不可交互装饰 |
| `clearable` | `boolean` | `false` | 字段有值时显示 ✕ 按钮 |
| `onClear` | `() => void` | — | 单击 ✕ 按钮时调用 |
| `wrapperClassName` | `string` | — | 应用在外部 `<div>` 包裹元素上的样式 |
| `value` | `string \| number` | — | 受控值 |
| `defaultValue` | `string \| number` | — | 非受控的默认值 |
| `onChange` | `ChangeEventHandler` | — | 变更处理函数 |
| `ref` | `Ref<HTMLInputElement>` | — | 转发的 ref — 始终指向 `<input>` |

## 设计令牌 (Design Tokens)

输入框高度被锁定在 `h-10` (40px)（由 `inputTokens.height` 控制）。
聚焦环：`--focus-ring` — 品牌蓝 2px 聚焦环，带有 2px 的偏移。
前缀/后缀定位在 `left-3` / `right-3` (即内边距为 12px)。

---

## 可访问性 (Accessibility)

- 渲染原生 `<input>` — 请始终将其与可见的 `<label>` 或 `aria-label` 配对。
- 使用 `aria-invalid="true"` 结合 `aria-describedby` 来关联错误提示消息。
- ✕ 清除按钮具有 `aria-label="Clear"` — 屏幕阅读器可正确朗读它。
- 前缀和后缀的内容跨度带有 `aria-hidden` 属性 — 它们是装饰性的并且会被辅助技术忽略。
- 聚焦环对键盘导航始终可见。
