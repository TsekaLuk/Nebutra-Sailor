## 概览 (Overview)

`Terminal` 是由三个组件构成的集合，用于展示命令行风格的内容：

- **`Terminal`** —— 带有红绿黄交通灯按钮的 macOS 风格外框。
- **`TypingAnimation`** —— 实现逐字输入的打字机效果。
- **`AnimatedSpan`** —— 实现整行文本的一次性淡入效果。

默认情况下，子元素会按顺序（in sequence）执行：前一个元素完成后，下一个元素才会开始执行。支持通过 `startOnView` 实现进入视口后触发。

## 使用方法 (Usage)

```tsx
import { Terminal, TypingAnimation, AnimatedSpan } from "@nebutra/ui/primitives";

<Terminal>
  <TypingAnimation>pnpm dlx shadcn@latest init</TypingAnimation>
  <AnimatedSpan className="text-green-400">✔ 预检通过 (Preflight checks).</AnimatedSpan>
  <AnimatedSpan className="text-green-400">✔ 正在验证 Tailwind CSS.</AnimatedSpan>
  <TypingAnimation>成功！项目已初始化。(Success! Project initialized.)</TypingAnimation>
</Terminal>
```

### 禁用自动排序 (Without sequencing)

```tsx
<Terminal sequence={false}>
  <AnimatedSpan delay={0}>行 1 (立即显示)</AnimatedSpan>
  <AnimatedSpan delay={500}>行 2 (500ms 后显示)</AnimatedSpan>
  <AnimatedSpan delay={1000}>行 3 (1s 后显示)</AnimatedSpan>
</Terminal>
```

## 属性 (Props)

### `Terminal`

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | `TypingAnimation` 和 `AnimatedSpan` 子组件 |
| `sequence` | `boolean` | `true` | 是否启用自动排序执行模式 |
| `startOnView` | `boolean` | `true` | 终端进入视口时开始动画 |
| `className` | `string` | — | 额外的 CSS 类名 |

### `TypingAnimation`

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `string` | — | 要打字的文本（必须为字符串） |
| `duration` | `number` | `60` | 每个字符输入的毫秒数 |
| `delay` | `number` | `0` | 开始前的延迟（仅非排序模式生效） |
| `as` | `React.ElementType` | `"span"` | 渲染的 HTML 元素或组件 |
| `startOnView` | `boolean` | `true` | 进入视口时开始动画（仅非排序模式生效） |

### `AnimatedSpan`

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | 要淡入展示的内容 |
| `delay` | `number` | `0` | 出现前的延迟（仅非排序模式生效） |
| `startOnView` | `boolean` | `false` | 进入视口时开始动画（仅非排序模式生效） |
| `className` | `string` | — | 额外的 CSS 类名 |

## 注意事项 (Notes)

- `TypingAnimation` 的 `children` 必须是纯 `string` —— 如果传入 JSX 会抛出错误。
- 在排序（sequenced）模式下，单个子项的 `delay` 属性会被忽略，定时由前一个子项的完成状态驱动。
- 终端容器默认设置了 `max-h-[400px]` —— 可以通过 `className` 覆盖。
