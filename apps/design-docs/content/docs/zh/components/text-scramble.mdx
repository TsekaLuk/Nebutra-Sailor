---
title: "文本乱码 (Text Scramble)"
description: "字符在解析为最终文本之前会随机变换符号。可在挂载、悬停或任何事件中触发。"
---

## 使用方法 (Usage)

```tsx
import { TextScramble } from "@nebutra/ui/primitives"
```

## 示例 (Examples)

### 挂载时自动播放 (Auto-play on Mount)

```tsx
<TextScramble className="font-mono text-sm uppercase">
  Hello World
</TextScramble>
```

### 悬停触发 (Hover Trigger)

```tsx
function HoverScramble() {
  const [trigger, setTrigger] = useState(false);

  return (
    <TextScramble
      as="span"
      trigger={trigger}
      onScrambleComplete={() => setTrigger(false)}
      onHoverStart={() => setTrigger(true)}
      className="font-mono cursor-pointer"
    >
      请将鼠标悬停此处 (Hover me)
    </TextScramble>
  );
}
```

### 自定义字符集 (Custom Character Set)

```tsx
<TextScramble characterSet="░▒▓█▀▄▌▐" duration={1.5}>
  赛博模式 (Cyber Mode)
</TextScramble>
```

### 慢动作揭示 (Slow Motion)

```tsx
<TextScramble duration={2} speed={0.08} className="text-2xl">
  缓慢揭示 (Slow Reveal)
</TextScramble>
```

## 属性 (Props)

`TextScramble` 继承了 Framer Motion 的 `MotionProps`。

| 属性 | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `children` | `string` | — | 最终要解析出的文本内容 |
| `duration` | `number` | `0.8` | 乱码持续的总时间（秒） |
| `speed` | `number` | `0.04` | 字符更新的间隔时间（秒） |
| `characterSet` | `string` | 字母数字 | 乱码阶段使用的字符集 |
| `as` | `React.ElementType` | `"p"` | 渲染为指定的 HTML 元素 |
| `trigger` | `boolean` | `true` | 设置为 `false` 以暂停；切换回 `true` 以重新播放 |
| `onScrambleComplete` | `() => void` | — | 动画完成时的回调函数 |
| `className` | `string` | — | 额外的 CSS 类名 |
| `...motionProps` | `MotionProps` | — | 所有 Framer Motion 属性 |

## 注意事项 (Notes)

- 源文本中的空格会被保留，永不参与乱码变换。
- 先将 `trigger` 设置为 `false` 再设为 `true` 可以重头播放动画。
