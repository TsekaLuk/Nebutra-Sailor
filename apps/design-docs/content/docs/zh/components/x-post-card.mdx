## 概览 (Overview)

`XPostCard` 和 `ClientXPostCard` 使用 `react-tweet` 库嵌入 X/Twitter 帖子。`XPostCard` 将客户端组件包裹在 Suspense 边界内，是绝大多数场景下的首选方案。在帖子加载过程中，会显示内置的 `XPostSkeleton` 骨架图。

## 安装 (Installation)

```tsx
import {
  XPostCard,
  ClientXPostCard,
  XPostSkeleton,
} from "@nebutra/ui/primitives";
```

`react-tweet` 是一个同僚依赖（peer dependency）—— 如果尚未安装，请执行：

```bash
pnpm add react-tweet
```

## 使用方法 (Usage)

### 基础用法（推荐） (Basic)

```tsx
<XPostCard id="1441032681968212480" />
```

### 带有自定义加载 fallback (With custom loading fallback)

```tsx
<XPostCard
  id="1441032681968212480"
  fallback={<div className="animate-pulse h-48 rounded-lg bg-muted" />}
/>
```

### 纯客户端组件 (Client-only component)

当您需要在已经自行管理 Suspense 边界的客户端组件中嵌入时，请使用 `ClientXPostCard`：

```tsx
"use client";

import { ClientXPostCard } from "@nebutra/ui/primitives";

export default function TweetSection() {
  return <ClientXPostCard id="1441032681968212480" />;
}
```

### 自定义 API URL (Custom API URL)

覆盖推文获取接口（例如用于缓存或代理）：

```tsx
<XPostCard
  id="1441032681968212480"
  apiUrl="/api/tweet"
/>
```

### 错误处理 (Error handling)

```tsx
<XPostCard
  id="1441032681968212480"
  onError={(error) => {
    console.error("加载推文失败:", error);
    return error;
  }}
/>
```

### 独立骨架图 (Skeleton standalone)

```tsx
<XPostSkeleton className="max-w-md" />
```

## 属性 (Props)

### XPostCard / ClientXPostCard

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `id` | `string` | — | **必需。** X/Twitter 帖子 ID |
| `className` | `string` | — | 卡片容器的额外 CSS 类名 |
| `apiUrl` | `string` | — | 用于获取推文数据的自定义 API URL |
| `fetchOptions` | `RequestInit` | — | 传递给推文获取请求的自定义选项 |
| `fallback` | `ReactNode` | `<XPostSkeleton />` | 加载时显示的内容 |
| `components` | `TwitterComponents` | — | 用于 react-tweet 内部的自定义组件覆盖 |
| `onError` | `(error: Error \| undefined) => Error \| undefined` | — | 错误处理回调函数 |

### XPostSkeleton

| 属性 (Prop) | 类型 | 默认值 | 描述 |
|------|------|---------|-------------|
| `className` | `string` | — | 额外的 CSS 类名 |

## 注意事项 (Notes)

- 该卡片使用了设计系统令牌（`bg-card`、`text-card-foreground`、`border`），因此会自动匹配您的主题配色。
- `ClientXPostCard` 是实际执行渲染的组件 —— `XPostCard` 仅负责将其包裹在 Suspense 中。
- 请在服务端组件（Server Components）或任何需要 Suspense 边界的地方使用 `XPostCard`。
- 如果未提供 `onError`，则会显示 `react-tweet` 默认的 `TweetNotFound`（推文未找到）组件。
