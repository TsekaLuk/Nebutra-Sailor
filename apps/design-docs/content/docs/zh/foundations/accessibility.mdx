## Overview

Accessibility primitives live in `apps/docs-hub/design-system/src/primitives/accessibility.ts`. The file exports style objects, ARIA attribute helpers, and utility functions that implement WCAG 2.1 AA requirements. These are building blocks — components in the design system use them internally, and application code can import them directly when building custom patterns.

Exported members:

| Export | Type | Purpose |
|---|---|---|
| `visuallyHidden` | `StyleObject` | Hide visually, preserve for screen readers |
| `focusRing` | `StyleObject` | Keyboard focus indicator styles |
| `skipLinkStyle` | `StyleObject` | Skip-to-content link (shown on focus) |
| `ariaPatterns` | Object | Reusable ARIA attribute sets |
| `minTouchTarget` | Object | WCAG 2.5.5 minimum touch target |
| `prefersReducedMotion()` | Function | Runtime motion preference check |
| `contrastRequirements` | Object | WCAG contrast ratio thresholds |

## Design Rationale

Accessibility cannot be an afterthought bolted on after a component ships. These primitives exist to make correct behavior the path of least resistance — it should be easier to use `visuallyHidden` than to guess the six CSS properties required to hide something correctly, and easier to call `ariaPatterns.loading` than to remember `aria-busy` and `aria-live` separately.

<Warning>
These primitives establish the baseline. They do not replace semantic HTML. Use the correct native element (`<button>`, `<nav>`, `<main>`, `<label>`) before reaching for ARIA attributes.
</Warning>

## `visuallyHidden`

Removes an element from the visual layout while keeping it in the accessibility tree.

```ts
import { visuallyHidden } from "@nebutra/ui/primitives/accessibility";

// Style object — spread into inline styles or CSS-in-JS
const style = visuallyHidden;
// {
//   position: "absolute",
//   width: "1px",
//   height: "1px",
//   padding: 0,
//   margin: "-1px",
//   overflow: "hidden",
//   clip: "rect(0, 0, 0, 0)",
//   whiteSpace: "nowrap",
//   border: 0,
// }
```

Use for: labels on icon buttons, screen-reader-only announcements, visually redundant text.

Do not use `display: none` or `visibility: hidden` to hide content that screen readers should announce.

## `focusRing`

Standard keyboard focus ring that meets WCAG 2.4.7 (Focus Visible).

```ts
import { focusRing } from "@nebutra/ui/primitives/accessibility";

// focusRing resolves to:
// {
//   outline: "2px solid",
//   outlineColor: "accent.fg",
//   outlineOffset: "2px",
// }
```

Apply to any interactive element that receives keyboard focus. The design system components apply this automatically via `&:focus-visible` selectors.

<Tip>
Use `:focus-visible` (not `:focus`) to show the ring only for keyboard users. Mouse clicks should not trigger the focus ring on most interactive elements.
</Tip>

## `skipLinkStyle`

A skip navigation link visible only on keyboard focus. Required for WCAG 2.4.1 (Bypass Blocks).

```tsx
import { skipLinkStyle } from "@nebutra/ui/primitives/accessibility";

// In your root layout
<a href="#main-content" style={skipLinkStyle}>
  Skip to main content
</a>
```

The link is visually hidden at rest and becomes visible (fixed to top-left, styled with `focusRing`) when it receives keyboard focus.

## `ariaPatterns`

Pre-composed ARIA attribute objects for common UI states.

### `ariaPatterns.loading`

```tsx
import { ariaPatterns } from "@nebutra/ui/primitives/accessibility";

<div {...ariaPatterns.loading}>
  Loading content...
</div>
// Applies: aria-busy="true", aria-live="polite"
```

### `ariaPatterns.expandable(expanded)`

```tsx
<button {...ariaPatterns.expandable(isOpen)}>
  Toggle section
</button>
// Applies: aria-expanded={isOpen}
```

### `ariaPatterns.currentPage`

```tsx
<a href="/dashboard" {...ariaPatterns.currentPage}>
  Dashboard
</a>
// Applies: aria-current="page"
```

### `ariaPatterns.required`

```tsx
<input {...ariaPatterns.required} />
// Applies: aria-required="true"
```

### `ariaPatterns.invalid(message?)`

```tsx
<input
  {...ariaPatterns.invalid("Email is required")}
  id="email"
/>
// Applies: aria-invalid="true", aria-errormessage="Email is required"
```

## `minTouchTarget`

WCAG 2.5.5 (Target Size) requires a minimum 44×44px interactive area.

```ts
import { minTouchTarget } from "@nebutra/ui/primitives/accessibility";

// minTouchTarget = { minWidth: 44, minHeight: 44 }

<button style={minTouchTarget}>
  <IconClose size={16} />
</button>
```

Apply to any interactive element whose visible area is smaller than 44×44px — icon buttons, close icons, toggle switches.

## `prefersReducedMotion()`

Runtime check for the OS-level reduce-motion preference.

```ts
import { prefersReducedMotion } from "@nebutra/ui/primitives/accessibility";

const duration = prefersReducedMotion() ? 0 : 300;

element.animate([{ opacity: 0 }, { opacity: 1 }], {
  duration,
  easing: "ease-out",
});
```

Returns `false` on the server (SSR-safe). Use in combination with CSS `@media (prefers-reduced-motion: reduce)` for complete coverage.

## `contrastRequirements`

WCAG contrast ratio thresholds for programmatic contrast checking.

| Token | Value | Standard | Context |
|---|---|---|---|
| `contrastRequirements.normalText` | 4.5 | AA | Body text (< 18px or < 14px bold) |
| `contrastRequirements.largeText` | 3 | AA | Large text (18px+ or 14px+ bold) |
| `contrastRequirements.normalTextAAA` | 7 | AAA | Enhanced normal text |
| `contrastRequirements.largeTextAAA` | 4.5 | AAA | Enhanced large text |

```ts
import { contrastRequirements } from "@nebutra/ui/primitives/accessibility";

function isAccessible(contrastRatio: number, isLargeText: boolean): boolean {
  const required = isLargeText
    ? contrastRequirements.largeText
    : contrastRequirements.normalText;
  return contrastRatio ≥ required;
}
```

The design system's semantic color tokens are tested to meet AA contrast requirements in both light and dark modes.

## Usage Guidelines

### Keyboard navigation

Every interactive element must be reachable and operable by keyboard. Never suppress default focus behavior without providing an equivalent custom implementation.

```tsx
// Wrong — removes focus without replacement
<div onClick={handleClick} tabIndex={-1} style={{ outline: "none" }}>

// Correct — uses semantic button, focus ring applies automatically
<button onClick={handleClick}>
```

### Motion

Wrap all animations in a reduced-motion check:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

The marketing `marketingTransitions` values should be replaced with instant transitions when `prefersReducedMotion()` returns `true`.

## Do / Don't

**Do** use semantic HTML elements as the primary accessibility mechanism. ARIA patterns supplement; they do not replace native semantics.

**Do** apply `visuallyHidden` to provide context for icon-only controls (labels, descriptions, announcements).

**Do** apply `minTouchTarget` to all interactive elements with a visual footprint smaller than 44×44px.

**Do** verify contrast ratios using `contrastRequirements` thresholds when introducing new color combinations.

**Don't** use `aria-label` on non-interactive elements. If an element does not receive focus or interaction, it does not need an `aria-label`.

**Don't** suppress the default focus ring with `outline: none` or `outline: 0` unless `focusRing` is applied via `:focus-visible`.

**Don't** trigger animations when `prefersReducedMotion()` returns `true`.
