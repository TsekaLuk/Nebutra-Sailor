## Prohibited Uses

<Warning>
  The following color substitutions are strictly forbidden:
</Warning>

| Prohibited | Reason |
|-----------|--------|
| Tailwind `indigo-*` 或 `blue-*` 作为品牌蓝 | 严禁直接套用，请务必使用全局指定的原生命名 CSS 变量。 |
| Tailwind `teal-500` for brand cyan | Teal is not Nebutra cyan |
| Tailwind `gray-*` for neutrals | Nebutra neutrals have a blue undertone |
| Orange or red as primary action color | Only brand blue is used for primary actions |
| Cyan as text on light backgrounds | Fails WCAG contrast — use only on dark backgrounds |
| Brand blue as a flat background fill | Use the brand gradient for hero/CTA surfaces |
| Any non-brand color for `info` semantic | Info must use brand blue `#6366f1`, not sky/teal |

---

## 不要 （Don't）

不要将 Tailwind 的默认 `bg-secondary` 或变量 `--secondary` 用作组件的弱化背景轨道（例如进度条 Progress、滑块 Sliders 等组件的底轨）。因为在云毓规范中，`--secondary` 被明确映射为了高饱和度的云毓青 `#0BF1C3`。若直接套用，会导致背景轨道呈现出难以直视的赛博亮色，且无法满足无障碍对比度要求。如果需要弱化轨道颜色，请务必使用带透明度的品牌主色（如 `bg-primary/20`）。

## Do / Don't

**Do** 始终使用全局变量（如 `bg-primary`、`text-primary` 或自定义色名 `blue-500` 即 `#6366f1`）应用于所有的主交互（如按钮、链接、输入框高光等）。

**Do** use the full 11-step scale for subtle tints, hover states, and dark mode adaptations.

**Do** 将 `blue-500` 与 `cyan-500` 结合起来组构为品牌的专属高定渐变色（Brand Signature）。

**Do** use `neutral-950` as the dark mode page background.

**Don't** 严禁直接在组件代码里写明 `bg-indigo-500` 或是 `bg-blue-500` 等 Tailwind 默认调色盘色系。一切色彩映射都要回源到 `bg-primary` 或者全局的语义变量上，以确保支持一键更替（比如之后出黑夜/暗网特型）。

**Don't** use cyan on white — it fails WCAG contrast.

**Don't** approximate brand colors with Tailwind defaults (`blue-500`, `emerald-400`).

**Don't** introduce new color scales outside the defined blue, cyan, neutral, and semantic palettes.

**Don't** use brand blue as a solid fill where the brand gradient is expected (hero sections, primary CTAs).

**Don't** 不要将 Tailwind 的默认 `bg-secondary` 或变量 `--secondary` 用作组件的弱化背景轨道（例如进度条 Progress、滑块 Sliders 等）。在云毓主题中，`--secondary` 被映射为高饱和度的青色 `#80ffe0`。若直接套用，会导致背景轨道呈现出难以直视的赛博亮色，且大幅破坏无障碍阅读。请务必使用带透明度的品牌主色（如 `bg-primary/20`）作为任何组件的底层轨道。
