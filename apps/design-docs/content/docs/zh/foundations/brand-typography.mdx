## Type Hierarchy

Nebutra uses a four-tier type hierarchy. Each tier has a specific role and must not be used outside its designated context.

## Geist Sans — Display Typeface

Geist Sans is the display typeface for all high-impact text. It is a variable font supporting weights 100 through 900, enabling precise optical sizing.

### When to Use

- Hero headlines and page titles
- Section headings on marketing/landing pages
- Large numeric displays (metrics, dashboards, pricing)
- Brand-forward UI labels

### Weights

| Weight | Value | Use Case |
|--------|-------|----------|
| Thin | 100 | Oversized decorative display text |
| ExtraLight | 200 | Large ambient headlines |
| Light | 300 | Subtitle/tagline at hero scale |
| Regular | 400 | Large body on marketing pages |
| Medium | 500 | Section titles, card headings |
| SemiBold | 600 | Primary headings (H2, H3) |
| Bold | 700 | Hero headlines (H1) |
| ExtraBold | 800 | Extra emphasis on hero text |
| Black | 900 | Maximum impact display text |

### Code

```css
font-family: "Geist", sans-serif;
font-weight: 700;
font-variation-settings: "wght" 700;
```

```tsx
<h1 className="font-display font-bold text-heading-72">
  Enterprise SaaS Platform
</h1>
```

## Poppins — English Body Typeface

Poppins is the primary body typeface for all English text across the platform. Its geometric construction pairs cleanly with Geist Sans.

### When to Use

- Body copy and paragraphs
- Navigation labels and menu items
- Button text and form labels
- Descriptions, captions, and metadata
- All general-purpose English UI text

### Weights

| Weight | Value | Use Case |
|--------|-------|----------|
| Regular | 400 | Body copy, descriptions, paragraph text |
| Medium | 500 | Labels, navigation items, captions, secondary headings |
| SemiBold | 600 | Subheadings, button labels, emphasis text |
| Bold | 700 | Main headings in app UI (not hero), strong emphasis |

<Warning>
  Do not use Poppins for hero/display text. Use Geist Sans for headlines and high-impact text.
  Do not use weights below 400 or above 700 for Poppins — only 400, 500, 600, and 700 are brand-approved.
</Warning>

### Code

```css
font-family: "Poppins", sans-serif;
font-weight: 400;
```

```tsx
<p className="font-sans font-normal text-copy-16">
  Nebutra transforms raw data into intelligent cloud-native outcomes.
</p>

<span className="font-sans font-medium text-copy-14">
  Navigation Label
</span>
```

## vivo Sans — Chinese Body Typeface

vivo Sans is the primary typeface for all Chinese (CJK) UI text. It provides excellent screen rendering at small sizes and harmonizes with Poppins for mixed-language interfaces.

### When to Use

- All Chinese body text and UI labels
- Mixed Latin + Chinese content (apply via `font-cn` class)
- Chinese marketing copy
- CJK data labels and descriptions

### Weights

| Weight | Value | Use Case |
|--------|-------|----------|
| Light | 300 | Subtle decorative Chinese text |
| Regular | 400 | Chinese body copy |
| Medium | 500 | Chinese labels, navigation |
| DemiBold | 600 | Chinese subheadings |
| Bold | 700 | Chinese headings |

### Code

```css
font-family: "vivo Sans", "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
font-weight: 400;
```

```tsx
<p className="font-cn font-normal text-copy-16">
  云端聚合 · 引领未来
</p>
```

---

## JetBrains Mono — Code Typeface

JetBrains Mono is the monospace typeface for all code, terminal output, and technical data displays. Fira Code serves as the fallback.

### When to Use

- Code blocks and inline code
- Terminal/CLI output
- API endpoint displays
- Technical data values (IDs, hashes, tokens)
- Configuration examples

### Code

```css
font-family: "JetBrains Mono", "Fira Code", ui-monospace, Consolas, monospace;
font-weight: 400;
```

```tsx
<code className="font-mono text-copy-14">
  const apiKey = process.env.NEBUTRA_API_KEY
</code>
```

## Font Stacks

### CSS Custom Properties

```css
:root {
  /* Primary — display and hero text */
  --font-display: "Geist", "Poppins", "vivo Sans", "PingFang SC",
                   "Microsoft YaHei", "Noto Sans SC", sans-serif;

  /* Body — English UI text */
  --font-sans: "Poppins", "vivo Sans", "PingFang SC",
               "Microsoft YaHei", "Noto Sans SC", sans-serif;

  /* Body — Chinese UI text */
  --font-cn: "vivo Sans", "PingFang SC", "Microsoft YaHei",
             "Noto Sans SC", sans-serif;

  /* Code — monospace */
  --font-mono: "JetBrains Mono", "Fira Code", ui-monospace,
               Consolas, "Liberation Mono", "Courier New", monospace;

  /* Universal stack (global fallback) */
  --font-stack: "Geist", "Poppins", "vivo Sans", "PingFang SC",
                "Microsoft YaHei", "Noto Sans SC", sans-serif;
}
```

### Tailwind Utility Classes

```tsx
// Display / hero headlines
<h1 className="font-display">Enterprise SaaS Platform</h1>

// English body text
<p className="font-sans">Body copy in English</p>

// Chinese body text
<p className="font-cn">中文正文内容</p>

// Code / monospace
<code className="font-mono">const x = 1</code>
```

## CJK Rhythm

Mixed Latin + Chinese content requires special handling to maintain visual rhythm.

### The `font-cn` Class

Apply `font-cn` to any container that displays Chinese text. This switches the font stack to vivo Sans and adjusts the following:

- **Line height** increases from 1.5 to 1.75 (CJK characters need more vertical space)
- **Letter spacing** resets to 0 (CJK does not use letter-spacing)
- **Word spacing** is managed automatically by the browser for CJK

```tsx
// Pure Chinese content
<p className="font-cn text-copy-16">
  无锡云毓智能科技有限公司致力于企业级SaaS平台建设
</p>

// Mixed Latin + Chinese
<div className="font-cn">
  <p>Nebutra (云毓智能) 是一个企业级 SaaS 平台</p>
</div>
```

### Size Adjustments

CJK glyphs are optically larger than Latin characters at the same font size. Apply a slight reduction when mixing:

```css
/* Mixed content adjustment */
.font-cn {
  font-family: var(--font-cn);
  line-height: 1.75;
  font-size-adjust: 0.52;
}
```

### Punctuation

- Use Chinese punctuation (。，！？；：""''） in Chinese sentences
- Use English punctuation in English sentences
- In mixed content, follow the language of the surrounding clause

## Do / Don't

**Do** use Geist Sans for all hero and display text above 24px.

**Do** use Poppins for all English UI body text.

**Do** use vivo Sans (via `font-cn`) for all Chinese text.

**Do** apply `font-cn` to containers with CJK content to activate proper line-height.

**Do** use JetBrains Mono for all code and technical data.

**Don't** use Poppins for hero headlines — use Geist Sans.

**Don't** use Geist Sans for body copy — use Poppins.

**Don't** use non-brand fonts (Inter, Roboto, Arial, SF Pro) in Nebutra products.

**Don't** use Poppins weights outside 400-700.

**Don't** set letter-spacing on body text — tracking adjustments belong only to headings, buttons, and overlines.

**Don't** mix `font-sans` and `font-cn` on the same element — choose one based on the content language.
