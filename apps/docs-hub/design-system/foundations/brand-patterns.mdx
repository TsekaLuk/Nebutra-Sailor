## Visual DNA Overview

Five visual elements form the Nebutra design DNA:

| Element | Source | Role |
|---------|--------|------|
| **Hexagonal Grid** | Logo geometry | Primary structural pattern |
| **Blue-to-Cyan Gradient** | Brand colors | Color signature (see [Gradients](/design-system/foundations/brand-gradients)) |
| **Grain/Noise Textures** | Analog warmth | Depth and tactility (see [Gradients](/design-system/foundations/brand-gradients)) |
| **Glassmorphism** | Dark-first surfaces | Layered transparency |
| **Dark-First Design** | Brand philosophy | Default surface treatment |

## Hexagonal Grid

The hexagonal grid is the core visual DNA — extracted directly from the logo's geometric construction. Use for hero backgrounds, feature sections, and ambient decoration.

### HexGrid Component

```tsx
import { HexGrid } from "@nebutra/custom-ui"

// Default — subtle background pattern
<HexGrid
  size={60}
  gap={4}
  strokeColor="rgba(0, 51, 254, 0.08)"
  className="absolute inset-0"
/>

// Accent — gradient-highlighted cells
<HexGrid
  size={48}
  gap={3}
  strokeColor="rgba(0, 51, 254, 0.12)"
  fillColor="rgba(11, 241, 195, 0.03)"
  highlightCells={[5, 12, 19, 26]}
  highlightColor="rgba(0, 51, 254, 0.15)"
  className="absolute inset-0"
/>

// Animated — cells fade in on scroll
<HexGrid
  size={60}
  gap={4}
  strokeColor="rgba(0, 51, 254, 0.1)"
  animated
  staggerDelay={0.02}
  className="absolute inset-0"
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `number` | `60` | Hexagon cell size in pixels |
| `gap` | `number` | `4` | Gap between cells |
| `strokeColor` | `string` | `rgba(0,51,254,0.08)` | Hex cell border color |
| `fillColor` | `string` | `transparent` | Hex cell fill color |
| `highlightCells` | `number[]` | `[]` | Indices of cells to highlight |
| `highlightColor` | `string` | `rgba(0,51,254,0.15)` | Color for highlighted cells |
| `animated` | `boolean` | `false` | Enable entrance animation |
| `staggerDelay` | `number` | `0.02` | Delay between cell animations |

### Usage Guidelines

- Keep stroke opacity between 5-15% for subtle backgrounds
- Highlight no more than 10% of cells to avoid visual clutter
- Use brand blue (`#0033FE`) for strokes and brand cyan (`#0BF1C3`) for highlights
- Always place behind content with `absolute inset-0` and low `z-index`

## Dot Pattern

A minimal dot grid for subtle, low-key backgrounds. Use where the hexagonal grid is too prominent.

```tsx
// CSS implementation
<div
  className="absolute inset-0"
  style={{
    backgroundImage: 'radial-gradient(circle, rgba(0, 51, 254, 0.12) 1px, transparent 1px)',
    backgroundSize: '24px 24px',
  }}
/>
```

```tsx
// Tailwind implementation
<div className="absolute inset-0 bg-[radial-gradient(circle,rgba(0,51,254,0.12)_1px,transparent_1px)] bg-[size:24px_24px]" />
```

### When to Use

- Dashboard backgrounds
- Empty states
- Settings pages
- Secondary content areas where hexgrid is too visually heavy

## Flickering Grid

A dynamic grid pattern where cells randomly flicker between visible and hidden states. Use for tech-forward sections that need energy and dynamism.

```tsx
import { FlickeringGrid } from "@nebutra/custom-ui"

<section className="relative h-[500px]">
  <FlickeringGrid
    cellSize={16}
    color="#0033FE"
    maxOpacity={0.15}
    flickerChance={0.3}
    className="absolute inset-0"
  />
  <div className="relative z-10">
    {/* Content */}
  </div>
</section>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `cellSize` | `number` | `16` | Grid cell size in pixels |
| `color` | `string` | `#0033FE` | Cell color |
| `maxOpacity` | `number` | `0.15` | Maximum cell opacity |
| `flickerChance` | `number` | `0.3` | Probability of a cell flickering per frame |

### When to Use

- Real-time data processing sections
- API/infrastructure feature pages
- Technical capability showcases

## Warp Background

A WebGL distortion effect that creates a space-warp visual. Use for immersive, full-bleed hero sections.

```tsx
import { WarpBackground } from "@nebutra/custom-ui"

<section className="relative min-h-screen">
  <WarpBackground
    speed={0.005}
    intensity={0.3}
    colorA="#0033FE"
    colorB="#0BF1C3"
    className="absolute inset-0"
  />
  <div className="relative z-10">
    {/* Hero content */}
  </div>
</section>
```

<Warning>
  WarpBackground uses WebGL. Limit to 1 per page and provide a CSS gradient fallback.
  See [Gradients & Textures](/design-system/foundations/brand-gradients) performance notes.
</Warning>

## Stars Canvas

A canvas-based starfield for space and cloud-themed sections. Reinforces the "cloud intelligence" brand narrative.

```tsx
import { StarsCanvas } from "@nebutra/custom-ui"

<section className="relative h-[600px] bg-neutral-950">
  <StarsCanvas
    count={200}
    speed={0.5}
    size={1.5}
    color="#ffffff"
    className="absolute inset-0"
  />
  <div className="relative z-10 text-white">
    {/* Cloud/space themed content */}
  </div>
</section>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `count` | `number` | `200` | Number of stars |
| `speed` | `number` | `0.5` | Drift speed |
| `size` | `number` | `1.5` | Maximum star radius in pixels |
| `color` | `string` | `#ffffff` | Star color |

### When to Use

- Cloud infrastructure pages
- Pricing/tier comparison sections
- About/mission pages
- Any section with a space/cloud metaphor

## Glassmorphism Recipe

Glassmorphism is a core visual treatment for Nebutra's dark-first surfaces. It creates layered depth through transparency, blur, and subtle borders.

### The Recipe

```css
/* Standard glass panel */
.glass {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(24px);
  -webkit-backdrop-filter: blur(24px);
  border: 1px solid rgba(255, 255, 255, 0.10);
  border-radius: 16px;
}

/* Elevated glass panel */
.glass-elevated {
  background: rgba(255, 255, 255, 0.08);
  backdrop-filter: blur(32px);
  -webkit-backdrop-filter: blur(32px);
  border: 1px solid rgba(255, 255, 255, 0.15);
  border-radius: 16px;
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);
}
```

### Tailwind Implementation

```tsx
// Standard glass panel
<div className="bg-white/5 backdrop-blur-xl border border-white/10 rounded-2xl">
  <div className="p-6">
    <h3 className="text-white font-semibold">Feature Title</h3>
    <p className="text-neutral-400 mt-2">Feature description text.</p>
  </div>
</div>

// Elevated glass panel (card with shadow)
<div className="bg-white/[0.08] backdrop-blur-[32px] border border-white/[0.15] rounded-2xl shadow-[0_8px_32px_rgba(0,0,0,0.3)]">
  <div className="p-8">
    <h3 className="text-white font-bold text-heading-24">Premium Feature</h3>
  </div>
</div>

// Glass with brand gradient border
<div className="relative bg-white/5 backdrop-blur-xl rounded-2xl overflow-hidden">
  <div className="absolute inset-0 rounded-2xl border border-transparent bg-[linear-gradient(135deg,rgba(0,51,254,0.3),rgba(11,241,195,0.3))] [mask:linear-gradient(#fff_0_0)_padding-box,linear-gradient(#fff_0_0)] [-webkit-mask-composite:xor] [mask-composite:exclude]" />
  <div className="relative p-6">
    {/* Content */}
  </div>
</div>
```

### Rules

- Glassmorphism is **dark-mode only** — never use on light backgrounds
- Minimum `backdrop-blur: 24px` — less creates muddy transparency
- Border opacity: `10-15%` white — more is too prominent
- Background opacity: `5-8%` white — more loses the glass effect
- Always pair with a dark page background (`neutral-950` or darker)

## Dark-First Design

Nebutra is a **dark-first** brand. The dark theme is the primary design surface, and light mode is the secondary adaptation.

### Why Dark-First

1. **Brand alignment** — the Blue-to-Cyan gradient is most vibrant on dark backgrounds
2. **Data density** — dashboards and data-heavy interfaces read better on dark surfaces
3. **Premium positioning** — dark interfaces convey sophistication and professionalism
4. **Eye comfort** — enterprise users spend hours in the product; dark reduces fatigue

### Implementation Rules

| Rule | Description |
|------|-------------|
| Design dark first | Start all designs on `neutral-950` backgrounds |
| Landing page is forced dark | The landing page never switches to light mode |
| Logo always white on landing | Use `text-white` for inline SVGs on landing page |
| Gradient on dark | The brand gradient is designed for dark backgrounds |
| Light mode is adaptation | Light mode uses lighter tints of brand colors, not different colors |
| Test dark first | Verify all components on dark backgrounds before checking light |

### Code Pattern

```tsx
// Dark-first component styling
<div className="bg-neutral-950 text-neutral-100">
  {/* Default: dark */}
</div>

// Light mode adaptation
<div className="bg-neutral-950 dark:bg-neutral-950 light:bg-white text-neutral-100 light:text-neutral-900">
  {/* Adapts to light mode */}
</div>

// Forced dark (landing page)
<div className="dark" data-theme="dark">
  {/* Always dark, regardless of system preference */}
</div>
```

## Pattern Combinations

### Recommended Combinations

| Primary Pattern | Secondary Pattern | Context |
|----------------|-------------------|---------|
| Hex Grid | Grain Texture | Hero sections |
| Hex Grid | Glassmorphism | Feature cards over hex background |
| Stars Canvas | Glassmorphism | Cloud/space themed pricing |
| Dot Pattern | None | Dashboard backgrounds |
| Flickering Grid | None | Tech-forward data sections |
| Mesh Gradient | Grain Texture | Immersive hero sections |
| Warp Background | Glassmorphism | Full-bleed landing hero |

### Prohibited Combinations

<Warning>
  The following pattern combinations create visual noise and must be avoided:
</Warning>

| Combination | Problem |
|-------------|---------|
| Hex Grid + Flickering Grid | Competing grid patterns create visual chaos |
| Stars Canvas + Flickering Grid | Too much motion competes for attention |
| Warp Background + Neural Noise | Two WebGL shaders overload GPU |
| Mesh Gradient + Warp Background | Conflicting motion and color fields |
| Any 2 WebGL patterns together | GPU resource contention, performance issues |

### Layering Order

When combining patterns, follow this z-index layering:

```
z-0: Background color (neutral-950)
z-10: Pattern layer (hex grid, dots, stars)
z-20: Texture layer (grain, noise)
z-30: Glass panels (glassmorphism)
z-40: Content (text, buttons, images)
z-50: Overlays (modals, tooltips)
```

## Do / Don't

**Do** use hexagonal grids as the primary decorative pattern — it is the visual DNA.

**Do** use glassmorphism exclusively on dark backgrounds.

**Do** design dark-first and adapt to light mode second.

**Do** keep pattern opacity subtle (5-15%) so content remains the focus.

**Do** limit to 1 primary pattern + 1 subtle texture per section.

**Don't** combine two competing grid patterns (hex + flickering).

**Don't** use more than 1 WebGL shader per page.

**Don't** use glassmorphism on light backgrounds — it requires dark surfaces to work.

**Don't** apply animated patterns to content-heavy sections — they distract from reading.

**Don't** use non-brand colors in patterns (e.g., purple hex grids, orange dots).

**Don't** skip the dark-first approach — light mode is always the secondary adaptation.
