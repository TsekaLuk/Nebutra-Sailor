## Overview

Icons are managed in `packages/icons` and published as the internal package `@nebutra/icons`. The full Geist icon set (541 icons) is sourced from [vercel.com/geist/icons](https://vercel.com/geist/icons) and converted to TypeScript React components via SVGR.

All icons accept a `size` prop and forward a `ref` to the root `<svg>` element. Colors inherit `currentColor` from the parent ‚Äî set icon color via the parent's `color` / `fill` CSS property.

---

## Installation

`@nebutra/icons` is a private workspace package. It is available in any app or package inside the monorepo:

```json
// package.json
{
  "dependencies": {
    "@nebutra/icons": "workspace:*"
  }
}
```

---

## Basic usage

```tsx
import { ArrowDown, LogoVercel, Search } from "@nebutra/icons";

// Default size (16px)
<ArrowDown />

// Custom size
<Search size={20} />

// Ref forwarding
<LogoVercel ref={myRef} size={24} />
```

## Size scale

All icons accept a `size` prop (number or string). Use these standard values for consistency:

| Token | Value | Use for |
|---|---|---|
| `sm` | `16` | Inline icons, tight UI |
| `md` | `20` | Buttons, inputs, default |
| `lg` | `24` | Navigation, standalone icons |
| `xl` | `32` | Feature illustrations, empty states |

```tsx
<ArrowDown size={16} />  // sm ‚Äî inline
<Search size={20} />     // md ‚Äî button / input
<Star size={24} />       // lg ‚Äî standalone
<Globe size={32} />      // xl ‚Äî empty state
```

## Props

```ts
import type { IconProps } from "@nebutra/icons";

interface IconProps extends React.SVGProps<SVGSVGElement> {
  /** Icon size in px (default: 16). Sets both width and height. */
  size?: number | string;
  // All standard SVGProps are forwarded (className, style, aria-*, etc.)
}
```

`size` overrides `width` and `height` props individually. To set non-square dimensions, pass `width` and `height` directly ‚Äî they take precedence over `size`.

## Accessibility

Icons used without adjacent visible text must have an accessible label.

```tsx
// Wrong ‚Äî no label for assistive technology
<button><Search size={16} /></button>

// Correct ‚Äî aria-label on the button, aria-hidden on the icon
<button aria-label="Search">
  <Search size={16} aria-hidden />
</button>

// Decorative icon alongside visible text ‚Äî hide from screen readers
<button className="flex items-center gap-2">
  <ArrowDown size={16} aria-hidden />
  Sort descending
</button>
```

## Color inheritance

Icons render as SVGs and inherit `currentColor`. Set color via the parent element's `color` property or a Tailwind class:

```tsx
// Tailwind ‚Äî inherits text color
<span className="text-blue-500">
  <Star size={16} aria-hidden />
</span>

// Direct style
<Check size={16} style={{ color: "var(--color-success)" }} aria-hidden />
```

## Adding or updating icons

Icons are generated from SVG files in `packages/icons/src/svg/`. To regenerate after updating SVGs:

```bash
pnpm --filter @nebutra/icons generate
pnpm --filter @nebutra/icons build
```

The generator (`scripts/generate.ts`) uses SVGR to convert each `.svg` file into a TypeScript React component and writes a barrel `src/index.ts`.

## Do / Don't

**Do** import icons exclusively from `@nebutra/icons`. Do not add `lucide-react`, `@heroicons/react`, or any other icon library as a direct dependency in app packages.

**Do** use the `size` prop to control dimensions. Never scale icons with CSS transforms, `width`/`height` style overrides, or `transform: scale()`.

**Do** pass `aria-hidden` on decorative icons and provide an accessible label on the parent interactive element.

**Don't** use icons smaller than 12px. SVG path detail is lost below this size and the icon becomes inaccessible.

**Don't** hardcode icon colors with `fill` or `stroke` overrides on the icon itself. Use `currentColor` inheritance via the parent element.
