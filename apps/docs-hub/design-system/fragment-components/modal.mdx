## Demo

```tsx
<Modal open={isOpen} onOpenChange={setIsOpen} title="Edit project">
  <Modal.Body>
    <FormItemLayout label="Project name" htmlFor="project-name">
      <Input id="project-name" defaultValue={project.name} />
    </FormItemLayout>
  </Modal.Body>
  <Modal.Footer>
    <Button variant="secondary" onClick={() => setIsOpen(false)}>Cancel</Button>
    <Button onClick={handleSave}>Save changes</Button>
  </Modal.Footer>
</Modal>
```

## Usage

```tsx
import { Modal } from "@nebutra/custom-ui";
import { useState } from "react";

export function CreatePipelineButton() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setOpen(true)}>New pipeline</Button>

      <Modal
        open={open}
        onOpenChange={setOpen}
        title="Create pipeline"
        description="Configure your new data pipeline."
        size="lg"
      >
        <Modal.Body>
          <CreatePipelineForm onSuccess={() => setOpen(false)} />
        </Modal.Body>
      </Modal>
    </>
  );
}
```

## Props

### `Modal`

| Prop | Type | Default | Description |
|---|---|---|---|
| `open` | `boolean` | — | Controls dialog open state |
| `onOpenChange` | `(open: boolean) => void` | — | Called when the dialog requests a state change |
| `title` | `string` | — | Dialog heading text |
| `description` | `string` | — | Muted subtitle below the title |
| `size` | `"sm" \| "md" \| "lg" \| "xl" \| "full"` | `"md"` | Max-width of the dialog panel |
| `hideCloseButton` | `boolean` | `false` | Hides the default X close button |
| `className` | `string` | — | Additional classes on the dialog panel |
| `children` | `React.ReactNode` | — | Dialog content (use sub-components for layout) |

### `Modal.Body`

Scrollable content region. Accepts any `React.ReactNode`.

### `Modal.Footer`

Sticky footer anchored to the bottom of the dialog, right-aligned by default.

| Prop | Type | Default | Description |
|---|---|---|---|
| `align` | `"start" \| "end" \| "between"` | `"end"` | Horizontal alignment of footer actions |
| `children` | `React.ReactNode` | — | Action buttons |

## Examples

### Small confirmation-style modal

```tsx
<Modal open={open} onOpenChange={setOpen} title="Remove member?" size="sm">
  <Modal.Body>
    <p className="text-sm text-gray-600">
      Alex Johnson will lose access to this workspace immediately.
    </p>
  </Modal.Body>
  <Modal.Footer>
    <Button variant="secondary" onClick={() => setOpen(false)}>Cancel</Button>
    <Button variant="danger" onClick={handleRemove}>Remove</Button>
  </Modal.Footer>
</Modal>
```

### Full-screen on mobile, large on desktop

```tsx
<Modal open={open} onOpenChange={setOpen} title="Import data" size="lg">
  <Modal.Body>
    <ImportWizard />
  </Modal.Body>
</Modal>
```

### Without a footer

```tsx
<Modal open={open} onOpenChange={setOpen} title="Keyboard shortcuts">
  <Modal.Body>
    <ShortcutList />
  </Modal.Body>
</Modal>
```

## Accessibility

- Renders as a native `<dialog>` element with `role="dialog"`.
- `aria-labelledby` points to the title; `aria-describedby` points to the description when provided.
- Focus is trapped inside the dialog while open (first focusable element receives focus on open).
- Focus returns to the trigger element on close.
- Pressing `Escape` closes the dialog and fires `onOpenChange(false)`.
- The backdrop click closes the dialog by default.
- Scrollable `Modal.Body` has `tabIndex={0}` to allow keyboard scrolling.
