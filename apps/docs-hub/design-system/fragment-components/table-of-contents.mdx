---
title: "TableOfContents"
description: "Scroll-tracking in-page table of contents that highlights the active section as the user scrolls."
---

## Demo

```tsx
<TableOfContents
  items={[
    { id: "overview", label: "Overview", level: 2 },
    { id: "installation", label: "Installation", level: 2 },
    { id: "configuration", label: "Configuration", level: 2 },
    { id: "auth-providers", label: "Auth providers", level: 3 },
    { id: "env-vars", label: "Environment variables", level: 3 },
    { id: "deployment", label: "Deployment", level: 2 },
  ]}
/>
```

## Usage

```tsx
import { TableOfContents } from "@nebutra/custom-ui";

export function DocLayout({ headings, children }: DocLayoutProps) {
  return (
    <div className="flex gap-12 max-w-7xl mx-auto px-6">
      <article className="flex-1 min-w-0">
        {children}
      </article>

      <aside className="w-56 shrink-0 hidden xl:block">
        <div className="sticky top-20">
          <p className="text-xs font-semibold uppercase tracking-wider text-gray-500 mb-3">
            On this page
          </p>
          <TableOfContents items={headings} />
        </div>
      </aside>
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `items` | `TocItem[]` | — | **Required.** Ordered list of section entries |
| `activeId` | `string` | — | Controlled active section id; if omitted, `IntersectionObserver` is used |
| `onItemClick` | `(id: string) => void` | — | Optional click handler (in addition to native scroll) |
| `offset` | `number` | `80` | Scroll offset in pixels used for intersection threshold (accounts for sticky headers) |
| `className` | `string` | — | Additional classes on the `<nav>` root |

### `TocItem`

```ts
interface TocItem {
  id: string;       // Matches the heading element's id attribute
  label: string;    // Display text
  level: 2 | 3 | 4; // Heading level — controls visual indentation
}
```

## Examples

### Controlled active id

When integrating with a custom scroll handler:

```tsx
const [activeId, setActiveId] = useState("overview");

useScrollSpy(items.map((i) => i.id), {
  onActiveChange: setActiveId,
  offset: 96,
});

<TableOfContents
  items={items}
  activeId={activeId}
/>
```

### Flat (h2 only) TOC

```tsx
<TableOfContents
  items={h2Headings.map((h) => ({ id: h.id, label: h.text, level: 2 }))}
/>
```

### Custom scroll offset for tall sticky header

```tsx
<TableOfContents items={items} offset={120} />
```

## Accessibility

- Renders as `<nav aria-label="Table of contents">`.
- Each link is a native `<a href="#{id}">` for correct browser history and keyboard navigation.
- Active item has `aria-current="true"`.
- Indentation for nested items (`level: 3`, `level: 4`) is purely visual via padding; the list is flat (`<ul>`) without nesting to avoid confusing screen reader virtual cursor navigation.
- Clicking a link moves focus to the target heading, which must have a matching `id` attribute.
