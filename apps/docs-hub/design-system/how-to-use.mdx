## Installation

<Steps>
  <Step title="Add the dependency">
    The design system is a workspace package. Add it to your app's `package.json`:

    ```json
    {
      "dependencies": {
        "@nebutra/design-system": "workspace:*"
      }
    }
    ```

    Then install from the repo root:

    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Wrap your app with DesignSystemProvider">
    Add the provider to your root layout. This sets up theme context and CSS variables.

    ```tsx
    // app/layout.tsx (Next.js App Router)
    import { DesignSystemProvider } from "@nebutra/design-system";

    export default function RootLayout({ children }: { children: React.ReactNode }) {
      return (
        <html>
          <body>
            <DesignSystemProvider>{children}</DesignSystemProvider>
          </body>
        </html>
      );
    }
    ```
  </Step>

  <Step title="Import and use components">
    All exports are available from the single package entry point:

    ```tsx
    import { Button, Card, Badge } from "@nebutra/design-system";

    export function MyPage() {
      return (
        <Card>
          <Badge variant="secondary">New</Badge>
          <Button>Get started</Button>
        </Card>
      );
    }
    ```
  </Step>
</Steps>

## Available Export Paths

| Import path | Contents |
|---|---|
| `@nebutra/design-system` | All components, hooks, and utilities |
| `@nebutra/design-system/theme` | Color tokens, brand overrides, marketing tokens |
| `@nebutra/design-system/typography` | Font families, type scale, fluid presets |
| `@nebutra/design-system/components` | UI components only |
| `@nebutra/design-system/icons` | Octicons re-exports |
| `@nebutra/design-system/utils` | `cn`, breakpoint helpers |
| `@nebutra/design-system/hooks` | `useDesignSystem`, `useTheme` |

## Theme Mode

Use `useDesignSystem` to read and toggle the current color mode:

```tsx
import { useDesignSystem } from "@nebutra/design-system";

function ThemeToggle() {
  const { toggleMode, resolvedMode } = useDesignSystem();
  return (
    <button onClick={toggleMode}>
      {resolvedMode === "dark" ? "Light mode" : "Dark mode"}
    </button>
  );
}
```

## Brand Overrides

Apply brand-specific colors and typography without touching global tokens:

```tsx
import { createTheme, DesignSystemProvider } from "@nebutra/design-system";

const brandTheme = createTheme("light", {
  colors: {
    accent: {
      fg: "#your-brand-color",
      emphasis: "#your-brand-emphasis",
    },
  },
  typography: {
    fontFamily: {
      normal: '"YourBrandFont", Inter, sans-serif',
    },
  },
});

<DesignSystemProvider theme={brandTheme}>{children}</DesignSystemProvider>
```

See [Theming](/design-system/foundations/theming) for the full `BrandOverrides` interface.
