## When to Use

Use the form pattern whenever you collect structured data from a user and need to validate or submit it. This pattern covers:

- Settings panels (account, billing, notification preferences)
- Create / edit flows (new resource, inline edit)
- Authentication forms (sign-in, sign-up)
- Confirmation dialogs that require typed input

Do not apply this pattern to single-field quick-edits. For a single toggle or a single select that saves on change, use the bare atom component (`ToggleSwitch`, `Select`) without form scaffolding.

## Structure

```
<form> (or FormControl root)
  └── Section (logical grouping with title)
       ├── FormControl (label + input + validation message)
       │    ├── FormControl.Label
       │    ├── Input | Textarea | Select | Checkbox | RadioGroup
       │    ├── FormControl.Caption (hint text)
       │    └── FormControl.Validation (error message)
       ├── FormControl (repeat per field)
       └── ButtonGroup (submit + cancel, bottom of section)
```

For multi-step flows, wrap each step in its own `Section`. Render step controls (`Previous` / `Next`) inside each section's action slot.

## Composition

### FormControl

`FormControl` from `@primer/react` owns the label–input relationship. Always use it instead of a bare `<label>` element.

```tsx
import { FormControl, TextInput, Button } from "@nebutra/design-system";

<FormControl>
  <FormControl.Label>API key name</FormControl.Label>
  <TextInput
    placeholder="e.g. production-server"
    value={name}
    onChange={(e) => setName(e.target.value)}
  />
  <FormControl.Caption>
    Used to identify this key in audit logs.
  </FormControl.Caption>
</FormControl>
```

### Validation

Surface field-level errors using `FormControl.Validation` paired with `validationStatus` on the input.

```tsx
<FormControl>
  <FormControl.Label>Email</FormControl.Label>
  <TextInput
    type="email"
    value={email}
    onChange={(e) => setEmail(e.target.value)}
    validationStatus={errors.email ? "error" : undefined}
  />
  {errors.email && (
    <FormControl.Validation variant="error">
      {errors.email}
    </FormControl.Validation>
  )}
</FormControl>
```

Validate on blur for new forms. Validate on change once a field has been submitted with an error (real-time correction UX).

### Grouping with Section

Separate logically distinct concerns into labelled sections. This aids scanability on long settings pages.

```tsx
import { Section } from "@nebutra/design-system";

<Section title="Personal information">
  {/* name, email fields */}
</Section>

<Section title="Security">
  {/* password, 2FA fields */}
</Section>
```

### Submit and cancel

Place submit/cancel controls at the end of the form (or section for multi-step). Destructive actions (`Delete`, `Revoke`) go in separate sections with a `Flash` warning above the button.

```tsx
import { Button, ButtonGroup } from "@nebutra/design-system";

<ButtonGroup>
  <Button variant="primary" type="submit" disabled={isSubmitting}>
    {isSubmitting ? "Saving…" : "Save changes"}
  </Button>
  <Button variant="invisible" onClick={onCancel}>
    Cancel
  </Button>
</ButtonGroup>
```

### Required fields

Mark required fields with `FormControl.Label required`. Do not use asterisk-only conventions without a legend explaining the symbol.

```tsx
<FormControl required>
  <FormControl.Label>Organization name</FormControl.Label>
  <TextInput … />
</FormControl>
```

### Complex input types

| Use case | Component |
|---|---|
| Long text | `Textarea` |
| Single choice from a list | `Select` or `RadioGroup` |
| Binary on/off | `Checkbox` or `ToggleSwitch` |
| Multi-selection | `CheckboxGroup` or `multi-select` fragment |
| Date | `DatePicker` |
| OTP / pin | `InputOtp` |
| Token input | `TextInputWithTokens` |

## Examples

### Create API key form

```tsx
import { FormControl, TextInput, Select, Button } from "@nebutra/design-system";
import { Section } from "@nebutra/design-system";

function CreateKeyForm({ onSubmit, onCancel }: Props) {
  const [name, setName] = useState("");
  const [permission, setPermission] = useState("read");
  const [errors, setErrors] = useState<Record<string, string>>({});

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    const errs: Record<string, string> = {};
    if (!name.trim()) errs.name = "Key name is required.";
    if (Object.keys(errs).length) { setErrors(errs); return; }
    onSubmit({ name, permission });
  };

  return (
    <form onSubmit={handleSubmit}>
      <Section title="Key details">
        <FormControl required>
          <FormControl.Label>Name</FormControl.Label>
          <TextInput
            value={name}
            onChange={(e) => setName(e.target.value)}
            validationStatus={errors.name ? "error" : undefined}
          />
          {errors.name && (
            <FormControl.Validation variant="error">
              {errors.name}
            </FormControl.Validation>
          )}
        </FormControl>

        <FormControl>
          <FormControl.Label>Permission</FormControl.Label>
          <Select value={permission} onChange={(e) => setPermission(e.target.value)}>
            <Select.Option value="read">Read only</Select.Option>
            <Select.Option value="write">Read + write</Select.Option>
          </Select>
        </FormControl>

        <ButtonGroup>
          <Button variant="primary" type="submit">Create key</Button>
          <Button variant="invisible" onClick={onCancel}>Cancel</Button>
        </ButtonGroup>
      </Section>
    </form>
  );
}
```
