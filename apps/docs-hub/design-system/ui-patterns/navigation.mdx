## When to Use

Choose the navigation pattern that matches the depth and frequency of the destinations being linked.

| Pattern | Use when |
|---|---|
| `Header` | Top-level global navigation, always visible |
| `NavList` / `Sidebar` | Second-level section navigation (settings, admin areas) |
| `UnderlineNav` / `Tabs` | Peer views of the same resource (overview, activity, settings) |
| `Breadcrumbs` | Hierarchical location in a nested resource tree |
| `Pagination` | Moving through a large ordered result set |
| `TreeView` | Hierarchical file-system or nested item navigation |

## Structure

### Global header

```
Header
  ├── Header.Item full → brand / logo
  ├── Header.Item → primary nav links (UnderlineNav or links)
  └── Header.Item → auth controls (UserButton, SignIn, SignUp)
```

### Settings page navigation

```
<div className="grid gap-8 md:grid-cols-[240px_1fr]">
  NavList                                ← vertical nav
    NavList.Group (optional label)
      NavList.Item (active state)
      NavList.Item
  <div>
    Section content                      ← main panel
  </div>
</div>
```

### Tab navigation (peer views)

```
UnderlineNav (or Tabs)
  UnderlineNav.Item (current page)
  UnderlineNav.Item
  UnderlineNav.Item
<div>
  {/* content for active tab */}
</div>
```

### Breadcrumb trail

```
PageHeader
  breadcrumbs={[
    { label: "Settings", href: "/settings" },
    { label: "API Keys", href: "/settings/api-keys" },
    { label: "Production key" },     ← current page, no href
  ]}
```

## Composition

### Global Header

The application shell at `apps/web/src/app/providers/design-system-shell.tsx` demonstrates the canonical `Header` pattern. `Header.Item full` takes all available space (flex-grow); subsequent `Header.Item` elements stack to the right.

```tsx
import { Header, Button } from "@nebutra/design-system";

<Header>
  <Header.Item full>
    <span className="text-xl font-bold">Nebutra</span>
  </Header.Item>
  <Header.Item>
    {/* auth controls */}
  </Header.Item>
</Header>
```

The `Header` sits outside `<main>`. The remaining viewport height is available to main content via `min-h-[calc(100vh-64px)]`.

### NavList (settings sidebar)

```tsx
import { NavList } from "@nebutra/design-system";

<NavList aria-label="Settings navigation">
  <NavList.Group title="Account">
    <NavList.Item href="/settings/general" aria-current={isGeneral ? "page" : undefined}>
      General
    </NavList.Item>
    <NavList.Item href="/settings/api-keys" aria-current={isApiKeys ? "page" : undefined}>
      API Keys
    </NavList.Item>
  </NavList.Group>
  <NavList.Group title="Organisation">
    <NavList.Item href="/settings/members">Members</NavList.Item>
    <NavList.Item href="/settings/billing">Billing</NavList.Item>
  </NavList.Group>
</NavList>
```

Apply `aria-current="page"` to the active item. `NavList` renders an `<nav>` — always provide `aria-label` to distinguish it from other `<nav>` elements on the page.

### UnderlineNav (tab navigation)

Use `UnderlineNav` for switching between peer views of the same entity. The component renders as a `<nav>` with an underline indicator on the active item.

```tsx
import { UnderlineNav } from "@nebutra/design-system";

<UnderlineNav aria-label="Key details">
  <UnderlineNav.Item href={`/keys/${id}`} aria-current={tab === "overview" ? "page" : undefined}>
    Overview
  </UnderlineNav.Item>
  <UnderlineNav.Item href={`/keys/${id}/logs`} aria-current={tab === "logs" ? "page" : undefined}>
    Logs
  </UnderlineNav.Item>
</UnderlineNav>
```

For client-side tab switching without URL changes (e.g., within a modal), use the `Tabs` atom instead.

### Breadcrumbs

Pass the `breadcrumbs` prop to `PageHeader`. The last item in the array is rendered as the current page (no link, `selected={true}` on the `Breadcrumbs.Item`).

```tsx
<PageHeader
  title="Production key"
  breadcrumbs={[
    { label: "Settings", href: "/settings" },
    { label: "API Keys", href: "/settings/api-keys" },
    { label: "Production key" },
  ]}
/>
```

Breadcrumbs are appropriate for resources nested 2 or more levels deep. Do not add breadcrumbs to top-level pages (Dashboard, Settings root).

### Pagination

Use `Pagination` at the bottom of tables or lists when the total item count exceeds the page size. Pair it with the table pattern — render it below the `DataTable`.

```tsx
import { Pagination } from "@nebutra/design-system";

<Pagination
  pageCount={totalPages}
  currentPage={page}
  onPageChange={(p) => setPage(p)}
/>
```

For server-side pagination, update the URL on page change and derive `page` from the URL param.

### Table of Contents

For long content pages, use the `TableOfContents` fragment. It renders a sticky right-rail list of section anchors and highlights the active section as the user scrolls.

## Examples

### Settings page layout with NavList

```tsx
export default function SettingsLayout({ children }: { children: ReactNode }) {
  const pathname = usePathname();

  return (
    <Container size="lg">
      <PageHeader title="Settings" />
      <div className="grid gap-8 md:grid-cols-[240px_1fr]">
        <NavList aria-label="Settings">
          <NavList.Item
            href="/settings"
            aria-current={pathname === "/settings" ? "page" : undefined}
          >
            General
          </NavList.Item>
          <NavList.Item
            href="/settings/api-keys"
            aria-current={pathname.startsWith("/settings/api-keys") ? "page" : undefined}
          >
            API Keys
          </NavList.Item>
        </NavList>
        <div>{children}</div>
      </div>
    </Container>
  );
}
```
