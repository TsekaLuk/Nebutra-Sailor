07 — Guidelines

Do's & Don'ts

Do
  • Use brand gradient (135deg) on hero sections and primary CTAs
  • Use Geist Sans for headlines, Poppins for English body, vivo Sans for Chinese
  • Use #0033FE for info semantic states (not sky/teal)
  • Use neutral-950 (#020617) as dark mode background
  • Use emerge motion for content reveals, flow for horizontals
  • Use brand easing cubic-bezier(0.16, 1, 0.3, 1) for all transitions
  • Maintain WCAG 2.1 AA: 4.5:1 normal text, 3:1 large text
  • Use hex grid as primary decorative background pattern
  • Design dark-first, then adapt to light mode
  • Respect prefers-reduced-motion for all animations
Don't
  • Distort, rotate, recolor, or add shadows to the logo
  • Use #000000 as background — use neutral-950 (#020617)
  • Substitute brand blue with sky-500, teal, or generic blue-600
  • Use cyan (#0BF1C3) as text on light backgrounds (fails WCAG)
  • Use Inter, Roboto, Arial, SF Pro, or other non-brand fonts
  • Use Poppins for hero headlines (use Geist) or Poppins weights outside 400-700
  • Use generic ease-in-out or linear easing
  • Animate transitions longer than 0.8s
  • Use solid blue where brand gradient is expected
  • Use more than 1 WebGL shader per page
  • Add theme toggle to the landing page (forced dark)
  • Place glassmorphism on light backgrounds
