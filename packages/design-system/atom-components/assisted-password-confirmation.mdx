## Overview

`AssistedPasswordConfirmation` provides real-time visual feedback as the user types their password confirmation:

- Characters that match the original password are highlighted **green**
- Characters that don't match are highlighted **red**
- The input **shakes** (Framer Motion) if the user tries to type beyond the password length
- An optional hint shows the masked original password characters

## Usage

```tsx
import { AssistedPasswordConfirmation } from "@nebutra/custom-ui/primitives";

const [password, setPassword] = useState("");
const [isMatched, setIsMatched] = useState(false);

// In your form
<div>
  <input
    type="password"
    value={password}
    onChange={(e) => setPassword(e.target.value)}
    placeholder="Password"
  />
  <AssistedPasswordConfirmation
    password={password}
    onMatch={() => setIsMatched(true)}
    onConfirmChange={(value) => {
      if (value !== password) setIsMatched(false);
    }}
  />
</div>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `password` | `string` | — | **Required.** The original password to match against |
| `placeholder` | `string` | `"Confirm Password"` | Input placeholder text |
| `onConfirmChange` | `(value: string) => void` | — | Called when confirmation input changes |
| `onMatch` | `() => void` | — | Called when passwords fully match |
| `showHint` | `boolean` | `true` | Show masked password character indicators |
| `className` | `string` | — | Container CSS class |

## Notes

- The component tracks the confirmation text internally — it is **uncontrolled**
- Typing is blocked once the confirmation reaches the same length as `password`; further keypresses trigger a shake animation
- Use `onConfirmChange` + `onMatch` to integrate into form state management
- The shake animation uses Framer Motion's `motion.div` with `animate` variants
