## Overview

`HeroCard` wraps [HeroUI's Card](https://www.heroui.com/docs/components/card) component. It supports blur effects, press interactions (renders as a `<button>`), hover animations, and a three-part slot layout (Header / Body / Footer).

## Installation

```tsx
import {
  HeroCard,
  HeroCardHeader,
  HeroCardBody,
  HeroCardFooter,
} from "@nebutra/ui/primitives";
```

## Usage

### Basic card

```tsx
<HeroCard>
  <HeroCardHeader>
    <h4 className="font-semibold">Card Title</h4>
  </HeroCardHeader>
  <HeroCardBody>
    <p>Make beautiful websites regardless of your design experience.</p>
  </HeroCardBody>
  <HeroCardFooter>
    <button>Learn more</button>
  </HeroCardFooter>
</HeroCard>
```

### Hoverable

```tsx
<HeroCard isHoverable>
  <HeroCardBody>Hover over me</HeroCardBody>
</HeroCard>
```

### Pressable

Makes the card act as a button with press events.

```tsx
<HeroCard isPressable onPress={() => console.log("Pressed!")}>
  <HeroCardBody>Click me</HeroCardBody>
</HeroCard>
```

### Blurred (glass effect)

Requires a gradient background ancestor.

```tsx
<div className="bg-gradient-to-tr from-pink-500 to-yellow-500 p-8">
  <HeroCard isBlurred>
    <HeroCardBody>Blurred glass card</HeroCardBody>
  </HeroCard>
</div>
```

### Blurred footer

The footer becomes a frosted glass overlay at the bottom of the card.

```tsx
<HeroCard isFooterBlurred className="h-[300px] relative">
  <HeroCardBody>Background content</HeroCardBody>
  <HeroCardFooter className="absolute bottom-0 z-10 backdrop-blur">
    <button>View details</button>
  </HeroCardFooter>
</HeroCard>
```

## Props

### HeroCard

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `shadow` | `"none" \| "sm" \| "md" \| "lg"` | `"md"` | Shadow depth |
| `radius` | `"none" \| "sm" \| "md" \| "lg"` | `"lg"` | Border radius |
| `fullWidth` | `boolean` | `false` | Expand to fill container |
| `isHoverable` | `boolean` | `false` | Enable hover scale effect |
| `isPressable` | `boolean` | `false` | Render as button, enable press events |
| `isBlurred` | `boolean` | `false` | Apply blur effect to card background |
| `isFooterBlurred` | `boolean` | `false` | Apply blur effect to footer |
| `isDisabled` | `boolean` | `false` | Disable the card |
| `disableAnimation` | `boolean` | `false` | Disable all animations |
| `disableRipple` | `boolean` | `false` | Disable ripple effect on press |
| `allowTextSelectionOnPress` | `boolean` | `false` | Allow text selection when pressable |
| `classNames` | `Partial<Record<"base" \| "header" \| "body" \| "footer", string>>` | — | Per-slot class names |
| `onPress` | `(e: PressEvent) => void` | — | Press event handler |
| `onPressStart` | `(e: PressEvent) => void` | — | Press start handler |
| `onPressEnd` | `(e: PressEvent) => void` | — | Press end handler |
| `onPressChange` | `(isPressed: boolean) => void` | — | Press state change handler |
| `onPressUp` | `(e: PressEvent) => void` | — | Press up handler |

### HeroCardHeader / HeroCardBody / HeroCardFooter

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Slot content |
| `className` | `string` | Additional CSS classes |

## Notes

- Press events use React Aria's `PressEvent` interface for cross-pointer-type support.
- `isBlurred` works best on cards over colorful or gradient backgrounds.
- Requires `@heroui/card` to be installed in the consuming app.
