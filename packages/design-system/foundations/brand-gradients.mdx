## Brand Gradients

Four gradient variants cover all use cases. Each is defined with specific direction, stops, and context.

<CardGroup cols={2}>
  <Card title="Brand Primary" icon="arrow-right">
    `linear-gradient(135deg, #0033FE 0%, #0BF1C3 100%)`

    **Usage**: Hero sections, primary CTA buttons, logo gradient fill, feature highlights
  </Card>
  <Card title="Brand Reverse" icon="arrow-left">
    `linear-gradient(135deg, #0BF1C3 0%, #0033FE 100%)`

    **Usage**: Hover state on gradient elements, secondary emphasis
  </Card>
  <Card title="Section Vertical" icon="arrow-down">
    `linear-gradient(180deg, #0033FE 0%, #0BF1C3 100%)`

    **Usage**: Page section dividers, vertical feature strips, sidebar accents
  </Card>

    `radial-gradient(circle, #0BF1C3 0%, #0033FE 100%)`

  </Card>

### CSS Custom Properties

```css
:root {
  --gradient-brand: linear-gradient(135deg, #0033FE 0%, #0BF1C3 100%);
  --gradient-brand-reverse: linear-gradient(135deg, #0BF1C3 0%, #0033FE 100%);
  --gradient-brand-vertical: linear-gradient(180deg, #0033FE 0%, #0BF1C3 100%);
  --gradient-brand-glow: radial-gradient(circle, #0BF1C3 0%, #0033FE 100%);
}
```

### Tailwind Usage

```tsx
// Primary gradient button
<button className="bg-[linear-gradient(135deg,#0033FE_0%,#0BF1C3_100%)] text-white border-0 rounded-lg px-6 py-3">
  Get Started
</button>

// Hover: swap to reverse gradient
<button className="bg-[var(--gradient-brand)] hover:bg-[var(--gradient-brand-reverse)] text-white">
  Get Started
</button>

// Gradient text
<h1 className="bg-[var(--gradient-brand)] bg-clip-text text-transparent">
  Enterprise SaaS Platform
</h1>

// Vertical section divider
<div className="h-1 w-full bg-[var(--gradient-brand-vertical)]" />

// Glow background
<div className="absolute inset-0 bg-[var(--gradient-brand-glow)] opacity-20 blur-3xl" />
```

### React / CSS-in-JS

```tsx
// CSS custom property
<div style={{ background: 'var(--gradient-brand)' }} />

// Direct value
<div style={{ background: 'linear-gradient(135deg, #0033FE 0%, #0BF1C3 100%)' }} />
```

## Animated Gradients

### AnimatedGradientText

A text component with a slowly shifting gradient fill. Use for hero taglines and featured copy.

```tsx
import { AnimatedGradientText } from "@nebutra/custom-ui"

<AnimatedGradientText>
  Intelligent Data. Cloud Nurtured.
</AnimatedGradientText>
```

The gradient cycles between the brand primary and reverse directions over a 6-second loop, creating a subtle shimmer effect.

### GradientAnimatedText

A variant with configurable speed and direction. Use for interactive labels.

```tsx
import { GradientAnimatedText } from "@nebutra/custom-ui"

<GradientAnimatedText
  speed={4}
  direction="horizontal"
  className="text-heading-36 font-display font-bold"
>
  Cloud Intelligence
</GradientAnimatedText>
```

## Grain Texture

### GrainGradientBackground

Adds analog warmth to digital surfaces by overlaying a subtle noise texture on the brand gradient. This is the signature hero section treatment.

```tsx
import { GrainGradientBackground } from "@nebutra/custom-ui"

<section className="relative h-[600px]">
  <GrainGradientBackground
    intensity={0.15}
    className="absolute inset-0"
  />
  <div className="relative z-10">
    {/* Hero content */}
  </div>
</section>
```

### SVG feTurbulence Approach

For lightweight grain without a component dependency, use an inline SVG filter:

```css
.grain-overlay::after {
  content: "";
  position: absolute;
  inset: 0;
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='noise'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.65' numOctaves='3' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23noise)' opacity='0.08'/%3E%3C/svg%3E");
  pointer-events: none;
  opacity: 0.1;
}
```

<Tip>
  Grain texture adds analog warmth that counters the "too perfect" feel of pure CSS gradients.
  Apply at 8-15% opacity — any higher and it competes with content readability.
</Tip>

## Mesh Gradient

### MeshGradientBg

A WebGL-powered multi-point gradient that creates organic, flowing color fields. Use for immersive hero sections and feature landing pages.

```tsx
import { MeshGradientBg } from "@nebutra/custom-ui"

<section className="relative min-h-screen">
  <MeshGradientBg
    colors={["#0033FE", "#0BF1C3", "#002ad4", "#1af7c8"]}
    speed={0.003}
    className="absolute inset-0"
  />
  <div className="relative z-10">
    {/* Page content */}
  </div>
</section>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `colors` | `string[]` | Brand blue/cyan | Array of 4 HEX colors for gradient control points |
| `speed` | `number` | `0.003` | Animation speed multiplier |
| `className` | `string` | — | Container CSS classes |

## When to Use Each Texture

| Texture | Context | Performance | Fallback |
|---------|---------|-------------|----------|
| **Grain** | Hero sections, about pages | Low (CSS/SVG) | Solid gradient |
| **Mesh Gradient** | Immersive hero, feature landing | Medium (WebGL) | Static CSS gradient |
| **Neural Noise** | AI/ML features, deep-tech pages | High (WebGL) | Grain + gradient |
| **Waves** | Data streaming, pipeline features | Medium (WebGL) | CSS gradient |
| **Dithering** | Heritage/retro sections, subtle texture | Low (CSS) | Solid background |
| **AnimatedGradientText** | Hero taglines, featured copy | Low (CSS) | Static gradient text |

## Performance Notes

<Warning>
  WebGL shaders (MeshGradient, NeuroNoise, Waves) require GPU acceleration. They must not be used on every page.
</Warning>

### Guidelines

- **Maximum 1 WebGL background per page** — multiple shaders compete for GPU resources
- **Always provide a CSS fallback** — detect WebGL support and degrade gracefully
- **Disable on low-power devices** — use `prefers-reduced-motion` and battery API
- **Lazy-load shaders** — only initialize when the section enters the viewport

### CSS Fallback Pattern

```tsx
<section className="relative">
  {/* CSS fallback — always renders */}
  <div className="absolute inset-0 bg-[var(--gradient-brand)]" />

  {/* WebGL enhancement — lazy-loaded */}
  <Suspense fallback={null}>
    <MeshGradientBg className="absolute inset-0" />
  </Suspense>
</section>
```

### Reduced Motion

```tsx
import { useReducedMotion } from "framer-motion"

function HeroBackground() {
  const prefersReducedMotion = useReducedMotion()

  if (prefersReducedMotion) {
    return <div className="bg-[var(--gradient-brand)]" />
  }

  return <MeshGradientBg />
}
```

## Do / Don't

**Do** use the brand gradient (135deg) as the default for all gradient surfaces.

**Do** apply grain texture at 8-15% opacity for analog warmth.

**Do** provide CSS fallbacks for all WebGL shaders.

**Do** respect `prefers-reduced-motion` for all animated textures.

**Don't** change the gradient angle from 135deg for the primary brand gradient.

**Don't** use more than 1 WebGL background per page.

**Don't** apply grain at opacity > 15% — it reduces content readability.

**Don't** use non-brand colors in gradient stops (e.g., purple, orange, pink).

**Don't** use animated textures on content-heavy pages where they compete for attention.
