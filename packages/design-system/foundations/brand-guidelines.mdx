## Logo Usage Rules

### Clearance and Sizing

| Rule | Specification |
|------|--------------|
| **Minimum clearance** | All sides >= 1/4 of logo height |
| **Digital minimum height** | >= 35 px |
| **Print minimum height** | >= 6 mm |

### Approved Uses

- Logo on dark backgrounds (`neutral-950`, `neutral-900`) with white or gradient fill
- Logo with brand gradient fill on dark backgrounds
- Logomark alone when horizontal space is limited (mobile headers, favicons)
- Wordmark alone when the logomark has already been established on the page

### Prohibited Logo Uses

<Warning>
  The following logo treatments are strictly forbidden:
</Warning>

| Prohibited | Reason |
|-----------|--------|
| Distorting proportions (stretch, skew) | Destroys geometric precision |
| Rotating the logo | The hexagonal N has a fixed orientation |
| Recoloring with non-brand colors | Only brand gradient, white, or `currentColor` |
| Adding drop shadows or outer glow | Clutters the clean geometric form |
| Placing on low-contrast backgrounds | Logo must remain clearly visible |
| Placing on busy/photographic backgrounds | Use a solid or frosted backdrop behind the logo |
| Using the reverse (white) logo on light backgrounds | White logo requires dark or gradient backgrounds |
| Outlining or applying strokes | The logo is a filled shape, not a line drawing |
| Animating the logo shape itself | The logomark is static — only entrance motion is permitted |

### Logo on Landing Page

```tsx
// CORRECT — always white on landing (forced dark)
<LogoEnSVG className="text-white" width={160} />

// WRONG — never conditionally swap on landing
<LogoEnSVG className={isDark ? "text-white" : "text-black"} width={160} />
```

## Color Rules

### Mandatory

| Rule | Detail |
|------|--------|
| Primary actions use `blue-500` | Buttons, links, focus rings = `#0033FE` |
| Info semantic = brand blue | `#0033FE`, not sky or teal |
| Neutrals use blue-undertone scale | `#f8fafc` through `#020617` |
| Dark mode background = `neutral-950` | `#020617` |
| Light mode background = `white` | `#ffffff` |

### Prohibited Color Substitutions

| Prohibited | Correct Alternative |
|-----------|-------------------|
| Tailwind `sky-500` (`#0ea5e9`) for info | Brand blue `#0033FE` |
| Tailwind `teal-500` (`#14b8a6`) for accent | Brand cyan `#0BF1C3` |
| Tailwind `gray-*` for neutrals | Brand neutrals (`#f8fafc` - `#020617`) |
| Orange or red as primary action color | Brand blue `#0033FE` |
| Cyan (`#0BF1C3`) as text on light backgrounds | Fails WCAG — use on dark only |
| Generic `blue-600` for brand blue | Use exact `#0033FE` |
| Any green as brand accent | Brand cyan `#0BF1C3` only |

### Color Pairing Rules

- `blue-500` + `cyan-500` = brand gradient (the only approved two-color pairing)
- `blue-500` + `white` = primary button on light backgrounds
- `cyan-500` + `neutral-950` = accent text on dark backgrounds
- Never pair `blue-500` with `cyan-500` as two flat fills side by side — always use a gradient

## Typography Rules

### Font Requirements

| Context | Required Font | Fallback |
|---------|--------------|----------|
| Hero/display text | Geist Sans | System sans-serif |
| English body | Poppins | System sans-serif |
| Chinese body | vivo Sans | PingFang SC, Microsoft YaHei |
| Code | JetBrains Mono | Fira Code, system monospace |

### Prohibited Typography

| Prohibited | Reason |
|-----------|--------|
| Inter, Roboto, or Arial for body text | Non-brand fonts |
| SF Pro in web applications | Apple-only, not cross-platform |
| Poppins for hero headlines | Use Geist Sans for display text |
| Geist Sans for body copy | Use Poppins for body text |
| Poppins weights below 400 or above 700 | Only 400, 500, 600, 700 are approved |
| Custom or decorative fonts | Only brand-approved typefaces |
| Letter-spacing on body text | Tracking adjustments are for headings/buttons/overlines only |

### Weight Pairing Guidelines

| Heading Weight | Body Weight | Context |
|---------------|-------------|---------|
| Geist 700 (Bold) | Poppins 400 (Regular) | Hero + body |
| Geist 600 (SemiBold) | Poppins 400 (Regular) | Section heading + body |
| Poppins 700 (Bold) | Poppins 400 (Regular) | App heading + body |
| Poppins 600 (SemiBold) | Poppins 400 (Regular) | Subheading + body |
| Poppins 500 (Medium) | Poppins 400 (Regular) | Label + description |

## Motion Rules

### Mandatory Easing

| Context | Required Easing |
|---------|----------------|
| Content entering viewport | `[0.16, 1, 0.3, 1]` (brand) |
| Elements entering screen | `[0, 0, 0.2, 1]` (enter) |
| Elements leaving screen | `[0.4, 0, 1, 1]` (exit) |
| Interactive feedback | Spring physics (brand springs) |

### Duration Limits

| Rule | Limit |
|------|-------|
| Maximum UI transition duration | `0.8s` (reveal) |
| Hover/active feedback | `0.1s - 0.2s` |
| Content entrance | `0.4s - 0.6s` |
| Modal/panel transitions | `0.3s - 0.6s` |
| Page transitions | `0.4s - 0.8s` |

### Prohibited Motion

| Prohibited | Reason |
|-----------|--------|
| Generic `ease-in-out` | Use brand easing `[0.16, 1, 0.3, 1]` |
| `linear` easing | Never feels natural |
| Transitions longer than `0.8s` | Feels sluggish, blocks interaction |
| Infinite animations (except Pulse) | Only `pulse` is allowed to loop |
| Bouncing everything | `bouncy` spring is for button feedback only |
| Animating more than 3 properties | Keep motion focused |
| Motion without a data metaphor | Every animation must map to emerge, flow, or pulse |

### Reduced Motion

All motion must respect `prefers-reduced-motion`:

```tsx
import { useReducedMotion } from "framer-motion"

function AnimatedComponent() {
  const prefersReducedMotion = useReducedMotion()

  return (
    <motion.div
      initial={prefersReducedMotion ? false : { opacity: 0, y: 16 }}
      animate={{ opacity: 1, y: 0 }}
      transition={prefersReducedMotion ? { duration: 0 } : { duration: 0.6 }}
    />
  )
}
```

## Gradient Rules

### Mandatory

| Rule | Specification |
|------|--------------|
| Primary gradient angle | Always `135deg` |
| Gradient stops | `#0033FE 0%` to `#0BF1C3 100%` |
| Hover gradient | Reverse stops (`#0BF1C3 0%` to `#0033FE 100%`) |
| Vertical sections | `180deg` direction only |
| Radial glow | `circle` shape only |

### Prohibited Gradient Uses

| Prohibited | Reason |
|-----------|--------|
| Changing the 135deg angle for primary | Breaks visual consistency |
| Using solid blue where gradient is expected | Hero CTAs and brand surfaces must use gradient |
| Non-brand colors in gradient stops | Only `#0033FE` and `#0BF1C3` |
| Gradient on form inputs or controls | Gradients are for CTAs and display, not form elements |
| Three-color gradients | Brand gradient is strictly two-color |
| Opacity below 80% on gradient CTAs | Gradient buttons must be fully opaque |

## Dark Mode Rules

### Landing Page

| Rule | Detail |
|------|--------|
| Landing page is **always dark** | `data-theme="dark"` is hardcoded |
| Logo is always white | `text-white` on all logo variants |
| No theme toggle on landing | Dark mode is forced, not optional |
| Brand gradient renders on dark | Never display gradient on white landing |

### Application UI

| Rule | Detail |
|------|--------|
| Design dark first | All components designed on `neutral-950` first |
| Light mode is adaptation | Use lighter tints, not different color palettes |
| Same tokens, different values | CSS variables switch between modes |
| Test dark before light | Dark mode is the primary QA surface |

### Prohibited Dark Mode Practices

| Prohibited | Reason |
|-----------|--------|
| Conditionally swapping logo color on landing | Landing is forced dark — always use white |
| Using `#000000` as dark background | Use `neutral-950` (`#020617`) for blue undertone |
| Using pure white (`#ffffff`) text on dark | Use `neutral-100` (`#f1f5f9`) for softer contrast |
| Designing light-first, then adapting to dark | Dark-first ensures gradient and glassmorphism work correctly |

## Accessibility Requirements

### Minimum Standards

| Standard | Requirement |
|----------|-------------|
| **WCAG Version** | 2.1 AA minimum |
| **Normal text contrast** | >= 4.5:1 |
| **Large text contrast** | >= 3:1 (>= 18px bold or >= 24px regular) |
| **Focus indicators** | Visible focus ring on all interactive elements |
| **Keyboard navigation** | All interactive elements reachable via Tab |
| **Screen reader support** | Meaningful `aria-label` on all icons and non-text elements |
| **Color independence** | Information never conveyed by color alone |

### Brand Color Accessibility

| Combination | Contrast | WCAG AA |
|-------------|----------|---------|
| `blue-500` on white | 6.4:1 | Pass |
| `blue-500` on `blue-50` | 5.8:1 | Pass |
| White on `blue-500` | 6.4:1 | Pass |
| `cyan-500` on `neutral-950` | 12.1:1 | Pass |
| `cyan-500` on white | 1.7:1 | **Fail** |
| `neutral-900` on white | 16.1:1 | Pass |

<Warning>
  `cyan-500` (`#0BF1C3`) fails WCAG contrast on light backgrounds. Use only on dark backgrounds or as decorative/gradient fills.
  For links and interactive text on light backgrounds, always use `blue-500` (`#0033FE`).
</Warning>

### Focus Ring Specification

```css
/* Brand focus ring */
outline: 2px solid #0033FE;
outline-offset: 2px;

/* Tailwind */
focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-brand-blue-500
```

### Reduced Motion

All animated content must degrade gracefully when `prefers-reduced-motion: reduce` is active:
- Disable entrance animations (show content immediately)
- Disable Pulse animations
- Disable WebGL shaders (show CSS fallback)
- Keep essential functional transitions (dropdown open/close) but set duration to `0`

## Platform Consistency

### Same Tokens Everywhere

| Platform | Token Source | Rendering |
|----------|-------------|-----------|
| **Web** | CSS custom properties + Tailwind | Browser rendering |
| **Mobile (React Native)** | JS token export from `@nebutra/brand` | Native rendering |
| **Email** | Inline HEX values from brand spec | HTML email rendering |
| **PDF** | HEX values from brand spec | PDF rendering |
| **Figma** | Synced design tokens | Design tool rendering |

### Cross-Platform Rules

- Use identical HEX values across all platforms — no "close enough" approximations
- Font stacks must include platform-appropriate fallbacks
- Gradients render as CSS on web, as image assets on email
- Motion tokens apply to web and mobile — email and PDF are static

## Compliance Checklist

Use this checklist before shipping any Nebutra-branded surface:

| Check | Requirement |
|-------|-------------|
| Logo clearance >= 1/4 height | All sides |
| Logo height >= 35px (digital) | Minimum size |
| Primary color is `#0033FE` | Not approximated |
| Accent color is `#0BF1C3` | Not approximated |
| Neutrals have blue undertone | Not pure gray |
| Info semantic = brand blue | Not sky/teal |
| Gradient angle is 135deg | Primary gradient |
| Geist Sans for display text | Not Poppins |
| Poppins for body text | Not Inter/Roboto |
| Brand easing curves used | Not generic ease |
| Max transition duration <= 0.8s | No sluggish motion |
| Dark-first design verified | Primary QA surface |
| WCAG AA contrast passed | 4.5:1 text, 3:1 large |
| `prefers-reduced-motion` handled | All animations |
| Focus rings visible | All interactive elements |
| Cyan not used as text on light | Dark backgrounds only |
