---
title: "FilterBar"
description: "Search input combined with filter controls for narrowing lists and table views."
---

## Demo

```tsx
<FilterBar
  searchPlaceholder="Search pipelines..."
  filters={[
    {
      key: "status",
      label: "Status",
      options: [
        { value: "active", label: "Active" },
        { value: "paused", label: "Paused" },
        { value: "error", label: "Error" },
      ],
    },
    {
      key: "env",
      label: "Environment",
      options: [
        { value: "production", label: "Production" },
        { value: "staging", label: "Staging" },
      ],
    },
  ]}
  onSearchChange={(q) => setQuery(q)}
  onFilterChange={(key, value) => setFilters((f) => ({ ...f, [key]: value }))}
/>
```

## Usage

```tsx
import { FilterBar } from "@nebutra/custom-ui";

export function ResourceList() {
  const [query, setQuery] = useState("");
  const [filters, setFilters] = useState<Record<string, string>>({});

  const filtered = resources.filter(
    (r) =>
      r.name.toLowerCase().includes(query.toLowerCase()) &&
      Object.entries(filters).every(([k, v]) => !v || r[k] === v),
  );

  return (
    <>
      <FilterBar
        searchPlaceholder="Search resources..."
        filters={filterDefs}
        onSearchChange={setQuery}
        onFilterChange={(key, value) =>
          setFilters((prev) => ({ ...prev, [key]: value }))
        }
      />
      <ResourceTable data={filtered} />
    </>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `searchPlaceholder` | `string` | `"Search..."` | Placeholder text inside the search field |
| `searchValue` | `string` | — | Controlled value for the search input |
| `onSearchChange` | `(value: string) => void` | — | Called on every keystroke |
| `filters` | `FilterDefinition[]` | `[]` | Array of filter group definitions |
| `activeFilters` | `Record<string, string>` | `{}` | Controlled active filter state |
| `onFilterChange` | `(key: string, value: string) => void` | — | Called when a filter selection changes |
| `onReset` | `() => void` | — | Called when the "Clear filters" button is clicked |
| `showReset` | `boolean` | `true` | Whether to show the reset button when filters are active |
| `className` | `string` | — | Additional classes on the root container |

### `FilterDefinition`

```ts
interface FilterDefinition {
  key: string;
  label: string;
  options: Array<{ value: string; label: string }>;
}
```

## Examples

### Search-only (no filters)

```tsx
<FilterBar
  searchPlaceholder="Find a member..."
  onSearchChange={setQuery}
  filters={[]}
/>
```

### Controlled with active filter state

```tsx
<FilterBar
  searchValue={query}
  onSearchChange={setQuery}
  filters={filterDefs}
  activeFilters={activeFilters}
  onFilterChange={(key, value) =>
    setActiveFilters((prev) => ({ ...prev, [key]: value }))
  }
  onReset={() => {
    setQuery("");
    setActiveFilters({});
  }}
/>
```

## Accessibility

- The search input has `role="searchbox"` and `aria-label="Search"`.
- Each filter dropdown has an associated visible `<label>`.
- Active filter count is announced via `aria-live="polite"` when filters change.
- The reset button has `aria-label="Clear all filters"`.
- All interactive elements are keyboard-accessible and follow logical tab order.
