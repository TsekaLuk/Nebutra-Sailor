---
title: "InfoTooltip"
description: "Hover tooltip triggered by an info icon, used to surface explanatory content without cluttering the UI."
---

## Demo

```tsx
<FormItemLayout
  label="Retention period"
  htmlFor="retention"
  labelSuffix={
    <InfoTooltip content="Data older than this threshold is automatically purged from storage." />
  }
>
  <Select id="retention" options={retentionOptions} />
</FormItemLayout>
```

## Usage

```tsx
import { InfoTooltip } from "@nebutra/ui";

export function PricingRow({ label, tooltip, value }: PricingRowProps) {
  return (
    <div className="flex items-center justify-between">
      <span className="flex items-center gap-1">
        {label}
        {tooltip && <InfoTooltip content={tooltip} />}
      </span>
      <span>{value}</span>
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `content` | `string \| React.ReactNode` | — | **Required.** Tooltip body content |
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"top"` | Preferred placement of the tooltip |
| `align` | `"start" \| "center" \| "end"` | `"center"` | Alignment relative to the trigger |
| `delayDuration` | `number` | `300` | Milliseconds before the tooltip appears |
| `iconSize` | `"sm" \| "md"` | `"sm"` | Size of the trigger info icon |
| `iconClassName` | `string` | — | Additional classes on the trigger icon |
| `className` | `string` | — | Additional classes on the tooltip content panel |

## Examples

### Rich content tooltip

```tsx
<InfoTooltip
  content={
    <div className="space-y-1">
      <p className="font-medium">Ingestion latency</p>
      <p className="text-xs text-gray-400">
        Time from event receipt to availability in queries. P99 target: 500ms.
      </p>
    </div>
  }
  side="right"
/>
```

### Left-aligned, bottom placement

```tsx
<InfoTooltip
  content="Pricing is calculated per million events ingested."
  side="bottom"
  align="start"
  delayDuration={0}
/>
```

### Larger trigger icon

```tsx
<InfoTooltip
  content="This setting affects all pipelines in the workspace."
  iconSize="md"
/>
```

## Accessibility

- The trigger icon button has `aria-label="More information"` and `type="button"`.
- The tooltip panel has `role="tooltip"` and is linked to the trigger via `aria-describedby`.
- Tooltip content is announced by screen readers when the trigger is focused via keyboard.
- Tooltip opens on both `hover` (mouse) and `focus` (keyboard) events.
- Pressing `Escape` dismisses an open tooltip.
- Icons inside the tooltip are `aria-hidden="true"`.
