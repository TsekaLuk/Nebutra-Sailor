## When to Use

Use chart patterns when you need to render quantitative data that benefits from visual comparison — time-series trends, proportional breakdowns, or multi-series comparisons.

Use a chart when:
- The data has at least 3 data points and a comparison axis (time, category)
- The insight is relational ("X is growing faster than Y")
- A table of the same data would require mental arithmetic the chart eliminates

Do not use a chart when:
- A single number or a `CounterLabel` communicates the same thing
- The data is static reference material
- The user needs the exact values (use a table alongside or instead)

## Structure

```
ChartContainer (owns ChartConfig, sets aspect ratio)
  └── ResponsiveContainer (fills parent width)
       └── BarChart | LineChart | AreaChart | PieChart
            ├── CartesianGrid (optional, muted stroke)
            ├── XAxis + YAxis (both optional for sparklines)
            ├── ChartTooltip + ChartTooltipContent
            ├── ChartLegend + ChartLegendContent (multi-series only)
            └── Bar | Line | Area (one per data series)
```

`ChartContainer` wraps every chart. It injects `ChartConfig` into context, applies CSS custom properties for per-series color tokens, and enforces the `aspect-video` default aspect ratio.

## Composition

### ChartConfig

Define one entry per data series. Map each key to a label and a color token or theme object.

```tsx
import { ChartConfig } from "@nebutra/custom-ui/primitives/chart";

const chartConfig: ChartConfig = {
  apiCalls: {
    label: "API Calls",
    color: "hsl(var(--chart-1))",
  },
  tokens: {
    label: "AI Tokens",
    color: "hsl(var(--chart-2))",
  },
} satisfies ChartConfig;
```

Theme-aware colors (light/dark variants):

```tsx
const chartConfig: ChartConfig = {
  revenue: {
    label: "Revenue",
    theme: {
      light: "#0070F3",
      dark: "#3291FF",
    },
  },
};
```

### ChartContainer

```tsx
import {
  ChartContainer,
  ChartTooltip,
  ChartTooltipContent,
} from "@nebutra/custom-ui/primitives/chart";
import { BarChart, Bar, CartesianGrid, XAxis } from "recharts";

<ChartContainer config={chartConfig} className="h-[300px]">
  <BarChart data={data}>
    <CartesianGrid vertical={false} />
    <XAxis dataKey="month" tickLine={false} axisLine={false} />
    <ChartTooltip content={<ChartTooltipContent />} />
    <Bar dataKey="apiCalls" fill="var(--color-apiCalls)" radius={4} />
  </BarChart>
</ChartContainer>
```

### Tooltip

`ChartTooltipContent` reads labels and colors from `ChartConfig` automatically. Pass `indicator="line"` for line charts and `indicator="dot"` (default) for bar charts.

### Legend

Add `ChartLegend` and `ChartLegendContent` when a chart has two or more series. Position the legend at `verticalAlign="bottom"` for horizontal bar and line charts; omit it for single-series charts.

## Examples

### Single-series bar chart (monthly API calls)

```tsx
const data = [
  { month: "Jan", apiCalls: 1200 },
  { month: "Feb", apiCalls: 1800 },
  { month: "Mar", apiCalls: 1500 },
];

const config: ChartConfig = {
  apiCalls: { label: "API Calls", color: "hsl(var(--chart-1))" },
};

<ChartContainer config={config} className="h-[240px]">
  <BarChart data={data}>
    <CartesianGrid vertical={false} />
    <XAxis dataKey="month" tickLine={false} axisLine={false} />
    <ChartTooltip content={<ChartTooltipContent />} />
    <Bar dataKey="apiCalls" fill="var(--color-apiCalls)" radius={4} />
  </BarChart>
</ChartContainer>
```

### Multi-series line chart

```tsx
const config: ChartConfig = {
  apiCalls: { label: "API Calls", color: "hsl(var(--chart-1))" },
  tokens:   { label: "AI Tokens", color: "hsl(var(--chart-2))" },
};

<ChartContainer config={config} className="h-[300px]">
  <LineChart data={data}>
    <CartesianGrid strokeDasharray="3 3" />
    <XAxis dataKey="month" />
    <ChartTooltip content={<ChartTooltipContent indicator="line" />} />
    <ChartLegend content={<ChartLegendContent />} />
    <Line type="monotone" dataKey="apiCalls" stroke="var(--color-apiCalls)" />
    <Line type="monotone" dataKey="tokens"   stroke="var(--color-tokens)" />
  </LineChart>
</ChartContainer>
```
