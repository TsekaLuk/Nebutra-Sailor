## When to Use

Apply layout patterns any time you compose a new page or a major page region. Layout decisions affect information hierarchy, responsive behavior, and spacing consistency across the application.

Use this pattern for:
- Full application pages (dashboard, settings, detail views)
- Content sections within a page (grouped fields, data regions)
- Marketing and documentation page shells

## Structure

### Dashboard page layout

```
<div className="min-h-screen bg-gray-50">       ← page shell
  <Header>                                        ← global nav
  <main>
    <Container size="xl">                         ← width constraint
      <PageHeader title="…" breadcrumbs={…} />   ← page-level heading
      <Section title="Overview">                  ← logical group
        <div className="grid gap-6 md:grid-cols-3">
          <Card>…</Card>
          <Card>…</Card>
          <Card>…</Card>
        </div>
      </Section>
      <Section title="Recent activity">
        {/* table or list */}
      </Section>
    </Container>
  </main>
</div>
```

### Two-column settings layout

```
<Container size="lg">
  <PageHeader title="Settings" />
  <div className="grid gap-8 md:grid-cols-[240px_1fr]">
    <NavList>                         ← settings nav (inner side menu)
      <NavList.Item href="/settings/general">General</NavList.Item>
      <NavList.Item href="/settings/api-keys">API Keys</NavList.Item>
    </NavList>
    <div>
      <Section title="General">…</Section>
    </div>
  </div>
</Container>
```

## Composition

### Container

`Container` constrains maximum width and applies responsive horizontal padding (`px-4 sm:px-6 lg:px-8`). Use it as the outermost wrapper inside `<main>`.

```tsx
import { Container } from "@nebutra/design-system";

// xl = max-w-7xl (1280px) — standard dashboard width
<Container size="xl">…</Container>

// lg = max-w-5xl (1012px) — settings, forms
<Container size="lg">…</Container>

// md = max-w-3xl — content-heavy single-column pages
<Container size="md">…</Container>
```

Size reference:

| Size | Max-width | Use |
|---|---|---|
| `sm` | 544px | Narrow modals, confirmation dialogs |
| `md` | 768px | Single-column content |
| `lg` | 1012px | Settings, forms, docs |
| `xl` | 1280px | Full dashboard pages |
| `full` | 100% | Edge-to-edge tables, full bleed sections |

### PageHeader

Place `PageHeader` immediately inside `Container`, before the first `Section`. It owns the page-level title, description, breadcrumb trail, and primary page actions.

```tsx
import { PageHeader } from "@nebutra/design-system";
import { Button } from "@nebutra/design-system";

<PageHeader
  title="API Keys"
  description="Manage programmatic access to the Nebutra API."
  breadcrumbs={[
    { label: "Settings", href: "/settings" },
    { label: "API Keys" },
  ]}
  actions={
    <Button variant="primary" onClick={onCreate}>
      New key
    </Button>
  }
/>
```

### Section

`Section` groups related content under an `h2` heading. It adds `mb-6` spacing below itself and a flex row for the heading and optional actions.

```tsx
import { Section } from "@nebutra/design-system";

<Section title="Usage this month" description="Resets on the 1st.">
  {/* charts, cards, table */}
</Section>
```

When sections need no visible heading (e.g., a card grid that follows a `PageHeader`), omit all Section props and use the element as a spacing wrapper only.

### Grid layouts

Use Tailwind grid utilities for responsive card grids. The spacing primitives define a 4px base unit — use `gap-6` (24px) for card grids and `gap-4` (16px) for compact form rows.

```tsx
// 3-column stat card grid
<div className="grid gap-6 md:grid-cols-2 lg:grid-cols-3">
  <Card>…</Card>
  <Card>…</Card>
  <Card>…</Card>
</div>

// 2-column form layout
<div className="grid gap-4 sm:grid-cols-2">
  <FormControl>…</FormControl>
  <FormControl>…</FormControl>
</div>
```

Reference grid presets in `primitives/layout.ts`: `gridLayouts.two`, `gridLayouts.three`, `gridLayouts.four`, `gridLayouts.auto`.

### Spacing scale

The design system uses a 4px base unit. Common values:

| Token | px | Tailwind | Use |
|---|---|---|---|
| `spacing[3]` | 16 | `gap-4` | Compact form rows |
| `spacing[4]` | 24 | `gap-6` | Card grids, section gaps |
| `spacing[5]` | 32 | `gap-8` | Between sections |
| `spacing[8]` | 64 | `pt-16` | Page-level vertical breathing room |

### Z-index layering

Use the z-index scale from `primitives/layout.ts` for overlapping layers. Do not use arbitrary z-index values.

| Layer | Value | Use |
|---|---|---|
| `base` | 0 | Normal document flow |
| `dropdown` | 1000 | ActionMenu, Select dropdown |
| `sticky` | 1020 | Sticky table headers |
| `fixed` | 1030 | Global navigation bar |
| `modalBackdrop` | 1040 | Dialog/Drawer overlay |
| `modal` | 1050 | Dialog/Drawer panel |
| `tooltip` | 1070 | Tooltip |

## Examples

### Dashboard home page

The dashboard page at `apps/web/src/app/(dashboard)/page.tsx` follows this structure:

```tsx
<div className="min-h-screen bg-gray-50">
  <main className="mx-auto max-w-7xl px-4 py-6 sm:px-6 lg:px-8">
    <PageHeader title="Dashboard" />
    <div className="grid gap-6 md:grid-cols-2 lg:grid-cols-3">
      <Card>Welcome card</Card>
      <Card>API Calls stat</Card>
      <Card>AI Tokens stat</Card>
    </div>
  </main>
</div>
```

Replace `mx-auto max-w-7xl` with `<Container size="xl">` in new pages to inherit future Container changes automatically.
